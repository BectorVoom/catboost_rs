# Phase 6.6 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Source: 06.6-RESEARCH.md §Validation Architecture.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Rust built-in `cargo test` + `approx` 0.5.1 (`abs_diff_eq!`) |
| **Config file** | none (workspace `Cargo.toml`; rustfmt/clippy defaults) |
| **Quick run command** | `cargo test -p <crate-under-change>` (e.g. `cargo test -p cb-train`) |
| **Full suite command** | `cargo test --workspace` (per-crate if disk pressure blocks linking) |
| **Estimated runtime** | ~per-crate seconds to low minutes; workspace longer |

---

## Sampling Rate

- **After every task commit:** Run the specific new oracle test (`cargo test -p <crate> --test <name>`)
- **After every plan wave:** Run the changed crate's full suite + the no-regression locks (`fstr_oracle_test`, `shap_oracle_test`, ordered/tensor-CTR e2e oracles)
- **Before `/gsd-verify-work`:** `cargo test --workspace` green (per-crate fallback under disk pressure)
- **Max feedback latency:** keep per-task commands per-crate; avoid full-workspace links on every commit
