## Fixed Issues

### CR-01: Bare slice indexing in production violates `indexing_slicing = "deny"`

**Files modified:** `crates/cb-train/src/tree.rs`, `crates/cb-model/src/json.rs`, `crates/cb-train/src/feature_selection.rs`
**Commit:** `1ef9d50`
**Applied fix:** Replaced every bare `Vec` index at the listed sites with checked access in the
file's established style:
- `tree.rs:973` `node_depth[id]` → `node_depth.get(id).copied().unwrap_or(0)`
- `tree.rs:990-991` `nodes[id]=...; node_split[id]=...` → `if let Some(slot) = nodes.get_mut(id)` / `node_split.get_mut(id)`
- `tree.rs:1012` `node_docs[leaf].clone()` → `node_docs.get(leaf).cloned().unwrap_or_default()`
- `tree.rs:1090` `node_depth[node]` → `node_depth.get(node).copied().unwrap_or(0)`
- `tree.rs:1095` `&node_docs[node]` → `let Some(docs) = node_docs.get(node) else { return; }`
- `tree.rs:1155` `node_id_to_leaf_id[id]=u32::MAX` → `if let Some(slot) = node_id_to_leaf_id.get_mut(id)`
- `tree.rs:1186-1187` `node_to_leaf[id]=...; node_id_to_leaf_id[id]=...` → checked `get_mut` for both
- `json.rs:275` `node_id_to_leaf_id[id]=...` → `if let Some(slot) = node_id_to_leaf_id.get_mut(id)` (preserving the `u32::try_from` error path)
- `feature_selection.rs:252,326` `retained[g]` → `retained.get(g).copied().unwrap_or(false)`

All indices are constructed in-bounds today, so the checked forms preserve byte-identical
behavior on every valid model/training path. Scanned the same finalization loop and functions
for additional bare `[idx]` — none found. No `unwrap()`/`expect()` introduced.

### WR-01: `use_best_model` never truncates non-symmetric models

**Files modified:** `crates/cb-train/src/boosting.rs`
**Commit:** `fe0e5c9`
**Applied fix:** Added `non_symmetric_trees.truncate(best + 1);` alongside the existing
`trees.truncate(best + 1);`, inside the existing `if let Some(best) = best_model.best_iteration()`
guard. No-op when `best_iteration()` is `None` or `non_symmetric_trees` is empty, so existing
oracle tests are unaffected.

### WR-02: ragged non-symmetric `n_leaves` mis-strides

**Files modified:** `crates/cb-model/src/apply.rs`
**Commit:** `f32a4ec`
**Applied fix:** Guarded the multi-dim non-symmetric `n_leaves` computation per the reviewer
snippet — `if dim == 0 || tree.leaf_values.len() % dim != 0 { 0 } else { len / dim }`. Only
affects malformed (ragged-buffer) loaded models; valid models follow the unchanged divisor path,
so oracles are unaffected.

### WR-04: pure-leaf sentinel split treated as real feature 0

**Files modified:** `crates/cb-model/src/fstr.rs`, `crates/cb-model/src/shap.rs`
**Commit:** `157305e`
**Applied fix:** Added a `matches!(tree.step_nodes.get(node_idx), Some(&(0, 0)))` short-circuit
BEFORE reading the split feature in `interaction_dfs` (after the terminal accumulation) and in
`shap_recurse_non_symmetric` (after the terminal SHAP distribution). Per IN-04 the descent was
already correct (a `(0,0)` node has both child diffs zero, so the descent loop was already a
no-op), making this guard behavior-preserving. Confirmed by the passing fstr/advanced-fstr/shap
oracle tests.

### WR-05: feature-selection rounding shortfall not surfaced

**Files modified:** `crates/cb-train/src/feature_selection.rs`
**Commit:** `f8a7ed1`
**Applied fix:** Added a post-elimination-loop invariant — `if eliminated_features.len() !=
n_to_eliminate_total { return Err(CbError::Degenerate(...)); }`. The in-scope oracle fixture
eliminates the exact budget, so the check passes for it (confirmed by the passing
`feature_selection_oracle_test`). Flagged `requires human verification` because it is a logic
guard whose invariant semantics should be confirmed against upstream behavior.
