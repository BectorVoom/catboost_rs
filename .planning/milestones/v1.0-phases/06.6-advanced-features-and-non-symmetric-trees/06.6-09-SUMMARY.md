# Phase 06.6 Plan 09: Non-Symmetric Grower .cbm Save Gap-Closure Summary

**Fixed the CR-02 interior-sentinel mismatch so a Rust leaf-wise-grower-trained non-symmetric model saves to .cbm, reloads, and re-predicts within 1e-5 of the in-memory model — plus a CR-01 checked-u16 step-diff hardening.**

## Accomplishments
- `leaf_wise_grower` finalization now initializes `node_id_to_leaf_id` to `vec![u32::MAX; node_count]` and sets `BuiltNode::Interior` slots to `u32::MAX` explicitly, so the cbm serializer's `distinct_leaves` filter (cbm.rs:200) counts only real leaves. `save_cbm` on a grower-trained non-symmetric model now returns `Ok` instead of failing with `ModelError::SchemaVersion` (CR-02 closed).
- Step-node child diffs now convert via checked `u16::try_from(...).map_err(|_| CbError::OutOfRange(...))?`, surfacing extreme-`max_depth` overflow as a typed error instead of silently truncating the tree topology (CR-01 closed). No new error variant added.
- New oracle test `non_symmetric_grower_save_load_predict_roundtrip` trains via `grow_policy=Depthwise`, lifts (`from_trained`), `save_cbm` → `load_cbm` → `predict_raw`, and asserts reloaded predictions match the in-memory model ≤1e-5 — the SC-3 / FEAT-06 grower-produced path the prior tests never covered (they round-tripped an upstream-loaded model).
