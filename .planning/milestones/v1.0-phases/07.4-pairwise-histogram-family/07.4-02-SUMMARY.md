## Decisions Made
- **Separate symbol + arm despite identical MVP body (D-7.4-02):** kept `pairwise_hist_8bit_atomics_kernel` and `launch_pairwise_hist_8bit_*` fully distinct from the non-binary path — structural parity with upstream's own file split, and the right seam for the deferred per-thread cached-bin perf work. The MVP body equals the non-binary kernel with bits=8 because the shipped MVP already uses direct global atomics for every width.
- **`n_bins` fixed at comptime 256, no `bits` runtime arg:** the 8-bit-atomics family IS bits=8, so the launch arm takes no `bits` parameter and the kernel hard-codes `comptime!(256usize)`; the host arm validates `n_bins == 256` (`CbError::Degenerate` otherwise).
- **FROZEN 4-channel layout reused verbatim:** index `(feature*256+bin)*4+histId`, buffer length `n_features*256*4` via `pair_hist_binsums_len` — never `* 2` (Pitfall 2).
- **n_objects=300 in the fixture:** chosen `> 256` so the deterministic feature-major bin spread exercises the full 8-bit range (bins 0..255), not just a 5-bit slice.
