## Accomplishments
- `pairwise_hist_binary_kernel<F>` — the DISTINCT binary family (D-7.4-02): a SEPARATE `#[cube(launch)]` symbol with `n_bins` fixed at `comptime!(2usize)` (the 2-bin line, a bin COUNT, NOT a warp literal — D-09) and the read bins masked to the bit (`& comptime!(1u32)`, the shipped 7.3 binary precedent at `pointwise_hist2_binary_kernel`). It accumulates each per-pair weight into the FROZEN 4-channel cell via direct `Atomic<F>::fetch_add` over the non-one-hot Compare->histId mapping. The binary family takes NO `one_hot` arg — upstream `pairwise_hist_binary.cu` has no one-hot variant (RESEARCH Pattern 2) — so the kernel hard-codes the non-one-hot `(ge, gt)` flag-collapsed writes (exactly the `else`-branch of the non-binary kernel). The upstream warp-tile 2x2 `(invBin1&invBin2)|((invBin1&bin2)<<8)|((bin1&invBin2)<<16)|((bin1&bin2)<<24)` cross-product reduces to that same per-pair Compare predicate — validated bit-exact by the self-oracle.
- The 2-bin device-resident seam: `launch_pairwise_hist_binary_handle`/`_into` (no readback on the hand-off), `launch_pairwise_hist_binary` readback wrapper — clones of the Plan C half-byte arm with `n_bins` pinned to `2` (validated host-side), no `one_hot` parameter, and the binary kernel dispatched, reusing `pair_hist_binsums_len`, the full guard block (LengthMismatch / OutOfRange / `checked_mul` overflow / value-range / empty short-circuit), the backend-dispatched channel (f64 on rocm/cuda/cpu, f32 on wgpu), and the no-readback seam UNCHANGED.

## Decisions Made
- **Separate symbol + arm with no one-hot (D-7.4-02):** kept `pairwise_hist_binary_kernel` and `launch_pairwise_hist_binary_*` fully distinct from the non-binary/8-bit/half-byte paths — structural parity with upstream's own file split. The MVP body equals the non-binary kernel's non-one-hot path with the bit mask and n_bins fixed at 2; the binary family takes NO one-hot overlay (no upstream one-hot .cu).
- **`n_bins` fixed at comptime 2, no `bits` runtime arg:** the binary family IS 1-bit/2-bin, so the launch arm takes no `bits` parameter and the kernel uses `comptime!(2usize)`; the host arm validates `n_bins == 2` (`CbError::Degenerate` otherwise). The 2 is a bin COUNT, never a warp literal (D-09).
- **Bit mask (& 1) on the read bins:** carried from the shipped 7.3 pointwise binary precedent; the host reference indexes RAW, so the host-side value-range guard (`cindex < 2`) keeps the kernel and reference in lock-step.
- **The 2x2 cross-product reduces to the shared Compare predicate:** upstream's `(invBin1&invBin2)|((invBin1&bin2)<<8)|((bin1&invBin2)<<16)|((bin1&bin2)<<24)` is the warp-tile-distributed expression of the per-pair `(i leq/ge)x(j leq/ge)` channel selection; distilled to the per-pair `ge=(b1>=b2), gt=(b1>b2)` flag-collapsed writes it equals the non-one-hot Compare->histId mapping the other families share. Validated bit-exact (0.0 divergence).
- **FROZEN 4-channel layout reused verbatim:** index `(feature*2+bin)*4+histId`, buffer length `n_features*2*4` via `pair_hist_binsums_len` — never `* 2` (Pitfall 2; the only `2usize *` in the kernel is the in-channel `2*ge`/`2*gt` histId selector, not the layout multiplier).
