## Decisions Made
- **Confirm-and-oracle, not new kernel code:** Plans A/B already shipped the `#[comptime] one_hot` overlay on both kernels and all launch helpers (verified by grep + the upstream `Compare` predicate). Plan E's substantive deliverable is therefore the dedicated `one_hot` self-oracle and the no-op-detection swap assertion, not new kernel logic — exactly the plan's `<action>` ("if Plan A/B already plumbed it, no change beyond confirming").
- **SWAP sub-assertion is the load-bearing test (T-07.4-21):** a comptime flag that silently equals the non-one-hot path would still pass a naive same-predicate oracle (host and kernel share `add_pair_contrib`). The swap assertion — one_hot=true vs false MUST differ on a bin1!=bin2 fixture, AND the two host references must differ — closes that spoofing gap independent of the GPU.
- **Half-byte/binary excluded from one-hot:** their launch helpers take no `one_hot` arg (no upstream `*_one_hot.cu`); the `one_hot` test does not call them, keeping the variant set faithful (RESEARCH Pattern 2).
