# Phase 7.6 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Rust built-in `#[test]` (no external runner) |
| **Config file** | none — kernel oracles are `#[cfg(test)] mod` in `crates/cb-backend/src/kernels.rs` and submodules |
| **Quick run command** | `cargo test -p cb-backend --no-default-features --features rocm <module>::<test> -- --nocapture` |
| **Full suite command** | `cargo test -p cb-backend --no-default-features --features rocm -- --nocapture` |
| **Estimated runtime** | ~60–120 seconds (rocm in-env, gfx1100) |

**CRITICAL:** Always `--no-default-features --features rocm` — the default feature is `cpu`, which would silently measure CPU-vs-CPU (always bit-exact) and produce a meaningless gate. Per-crate invocation only (root disk ~100% full; full-workspace links fail).

---

## Sampling Rate

- **After every task commit:** Run `cargo test -p cb-backend --no-default-features --features rocm <touched-module> -- --nocapture` (per-crate; disk pressure)
- **After every plan wave:** Run full `cargo test -p cb-backend --no-default-features --features rocm -- --nocapture`
- **Before `/gsd-verify-work`:** Full rocm suite must be green in-env
- **Max feedback latency:** ~120 seconds

> **Orchestrator/human-run gate (D-06):** The rocm suite CANNOT run in a verifier subagent (no GPU access) and is NEVER run in GitHub Actions. The orchestrator (or human) executes it in-env on gfx1100 and records results.
