## Pattern Assignments

### `crates/cb-backend/src/gpu_runtime.rs` (service, transform — the load-bearing 7.2 work)

**Analog:** `gpu_runtime.rs::launch_block_reduce_f64` (lines 92–132, same file) — the 7.1 generic
launch shell over `SelectedRuntime`. Mirror it EXACTLY for the der launch family
(`launch_der_binary`, `launch_der_param`, `launch_der_unary` + a handle-returning seam variant).

**Imports + production-lint discipline** (lines 35–41) — copy verbatim:
```rust
use cubecl::features::{AtomicUsage, Plane};
use cubecl::prelude::*;
use cb_core::{CbError, CbResult};
use crate::kernels::{block_reduce_atomic_kernel, block_reduce_kernel, block_scan_kernel};
use crate::SelectedRuntime;
```
The der family adds `use crate::kernels::{gradient_kernel, logloss_gradient_kernel,
logloss_hessian_kernel, quantile_gradient_kernel, focal_gradient_kernel, focal_hessian_kernel};`.
**No `unwrap`/`expect`/`panic`/indexing in this production file** (workspace lints + D-13) — the
file-header comment block (lines 11–33) already states this; preserve that discipline.

**Per-call client + generic device launch pattern** (lines 98–123) — the seam to mirror:
```rust
let device = <SelectedRuntime as cubecl::Runtime>::Device::default();
let client = <SelectedRuntime as cubecl::Runtime>::client(&device);

let in_handle = client.create(cubecl::bytes::Bytes::from_elems(input.to_vec()));
let num_cubes = n.div_ceil(CUBE_DIM).max(1);
let out_handle = client.empty(num_cubes * std::mem::size_of::<f64>());
// ... (for der: out_handle is n*size_of::<f64>(), NOT num_cubes — elementwise, same length as input)

let count = CubeCount::Static(num_cubes as u32, 1, 1);
let dim = CubeDim { x: CUBE_DIM as u32, y: 1, z: 1 };

block_reduce_kernel::launch::<f64, SelectedRuntime>(
    &client, count, dim,
    unsafe { ArrayArg::from_raw_parts(in_handle, n) },
    unsafe { ArrayArg::from_raw_parts(out_handle.clone(), num_cubes) },
    use_plane,
);
```
Key generalization vs the CPU spike: `<SelectedRuntime as cubecl::Runtime>::Device::default()` +
`::client(&device)`, then `kernel::launch::<f64, SelectedRuntime>(...)`. Clone the output handle
into `from_raw_parts` so the original stays readable/returnable (the 7.1 idiom, line 121).

**CUBE_DIM launch geometry** (line 63) — reuse the existing const:
```rust
const CUBE_DIM: usize = 32;   // launch width, NOT a wave/warp-size reduction stride (D-09)
```
Der kernels are elementwise, so the CR-02 `CUBE_DIM <= BLOCK_REDUCE_SHMEM` assert (lines 71–75) is
**irrelevant** (no shared memory) — do not copy it into the der helpers.

**WR-05 typed-error read-back** (lines 128–131) — for the **self-oracle wrapper only**:
```rust
let bytes = client
    .read_one(out_handle)
    .map_err(|e| CbError::Degenerate(format!("CubeCL device read-back failed: {e:?}")))?;
Ok(bytemuck::cast_slice::<u8, f64>(&bytes).to_vec())
```
**Do NOT call `read_one` on the histogram hand-off path** (Pitfall 2 / SC-3): the handle-returning
seam returns `out_handle.clone()` (or the un-read handle) DIRECTLY as `CbResult<Handle>`. Read-back
lives only in the host wrapper the self-oracle calls.

**Parametric kernels — length-1 `Array<F>` device buffers** (from `cpu_runtime::launch_focal_f64`,
lines 248–249, 268–269 — generalize off `CpuRuntime`):
```rust
let alpha_handle = client.create(cubecl::bytes::Bytes::from_elems(vec![alpha]));
let gamma_handle = client.create(cubecl::bytes::Bytes::from_elems(vec![gamma]));
// ... in the launch:
unsafe { ArrayArg::from_raw_parts(alpha_handle, 1) },
unsafe { ArrayArg::from_raw_parts(gamma_handle, 1) },
```
Quantile/MAE and Focal pass `alpha`/`gamma`/`delta` as length-1 arrays read at index 0 — keeps the
kernel generic over `F: Float` (a scalar arg would force a non-generic `ScalarArgType` bound,
AGENTS.md). MAE routes through `quantile_gradient_kernel` at `(QUANTILE_ALPHA, QUANTILE_DELTA)` —
no separate MAE kernel (WR-04, `compute_gradients_one_dim` line 489–490).

**Constant-der2 handling (no kernel to launch)** — Pitfall 5, mirror `compute_gradients_one_dim`:
- RMSE der2 ≡ `-1.0` (line 468). For the device-handle path, allocate
  `client.create(Bytes::from_elems(vec![-1.0_f64; n]))` so 7.3 still receives a der2 handle.
- Quantile/MAE der2 ≡ `0.0` (lines 492, 501). Allocate `client.create(Bytes::from_elems(vec![0.0_f64; n]))`.
- There is **no** `quantile_hessian_kernel` / `rmse_hessian_kernel` — do not try to launch one.

**Handle type for the seam** (Open Q2 in RESEARCH): inspect the return type of `client.empty(...)`
at implementation time (it is `Clone`, read by `client.read_one(handle)` in 7.1). Name the seam
after that type and keep it `pub(crate)` inside `cb-backend` (D-7.2-05 — never crosses the
`cb-compute` trait).

### `crates/cb-backend/src/kernels/gradient_gpu.rs` (test, transform — NEW all-backend self-oracle)

**Primary analog:** `kernels/reduce.rs` (lines 1–185) and `kernels/scan.rs` (lines 1–185) — the
established all-backend (`#[cfg(test)]`, NOT cpu-gated) self-oracle shape that runs over
`SelectedRuntime` on rocm in-env + wgpu host. **Secondary analog:** `kernels/gradient.rs` (the CPU
spike) for the per-element launch + read-back shape.

**File-header / source-test-separation doc** (reduce.rs lines 1–15) — adapt:
```rust
//! Self-oracle for the GPU der1/der2 kernels (D-7.2-06, GPU-01 grad/hess): the device
//! derivative must match the `cb-compute::loss` CPU baseline within a REPORTED (not
//! signed-off) tolerance, over f32 and f64 including edge cases (empty, n=1, length not a
//! multiple of CUBE_DIM, large N, non-trivial weights).
//!
//! Source/test separation is mandatory (CLAUDE.md / AGENTS.md): the kernels live in
//! `kernels.rs`; all assertions live here. Test code may use `.unwrap()`/indexing (the
//! `lib.rs:1` `#[cfg(test)]` allow) — production `gpu_runtime.rs` may not.
//!
//! Runs on `rocm` in-env (gfx1100, wave32) + `wgpu` host. The reported max abs/rel
//! divergence is informational: the GPU-06 epsilon is signed off in Phase 7.6, NOT here.
```

**Generic-runtime launch + read-back harness** (reduce.rs lines 26–53; gradient.rs lines 13–48):
```rust
fn run_der<F>(approx: &[F], target: &[F]) -> Vec<F>
where F: Float + CubeElement + bytemuck::Pod {
    let n = approx.len();
    let device = <crate::SelectedRuntime as Runtime>::Device::default();
    let client = <crate::SelectedRuntime as Runtime>::client(&device);

    let approx_handle = client.create(cubecl::bytes::Bytes::from_elems(approx.to_vec()));
    let target_handle = client.create(cubecl::bytes::Bytes::from_elems(target.to_vec()));
    let out_handle = client.empty(n * std::mem::size_of::<F>());
    let num_cubes = n.div_ceil(32usize).max(1);

    gradient_kernel::launch::<F, crate::SelectedRuntime>(
        &client,
        CubeCount::Static(num_cubes as u32, 1, 1),
        CubeDim { x: 32u32, y: 1, z: 1 },
        unsafe { ArrayArg::from_raw_parts(approx_handle, n) },
        unsafe { ArrayArg::from_raw_parts(target_handle, n) },
        unsafe { ArrayArg::from_raw_parts(out_handle.clone(), n) },
    );
    let bytes = client.read_one(out_handle).unwrap();   // .unwrap() PERMITTED — test code only
    bytemuck::cast_slice::<u8, F>(&bytes).to_vec()
}
```
NOTE: tests may launch the kernel directly (like reduce.rs) OR go through the new
`gpu_runtime::launch_der_*` host-readback wrapper — prefer the wrapper so the production seam is
exercised. Either way, `.unwrap()`/indexing is allowed here (`lib.rs:1` `#[cfg(test)]` allow).

**Divergence reporter** (scan.rs lines 88–102) — copy verbatim:
```rust
fn max_divergence(device: &[f64], baseline: &[f64]) -> (f64, f64) {
    let mut max_abs = 0.0_f64;
    let mut max_rel = 0.0_f64;
    for i in 0..baseline.len() {
        let abs = (device[i] - baseline[i]).abs();
        let rel = if baseline[i].abs() > 0.0 { abs / baseline[i].abs() } else { abs };
        max_abs = max_abs.max(abs);
        max_rel = max_rel.max(rel);
    }
    (max_abs, max_rel)
}
```

**Per-test oracle shape** (reduce.rs lines 128–148; scan.rs lines 104–119) — the REPORT-not-sign-off
contract:
```rust
#[test]
fn rmse_gradient_matches_cpu_baseline_f64_non_cube_multiple() {
    // n=37 exercises the bounds-guard idle path (Pitfall 4).
    let approx: Vec<f64> = (0..37).map(|k| f64::from(k) * 0.5).collect();
    let target: Vec<f64> = (0..37).map(|k| f64::from(k) * -0.25 + 1.0).collect();

    let device = run_der(&approx, &target);
    let baseline: Vec<f64> = approx.iter().zip(&target)
        .map(|(&a, &t)| cb_compute::rmse_der1(a, t)).collect();   // the D-7.2-06 baseline

    let (abs, rel) = max_divergence(&device, &baseline);
    println!("[der rmse f64 n=37] REPORTED max abs_div={abs:.3e} rel_div={rel:.3e}");
    assert!(rel <= 1e-9 || abs <= 1e-9, "rmse der diverged: abs={abs:.3e} rel={rel:.3e}");
    // GPU-06 epsilon is 7.6's job (D-7.2-06) — this bound only catches a wrong formula.
}
```

**Baseline fns to oracle against** (from `cb-compute`, re-exported at `crates/cb-compute/src/lib.rs:51–60`):
`rmse_der1`/`rmse_der2`, `logloss_der1`/`logloss_der2`, `cross_entropy_der1`/`der2`,
`quantile_der1`/`der2` (+ `QUANTILE_ALPHA`/`QUANTILE_DELTA`), `mae_der1`/`der2`,
`focal_der1`/`focal_der2`, `FOCAL_P_MIN`. Logloss and CrossEntropy share the der path — one kernel,
two enum arms (Pitfall 6, `compute_gradients_one_dim` line 473).

**Required fixture coverage** (D-7.2-06 + RESEARCH Wave-0 gaps): per loss (RMSE, Logloss/CrossEntropy,
Quantile/MAE der2≡0, Focal) × f32 + f64; edge cases empty, n=1, n=37 (non-cube-multiple, exercises the
`if ABSOLUTE_POS < approx.len()` idle guard — every kernel already has it, lines 37/55/83/113/141/181),
large N. Asserted tolerance: f32 ~1e-3 rel, f64 ~1e-9 rel (the reduce/scan bound). Use `cb_compute`
fns elementwise as the per-object baseline (loop, since they take scalars).

**Tolerance bounds reference** (reduce.rs lines 119–123 f32; lines 143–147 f64): generous run-stable
relative bounds that catch a wrong fold without pinning the GPU-06 epsilon.

## Shared Patterns

### Generic-runtime device launch (the 7.2 backbone)
**Source:** `crates/cb-backend/src/gpu_runtime.rs:98–123` (`launch_block_reduce_f64`)
**Apply to:** every der launch helper in `gpu_runtime.rs` + the `gradient_gpu.rs` harness.
```rust
let device = <SelectedRuntime as cubecl::Runtime>::Device::default();
let client = <SelectedRuntime as cubecl::Runtime>::client(&device);
// ... client.create / client.empty / kernel::launch::<F, SelectedRuntime>(...)
```
NEVER hard-code `CpuRuntime`/`CpuDevice` in production GPU helpers (Anti-Pattern, RESEARCH). The
CPU spike may name `CpuRuntime` because it is `feature = "cpu"`-gated; the new GPU path must not.

### Device-residency handle seam (no host fold — SC-3 / D-7.2-04)
**Source:** the `out_handle = client.empty(...)` / `out_handle.clone()` idiom (gpu_runtime.rs:103, 121)
**Apply to:** the histogram hand-off path — return `CbResult<Handle>`, do NOT `read_one`.
The trait `compute_gradients` keeps its `Vec<f64>` return for CPU consumers; the histogram seam is a
SEPARATE handle-returning internal fn. Read-back lives only in the self-oracle wrapper.

### WR-05 typed-error read-back (no silent zero buffer)
**Source:** `crates/cb-backend/src/gpu_runtime.rs:128–131` (also `cpu_runtime.rs:289–291`)
**Apply to:** every host-readback wrapper (the self-oracle path).
```rust
let bytes = client.read_one(out_handle)
    .map_err(|e| CbError::Degenerate(format!("CubeCL device read-back failed: {e:?}")))?;
```
A read-back failure must surface as `CbError::Degenerate`, never a zero `Vec`/handle masquerading as
a valid derivative.

### Length-1 `Array<F>` loss params (generics-float)
**Source:** `crates/cb-backend/src/cpu_runtime.rs:248–249, 268–269` (`launch_focal_f64`); kernel sigs
`kernels.rs:75–82` (quantile), `133–140` (focal)
**Apply to:** Quantile/MAE and Focal launches. `client.create(Bytes::from_elems(vec![param]))` +
`ArrayArg::from_raw_parts(handle, 1)`. Keeps kernels `<F: Float>` generic (AGENTS.md).

### Per-loss host-side `match` dispatch (mirrors upstream `switch(ELossFunction)` — D-01)
**Source:** `crates/cb-backend/src/cpu_runtime.rs:459–549` (`compute_gradients_one_dim`)
**Upstream parity target:** `pointwise_targets.cu:246–270` (`PointwiseTargetImpl` one-thread-per-object,
`i = blockIdx*blockDim + threadIdx`, bounds-guard, `der[i] = weight * target.Der(...)`,
`der2[i]` only when non-null) + `:450` `PointwiseTargetKernel` switch.
**Apply to:** the der dispatch entry point — extend the existing `match loss { Loss::Rmse => ..., }`,
do NOT build a comptime mega-kernel (Anti-Pattern). One `#[cube]` kernel per loss (they already exist
that way) selected by host `match`, mirroring upstream's per-functor template instantiation.

### Weight folding stays DOWNSTREAM (Pitfall 1 / Assumption A1 — CONFIRM IN PLAN)
**Source:** `crates/cb-compute/src/runtime.rs` (`compute_gradients` takes NO weights);
`cb-train/boosting.rs` applies `der1 * weight` downstream.
**Apply to:** the der kernels — produce **UNWEIGHTED** der1/der2 to match the CPU baseline the
self-oracle compares against. Upstream folds `weight` IN-KERNEL (`der[i] = weight * ...`), but the
Rust baseline does not, so an in-kernel weight would diverge the oracle by exactly the weight factor.
**Open question for the planner (RESEARCH Open Q1):** does the 7.3 histogram seam consume `der1` or
`der1 * weight`? Recommendation: hand 7.3 unweighted der1/der2 handles + let the existing
`histogram_scatter_kernel` do the weight multiply. Add a plan checkpoint to confirm this contract.
