**Focal der1 (`focal_gradient_kernel`) AND der2 (`focal_hessian_kernel`), both parametric on `(alpha, gamma)` as length-1 `Array<F>` device buffers, now run device-resident over `SelectedRuntime` as device HANDLES (no host fold), self-oracled vs `cb_compute::focal_der1`/`focal_der2` on rocm gfx1100 (reported max rel_div <= ~1e-8, most ~1e-14, finite on ±40 saturated logits); the phase closes with a full-family hand-off lock proving every in-scope der family (RMSE / Logloss-CrossEntropy / Quantile-MAE / Focal) hands 7.3 der1+der2 handles with no read_one on the seam, and the SC-4 structural assertion (cb-compute cubecl-free, cb-core/cb-model byte-unchanged).**

- Extended the Plan-02 parametric der seam additively with the Focal two-kernel family: `DerParamKernel::FocalGradient` (der1, reusing the authored `focal_gradient_kernel`) and `DerParamKernel::FocalHessian` (der2, reusing `focal_hessian_kernel`). ## Decisions Made
- **Focal `gamma` is param[1] (where Quantile uses `delta`):** the seam's `[p0, p1]` slice carries `[alpha, gamma]` for Focal, read without indexing via `param_pair()`.
- **Focal der2 is a real hessian kernel:** the only in-scope two-kernel family; `FocalHessian` launches `focal_hessian_kernel` through the parametric seam rather than `const_der_handle`.
- **der1/der2 UNWEIGHTED (A1):** matches the `cb_compute::loss` baseline; weight folded downstream by 7.3 — no `weights` arg.
- **SC-4 programmatic check is a doc-note:** the authoritative gate is `cargo tree -e features -p cb-compute | grep -ci cubecl` (run in verification, == 0); the unit test asserts cb_compute der baselines are pure-CPU finite math (what cubecl-free guarantees) rather than duplicating the dependency-graph walk.
