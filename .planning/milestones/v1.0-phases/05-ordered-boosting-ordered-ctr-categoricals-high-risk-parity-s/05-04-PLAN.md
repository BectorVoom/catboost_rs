<objective>
Plain CTR (ORD-03) — all six CTR types computed as a whole-set target statistic with no per-object prefix — plus the model-side `ctr_data` (de)serialize and inference apply. Per D-06 (the key isolation) Plain-mode CTR is locked BEFORE ordered-permutation CTR, so a later per-object divergence localizes to the ordering math, never the CTR math. This extends the Phase-4 `.cbm`/`model.json`/`apply.rs` substrate with the `ctr_data` section.

Purpose: Plain CTR is the simpler target statistic (no read-before-increment prefix). Locking the full six-type math surface + the model-side bake/apply here means Wave 5's ordered CTR is a focused delta on accumulation ORDER alone.
Output: `cb-train::ctr` module (online/calc_ctr/final_ctr), `cb-model::ctr_data` serde, extended `apply.rs`, and the plain_ctr + ctr_data_roundtrip oracles.
</objective>
