# Phase 6.2 Plan 03: Wave-1 MultiClass (softmax) + MultiClassOneVsAll (LOSS-02) Summary

**Implemented the two multiclass losses on the Wave-0 N-dim spine: `MultiClass` (softmax — the ONLY cross-dimension-coupled loss, with a packed symmetric Hessian gradient and a hand-rolled per-leaf symmetric positive-definite Newton solve transcribing `hessian.cpp:22-52` bit-faithfully) and `MultiClassOneVsAll` (separable per-dimension diagonal sigmoid), plus the multi-dim single-shared-accumulator split-score column, the class-label `[0,k)` remap, and the softmax/sigmoid prediction transforms — each passing its own per-stage oracle (Splits / LeafValues / staged N-dim approx / Predictions) `<= 1e-5` against catboost 1.2.10, with the D-04 scalar oracle suite still byte-stable at dim=1.**

## Checkpoint Decision (Open Q1 — softmax symmetric Hessian solver)

The opening `checkpoint:decision` (hand-roll vs a vetted linalg crate) was **PRE-RESOLVED to the RESEARCH-recommended default: hand-roll**. `cb_compute::solve_symmetric_newton` transcribes `TSymmetricHessian::SolveNewtonEquation` (`hessian.cpp:22-52`) verbatim — the trace-epsilon regularizer at **`f32::EPSILON`** precision (Pitfall 5), then negate → solve → negate — via a dense SPD Cholesky reconstructed from the packed Hessian. **No new crate was added**, so the supply-chain gate (T-6.2-SC) is a no-op. The transcription was validated bit-exact (`<= 1.5e-9` leaf values, `<= 5.5e-9` staged approx) against catboost 1.2.10 in Python before any Rust was written.

## Accomplishments

- **Task 1a (der/Hessian + solver + class remap):** `calc_softmax` (max-subtraction before `exp`, T-6.2-02 NaN guard); `softmax_ders` (coupled `der1[d] = δ(d==t) - p[d]` + PACKED lower-triangular symmetric Hessian, `error_functions.h:687-728`); `multiclass_onevsall_ders` (diagonal sigmoid); `solve_symmetric_newton` + `cholesky_solve`. `Loss::MultiClass`/`Loss::MultiClassOneVsAll` variants + `validate` arms. `cb-backend::compute_gradients` dispatches the coupled softmax path (per-object packed Hessian `der2`, length `n·k(k+1)/2`) and the separable OneVsAll diagonal path BEFORE the per-dimension scalar loop. `cb-train` derives `approx_dimension = max(distinct_class, 2)`, builds the `LabelToClass` remap to `[0,k)` (typed `CbError` on an unmapped label, T-6.2-01), branches the leaf-delta solve (softmax → `compute_softmax_leaf_deltas`; everything else → the per-dimension scalar loop, with OneVsAll's diagonal Newton == the scalar Logloss Newton per dim), and gates multiclass to Newton. **tree.rs untouched in this task** (split-score isolated to Task 1b).
- **Task 1b (multi-dim split score):** `multi_dim_split_score` — a SINGLE shared `num`/`den` accumulator fed every dimension's per-leaf bucket stats, the transform applied ONCE after the dimension loop (Cosine couples num/den inside the sqrt; L2 linear), transcribed from `scoring.cpp:751-766`/`:1033-1049` + `score_calcers.h:47-97` + `short_vector_ops.h:61-81`. `tree.rs::score_candidate` routes through `multi_dim_candidate_score`, which infers `approx_dimension` from `der1.len()/n_objects` so at dim=1 the score is **BIT-identical** to the prior scalar `split_score` (a `to_bits()` unit test proves it) and the full Wave-0 scalar oracle suite stays green.
- **Task 3 (predict + metadata + fixtures + oracle):** `apply_multiclass_prediction` (softmax/sigmoid `Probability`, argmax `Class` mapped through `ClassToLabel`, object-major `(n,dim)` output per A4) + `MultiClassKind`. `cb_model::Model.class_to_label` threaded through `from_trained`. `gen_multiclass` (+ `--multiclass-only`) generates the two frozen fixtures offline (catboost 1.2.10, Cosine, Newton, `leaf_estimation_iterations:1`, bootstrap No, random_strength 0, thread_count 1). `multiclass_oracle_test.rs` gates Splits (also the Task-1b split-score isolation check), LeafValues (dim-major→leaf-major transpose), StagedApprox (dim-major→object-major transpose), and Predictions — all `<= 1e-5` for BOTH losses.
