## Shared / Meta Patterns

### (P-NDIM) Scalar → dimension-major N-dim widening — the Wave-0 spine

**Applies to:** `loss.rs`, `runtime.rs`, `cpu_runtime.rs`, `boosting.rs`, `bootstrap.rs`,
`tree.rs`, `compute_leaf_deltas`, `cbm.rs`.

**Layout (D-6.2-01):** one contiguous `Vec<f64>` indexed `approx[d * n + i]` (dimension-major,
outer = dim). At `dim == 1` this is byte-identical to today's `Vec<f64>`.

**The non-negotiable invariant (RESEARCH Pitfall 1):** keep the per-dimension reduction as an
**outer loop** so the dim=1 slice and `cb_core::sum_f64` order are exactly today's:
```rust
for d in 0..approx_dimension {
    let slice = &approx[d * n .. d * n + n];   // at dim==1 → approx[0..n], unchanged
    // …existing scalar reduction on `slice`, via cb_core::sum_f64…
}
```
Never fuse into a single `0..dim*n` loop (perturbs dim=1 summation order → D-04 gate fails on a
nonzero-but-≤1e-5 diff). Never sum der **across** dimensions in the leaf reduction (only the
split-score and MVS-norm do cross-dim reductions).

**Gate (D-04):** all ~38 existing fixtures under `crates/cb-oracle/fixtures/` re-run green at
dim=1 **before** any multiclass math. No fixture regeneration in Wave 0.

### (P-ADDLOSS) Add a new loss = variant + dispatch arm + validate + fixture + oracle test

**Applies to:** every Wave 1-3 loss. The 6.1 losses (LogCosh, Quantile, Poisson) are the
precedent — each touched the SAME five sites. The new losses follow it identically:

1. **`runtime.rs`** — add a `Loss::Variant` (params via the `Loss::Variant { params }` pattern;
   MultiQuantile carries `alpha: Vec<f64>`). Add a `Loss::validate()` arm (V5 input validation:
   `alpha ∈ [0,1]`, `delta ≥ 0`, class index `[0,k)`).
2. **`loss.rs`** — add the `*_der1`/`*_der2` helper(s) with the upstream `error_functions.{h,cpp}`
   file:line in the doc comment (the separable losses **reuse** existing scalar helpers per
   dimension; only softmax adds new coupled math).
3. **`cpu_runtime.rs`** — add a `match loss { … }` arm.
4. **`boosting.rs::validate_leaf_method`** — gate any unsupported `(loss, leaf_method)` combo.
5. **`gen_fixtures.py`** + **fixtures/** + **`tests/…_oracle_test.rs`** — generate the frozen
   fixture offline (catboost 1.2.10, `thread_count=1`), commit it, clone `wave3_quantile_…` test.

## Pattern Assignments

### `crates/cb-compute/src/runtime.rs` (enum + compute-trait seam)

**Analog:** itself — `Loss` enum (`runtime.rs:47`), `Derivatives` (`runtime.rs:271`),
`compute_gradients` (`runtime.rs:296`), `Loss::validate` (`runtime.rs:184`).

**Current `compute_gradients` signature (`runtime.rs:296`) — the central seam to widen:**
```rust
fn compute_gradients(&self, loss: Loss, approx: &[f64], target: &[f64])
    -> CbResult<Derivatives>;
```
**N-dim change (D-6.2-03):** gains `approx_dimension: usize`; `approx` becomes the dimension-major
`&[f64]` of length `dim*n`; `target` stays per-object (or `dim*n` for multilabel). `Derivatives`
(`der1: Vec<f64>`, `der2: Vec<f64>`) widen to `dim*n` (diagonal losses) — softmax needs a packed
symmetric Hessian, so plan whether `der2` grows or a new field carries the lower-triangular pack
(RESEARCH Code Examples: `der2[idx++] = derY*(derY-1)` diag, `derY*derRef[dimX]` off-diag).

**`Loss::Variant { params }` precedent to copy (`runtime.rs:85` Quantile, `:125` Expectile):**
```rust
Quantile {
    alpha: f64,
    delta: f64,
},
```
→ MultiQuantile mirrors this with `alpha: Vec<f64>` (NOTE: `Vec<f64>` is not `Copy`, so the
`#[derive(… Copy)]` on `Loss` at `runtime.rs:46` must be reconsidered — Quantile already forced
dropping `Eq`; MultiQuantile forces dropping `Copy`. Flag as a cross-crate ripple: every `loss:
Loss` by-value call site, e.g. `boosting.rs:1597`, `cpu_runtime.rs:451`, must tolerate a non-`Copy`
`Loss` — pass by reference or clone).

**`Loss::validate` arm precedent (`runtime.rs:223` Quantile):**
```rust
Self::Quantile { alpha, delta } => {
    if !alpha.is_finite() || !(0.0..=1.0).contains(&alpha) {
        return Err(cb_core::CbError::OutOfRange(format!(
            "Quantile alpha must be finite and in [0, 1], got {alpha}"
        )));
    }
    if !delta.is_finite() || delta < 0.0 { /* … */ }
}
```
→ MultiQuantile validates each `alpha[k] ∈ [0,1]`; multiclass validates class-index `[0,k)`.

### `crates/cb-compute/src/loss.rs` (per-loss der math)

**Analog:** the scalar der helpers — `quantile_der1` (`loss.rs:166`), `poisson_der1`
(`loss.rs:339`), `sigmoid` (`loss.rs:54`).

**Separable-reuse precedent — the MultiQuantile der is literally `quantile_der1` per dimension:**
```rust
// loss.rs:166 — REUSED verbatim per dimension for MultiQuantile (D-6.2-05)
pub fn quantile_der1(approx: f64, target: f64, alpha: f64, delta: f64) -> f64 {
    let val = target - approx;
    if val.abs() < delta { 0.0 }
    else if val > 0.0 { alpha } else { -(1.0 - alpha) }
}
```
MultiQuantile loops `for k in 0..alpha.len()` calling `quantile_der1(approx[k*n+i], target_i,
alpha[k], delta)`, `der2 = 0` (RESEARCH Pattern 2). MultiClassOneVsAll / MultiLogloss /
MultiCrossEntropy similarly reuse a per-dim `sigmoid` (`loss.rs:54`).

**New coupled math (the ONLY genuinely new helper) — softmax der + symmetric Hessian.** Follow
the `focal_der1`/`focal_der2` doc-comment discipline (`loss.rs:105-146`: verbatim transcription
with `error_functions.h:1684-1709` file:line). Source for softmax: `error_functions.h:687-728`
(RESEARCH Code Examples) — `CalcSoftmax` with max-subtraction (`eval_processing.h:18`, V5 NaN
guard), `der[d] = δ(d=target) - p[d]`, packed lower-triangular Hessian.

**`sigmoid` upstream-form precedent (`loss.rs:54`) — copy this max-subtraction discipline for softmax:**
```rust
pub fn sigmoid(approx: f64) -> f64 {
    let e = approx.exp();           // f64::exp — oracle absorbs the FastExp gap (A2)
    1.0 - 1.0 / (1.0 + e)
}
```

### `crates/cb-backend/src/cpu_runtime.rs` (backend dispatch)

**Analog:** its own `match loss` (`cpu_runtime.rs:469`), one arm per loss.

**Add-an-arm precedent (`cpu_runtime.rs:504` Quantile — the MultiQuantile pattern):**
```rust
Loss::Quantile { alpha, delta } => {
    let der1 = launch_quantile_f64(approx, target, alpha, delta)?;
    let der2 = vec![0.0_f64; approx.len()];     // QUANTILE_DER2 = 0 → Exact leaf
    Ok(Derivatives { der1, der2 })
}
```
**N-dim change:** add the `approx_dimension` arg; separable arms loop `for d in 0..dim` over the
dim-major slices reusing the existing kernel launchers; the empty-input guard (`cpu_runtime.rs:462`)
and `LengthMismatch` guard (`:455`) widen to `dim*n`. Softmax (Wave 1) is a host reduction, NOT a
kernel (RESEARCH Architecture map) — handle it outside the kernel-launch arms.

### `crates/cb-train/src/boosting.rs` (train loop + leaf deltas + leaf-method gate)

**Analog:** itself. The three scalar seams to widen, with current line refs:

**Approx init (`boosting.rs:1228`):**
```rust
let bias = starting_approx(params, target);
let mut approx = vec![bias; n];          // → vec![0.0; dim*n], per-dim bias[d] (CalcOptimumConstApprox)
```

**Gradient call (`boosting.rs:1597`):**
```rust
let ders = runtime.compute_gradients(params.loss, &approx, target)?;  // + approx_dimension arg
```

**Approx update (`boosting.rs:1800`) — widen to per-dimension:**
```rust
for (i, &leaf) in leaf_value_leaf_of.iter().enumerate() {
    if let (Some(a), Some(&lv)) = (approx.get_mut(i), leaf_values.get(leaf)) {
        *a += lv;        // → approx[d*n+i] += learning_rate * leaf_delta[d][leaf]
    }
}
```
**Staged record (`boosting.rs:1808`):** `out.extend_from_slice(&approx)` already records the whole
buffer — at N-dim it records `dim*n` per iteration; the generator's StagedApprox flatten order
(object-major `(n,dim)` from `staged_predict`) is a defined transpose vs this dim-major buffer
(RESEARCH A4).

**`compute_leaf_deltas` (`boosting.rs:730`) — per-dim diagonal + softmax symmetric arm:**
The existing `match method { Gradient | Simple | Newton | Exact }` (`boosting.rs:742-795`) is the
per-dimension solver for diagonal losses — call it `for d in 0..dim` over `approx[d*n..]`. The
Exact arm (`boosting.rs:773`) already threads `(alpha, delta)` via:
```rust
let (quantile_alpha, quantile_delta) = match loss {
    Loss::Quantile { alpha, delta } => (alpha, delta),
    _ => (QUANTILE_ALPHA, QUANTILE_DELTA),
};
```
→ MultiQuantile passes `alpha[d]` per dimension into the SAME `exact_leaf_delta`. Softmax (Wave 1)
adds a NEW arm: per-leaf symmetric Hessian → `SolveNewtonEquation` (`hessian.cpp:22-52`,
trace-epsilon at `f32::EPSILON` — Pitfall 5; Open Q1 = pick/transcribe a small symmetric solver).

**`validate_leaf_method` (`boosting.rs:692`) — extend the combo gate:**
```rust
if matches!(method, LeafMethod::Exact)
    && !matches!(loss, Loss::LogCosh | Loss::Mae | Loss::Quantile { .. }) {
    return Err(CbError::OutOfRange(/* … */));
}
```
→ add `Loss::MultiQuantile { .. }` to the Exact-eligible set; gate MultiClass/OneVsAll to Newton
(1 iter — Pitfall 2), MultiLogloss/MultiCrossEntropy to Newton (pin iters:1 — Pitfall 2).
