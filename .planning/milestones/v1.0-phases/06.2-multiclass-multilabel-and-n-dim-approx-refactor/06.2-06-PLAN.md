<objective>
Close GAP-1 / CR-01 (Critical, 06.2-REVIEW.md): the public multi-dimensional apply
path. Today `predict_raw_one` (apply.rs:210-229) reads `tree.leaf_values.get(leaf)`
with `leaf ∈ [0, n_leaves)`, so for a DIMENSION-MAJOR multi-output tree
(`leaf_values` length `approx_dimension * n_leaves`, stored LEAF-MAJOR on the wire)
it returns ONLY dimension 0. There is no `predict_raw_multi`, so
`apply_multiclass_prediction` (predict.rs:167) has NO public producer — every
`.cbm`/json-loaded MultiClass / MultiClassOneVsAll / MultiLogloss / MultiCrossEntropy
/ MultiQuantile model predicts silently-wrong single-dimension scores on the
published surface. Additionally, 06.2-03 left the `.cbm` and json deserialize paths
with `class_to_label: Vec::new()` (an empty stub — cbm.rs:422, json.rs:288), so a
loaded multiclass model cannot recover its original labels.

Purpose: make the PUBLIC load-model→predict surface correct for multi-output models
(LOSS-02, LOSS-03 MultiQuantile), closing the only Critical defect blocking 6.2 that
lives in cb-model.

Output: `predict_raw_multi` (dimension-aware apply emitting the dim-major raw approx),
a `Model::predict`-level multiclass wiring through `apply_multiclass_prediction`, the
`class_params`/`multiclass_params` deserialize round-trip on BOTH wire formats, and a
public load-model→predict oracle test (≤1e-5 vs upstream `predictions.npy`) for all
five multi-output losses.
</objective>
