<objective>
Extend the single-tree device loop (Plan C) into a **full multi-tree Plain-boosting pass** that runs device-resident across iterations: each tree's leaf updates feed the next tree's derivatives (recomputed device-side via the 7.2 der seam), with persistent device buffers reused across the boosting run and ONE client threaded throughout.

The cross-oracle (`grow_loop::multi_tree`) asserts each tree's STRUCTURE matches the CPU `cb-train` multi-tree reference EXACTLY and REPORTS leaf-value divergence. Critically, this plan exercises the D-7.5-06 non-determinism budget: because D-03 in-kernel atomics make summation order non-deterministic, float noise compounds tree-over-tree; if it ever flips a near-tie split argmax (changing structure), the harness REPORTS that argmax-flip boundary rather than masking it — the epsilon is signed off only in 7.6.

Purpose: prove the device loop holds parity across a realistic multi-tree run (not just one tree) and characterize the atomic-jitter structure-stability boundary.
Output: `grow_boosting_pass`, the `grow_loop::multi_tree` cross-oracle, and a REPORTED argmax-flip characterization.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host driver → device launches (per iteration) | fixture handles + iterations/learning_rate/depth cross into the per-tree launch geometry; no external/untrusted input |
| device → host (O(1) per level, per tree) | only the BestSplit descriptor + per-tree 2^depth part-stats cross; running approx/der stay device-resident |

Pure in-process GPU compute; surface is buffer-index correctness across a multi-iteration persistent loop + the compounding float-noise (numeric, not security).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07.5-04-01 | Tampering | persistent-buffer reuse stale-state across iterations (a tree reading the prior tree's scratch) | mitigate | explicit per-iteration reset/overwrite of partition/part-stats scratch; the cross-oracle would catch a stale-state structure divergence |
| T-07.5-04-02 | Tampering | integer overflow in iterations * 2^depth leaf bookkeeping | mitigate | `checked_mul` on iteration/leaf index math → `CbError::Degenerate` |
| T-07.5-04-03 | Tampering/DoS | reading der/approx/part-stats through the wrong client or a 0-len handle | mitigate | ONE `&client` threaded across all iterations; never read a 0-len handle (Pitfall 3) |
| T-07.5-04-04 | Spoofing | silent zero buffer on a per-iteration read-back failure → a fabricated tree | mitigate | every `read_one` → `CbError::Degenerate` on failure; the per-tree cross-oracle also catches fabricated structure |
| T-07.5-04-05 | Tampering | NaN/Inf approx/der propagating across iterations from an empty leaf | mitigate | `calc_average` `count>0` guard + the 7.2 der guards transcribed; degenerate-approx → typed error |
| T-07.5-04-SC | Tampering | npm/pip/cargo installs | accept | no package installs (no new deps); RESEARCH Package Legitimacy Audit: N/A |
