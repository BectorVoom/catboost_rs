## Pattern Assignments

### `crates/cb-backend/src/kernels.rs` — `find_optimal_split_kernel<ScoreCalcer comptime>` (kernel, transform→argmin)

**Analog (mechanism):** `block_reduce_atomic_kernel` (1520-1580) for the wave-agnostic intra-cube fold + atomic finalize; `block_scan_kernel` (1611-1700) for the `#[comptime]` flag + plane/shared-mem structure. **Formula source (FROZEN, transcribe verbatim):** `cb-compute/src/score.rs`.

**Comptime score-calcer selection** (RESEARCH Pattern 1) — the same `#[comptime]` mechanism as `block_reduce_atomic_kernel`'s `use_plane` (line 1524) and `block_scan_kernel`'s `inclusive` (1615). Carry a `#[comptime] score_fn: <enum/u32>` (L2/Cosine/SolarL2/LOOL2/SatL2) so the inner `AddLeaf` arithmetic is JIT-monomorphized — zero device branch in the per-leaf loop.

**The per-leaf AddLeaf arithmetic to transcribe** — from `cb-compute/src/score.rs`, folded in **f64** (D-03 f64-finalize; the upstream `float` Score for L2/Solar/LOO/Sat is a 7.6 epsilon concern, NOT a structure concern):
```rust
// Source: crates/cb-compute/src/score.rs (FROZEN oracle).
// add_leaf_plain (score.rs:38-42):  avg = calc_average(swd, sw, scaled_l2); term = avg * swd
// L2/NewtonL2     (score.rs:39-55, :136):   score = Σ_leaf avg*swd
// Cosine/NewtonCosine (score.rs:73-85, :139): num = Σ avg*swd; den = 1e-100 + Σ avg²*sw; score = num/sqrt(den)
// SolarL2 (score.rs:182-197): weight>1e-20 ? (-sum*sum)*(1+2*ln(weight+1.0))/weight : 0.0   (NO scaled_l2 — IN-04)
// LOOL2   (score.rs:202-223): adj = weight>1 ? weight/(weight-1) : 0; adj*=adj; weight>0 ? adj*(-sum*sum)/weight : 0
// SatL2   (score.rs:228-247): adj = weight>2 ? weight*(weight-2)/(weight*weight-3*weight+1) : 0; weight>0 ? adj*(-sum*sum)/weight : 0
// calc_average (leaf.rs:83-89): count>0 ? sum_delta/(count+scaled_l2) : 0.0   (empty-leaf guard — transcribe the guard)
```
The Cosine `1e-100` seed (score.rs:78) is the FIRST summand — preserve it so the dim=1 accumulation order matches the CPU. Every `weight>1e-20`/`count>0`/`weight>0` guard transcribes into the device kernel (Security V5: no NaN/Inf from a degenerate empty leaf).

**Wave-agnostic intra-cube fold + atomic finalize** — clone the `block_reduce_atomic_kernel` skeleton (1526-1579): `use_plane` plane_sum path (1538-1554) OR `CUBE_DIM_X`-strided shared-mem tree-reduce (1556-1571); `if tid == 0u32 { acc[0].fetch_add(cube_partial); }` (1577-1578). NO literal 32/64 in any stride (D-09). Score fold in **f64** (Pitfall 4 — f32 widens jitter enough to flip a near-tie argmax → STRUCTURE break, D-7.5-06).

**The argmin block-reduce + lowest-index tie-break is GENUINELY-NEW logic** (see "No Analog Found"): the reduce carries a `(gain, feature_id, bin_id)` tuple and on `gain[a] == gain[b]` keeps the LOWER `(feature,bin)` index (upstream `pointwise_scores.cu:140-141`), which equals CPU `select_best_candidate` strict-first-wins over ascending (feature,bin) order (`tree.rs:291-302`).

### `crates/cb-backend/src/kernels.rs` — `scan_update_pointwise_kernel` (kernel, transform/prefix-sum)

**Analog:** `block_scan_kernel` (1611-1700) — reuse VERBATIM for the per-feature bin-axis prefix-sum (D-7.5-03 deferred-from-7.3 transform). The scan turns per-bin (Σder, Σweight) into cumulative "left of border b" stats so a candidate at border b reads `left = scan[b]`, `right = total - scan[b]` (mirrors CPU `FindOptimalSplitSingleFoldImpl`: `weightRight = part.Weight - weightLeft`).

**SCOPE caveat (RESEARCH A1 / Open Q1):** `block_scan_kernel` is single-cube only (`n ≤ CUBE_DIM=32`, documented at kernels.rs:1598-1604 and `launch_block_scan_f64` at gpu_runtime.rs:223-231). 8-bit features have 256 bins. The first scan-update plan must scope to ≤CUBE_DIM bins (low-bit-width fixtures) and add the cross-cube carry as an EXPLICIT tracked sub-task before the 8-bit path — NOT a silent cut. This is the documented 7.2/7.3 forward dependency.

**Pairwise variant** `scan_update_pairwise_kernel` (D-7.5-03 deferred-from-7.4): `ScanPairwiseHistograms`/`BuildBinaryFeatureHistograms`/`UpdatePairwiseHistograms` over the FROZEN 4-channel handle. Same `block_scan_kernel` structure over the 4-channel `(feature * n_bins + bin) * 4 + histId` layout (gpu_runtime.rs:1378). No host round-trip at the seam.

### `crates/cb-backend/src/gpu_runtime.rs` — `grow_oblivious_tree` host-light loop driver (GENUINELY-NEW; loop driver)

**Analog:** the STRUCTURE comes from the CPU oracle `cb-train::greedy_tensor_search_oblivious_perturbed` (`tree.rs:544-580`) — the per-level `for _level in 0..depth { best = select_level_*(...)?; chosen.push(best); }` skeleton, then `assign_leaves` + leaf-value estimation. The DEVICE wiring (persistent handles + O(1) read-back) has **no in-repo analog** — it is the RESEARCH research-flag item.

**The per-depth sequence to express** (RESEARCH Pattern 2, mirroring upstream `oblivious_tree_doc_parallel_structure_searcher.cpp:63-158`):
```text
upload ONCE: cindex, target/weight, indices, der1/der2 (7.2 handles), partition[], partStats[]
for depth d = 0..MaxDepth:
    launch_pointwise_hist2_handle (7.3)  [or launch_pairwise_hist*_handle (7.4)]   → binSums handle (device)
    launch_scan_update_pointwise         → cumulative leaf stats (device, in place over FROZEN handle)
    launch_find_optimal_split            → per-block BestSplit[] (device argmin, score calcer comptime)
    read_one(best_handle)                → O(1) ~16-byte BestSplit read-back  ← the ONLY per-level crossing
    host: integer split decision + strict-first-wins / lowest-index tie-break (match select_best_candidate)
    launch_partition_split               → reorder doc routing IN-PLACE (device, forward-bit leaf convention)
    launch_partition_update              → recompute partStats per new leaf (device)
at leaves: read_one(partStats)           → one read of 2^depth part-stats (RESEARCH A2: ≤65k floats, once/tree)
host: leaf value mu = calc_average(Sum, Weight, scaled_l2)   (leaf.rs:83-89)
```

**The BestSplit POD read-back** (the O(1) crossing) — clone the read-back idiom from the harness (`pointwise_hist.rs::read_handle_f64`, 180-) + `client.read_one(...).map_err(CbError::Degenerate)`:
```rust
#[repr(C)]
#[derive(Clone, Copy, bytemuck::Pod, bytemuck::Zeroable)]
struct BestSplit { feature_id: u32, bin_id: u32, score: f32, gain: f32 } // 16 bytes
let bytes = client.read_one(best_handle)
    .map_err(|e| CbError::Degenerate(format!("best-split read-back failed: {e:?}")))?;
let cands: &[BestSplit] = bytemuck::cast_slice(&bytes);  // resultSize = O(blocks), NOT O(features*bins)
```
RESEARCH Open Q2: the across-block argmin MAY finish host-side over the small per-block `BestSplit[]` (O(blocks) read-back is within "minimal round-trips") OR fully on-device — discretionary. Per-leaf partition-split MUST stay on device.

**Single-client rule** (Pitfall 3): thread ONE `ComputeClient` through the whole tree's loop; allocate/launch/read every handle through it; never re-resolve a fresh client mid-loop; never read a 0-len handle.

**Anti-pattern (forbidden):** reading the full score/histogram/partition buffer to host each level to argmin/partition host-side. Only the O(1) BestSplit descriptor (and the final 2^depth part-stats) may cross (D-05/SC-2).

## Shared Patterns

### Score / leaf / pairwise formulas = transcribe the FROZEN CPU oracle (SC-3)
**Source:** `crates/cb-compute/src/score.rs` (l2_split_score 49, cosine_split_score 73, multi_dim_split_score 110, solar/loo/sat terms 182/202/228, calc_average via leaf.rs), `crates/cb-compute/src/leaf.rs` (calc_average 83, cholesky_solve 282), `crates/cb-compute/src/pairwise_scoring.rs` (calculate_pairwise_score 471, calculate_score 367, calculate_pairwise_leaf_values 281).
**Apply to:** `find_optimal_split_kernel` (per-calcer AddLeaf), `pairwise_make_derivatives_kernel`/`select_best_split_kernel`, the leaf-value host step, and ALL host-reference baselines. Transcribe the guards (`count>0`, `weight>1e-20`, `1e-100` seed) verbatim — they are the Security-V5 NaN/Inf mitigation AND the parity contract. Fold in f64.

### Comptime specialization, no warp literals (D-09)
**Source:** `kernels.rs` `#[comptime] use_plane` (block_reduce_atomic_kernel:1524, block_reduce_kernel:1427), `#[comptime] inclusive` (block_scan_kernel:1615), `#[comptime] bits` (pointwise/pairwise hist), `BLOCK_REDUCE_SHMEM` const.
**Apply to:** the `#[comptime] score_fn` selector on `find_optimal_split_kernel`; the `inclusive` flag on `scan_update_*`; any shmem size a comptime `usize` const, never a topology/warp literal. Strides derive from `CUBE_DIM_X`/`PLANE_DIM`.

### In-kernel atomic merge + f64-finalize (D-03)
