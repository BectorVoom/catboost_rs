## Phase Goal (MVP — user story)

**As a** developer training CatBoost-rs on a GPU with a non-default score function, **I want to** have Cosine, SolarL2, LOOL2, and SatL2 split scores computed device-resident on rocm, **so that** GPU training matches the CPU path for every score function CatBoost-rs supports, not just L2.

After this plan: each score variant runs on rocm AND cross-oracles against its `cb-compute` reference, and a Cosine-scored device tree matches the CPU Cosine structure — a thin slice per variant, all riding the existing kernel.
