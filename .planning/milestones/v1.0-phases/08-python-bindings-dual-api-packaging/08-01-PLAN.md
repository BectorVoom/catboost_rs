Stand up the PyO3 binding crate and deliver the thinnest possible end-to-end vertical slice: a Python user runs `import catboost_rs`, constructs `CatBoostRegressor()`, calls `.fit(X, y)` on a float32 contiguous NumPy array, and gets predictions back from `.predict(X)` — all wired through the real `catboost-rs` facade (`CatBoostBuilder` → `Model::predict`), not a stub.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python caller → binding (`fit`/`predict`) | Untrusted NumPy buffer (arbitrary dtype/shape/strides) crosses into Rust |
| Binding → facade (`CatBoostBuilder`/`Model`) | Owned, validated data only crosses here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | `ingest_py::numpy_to_owned` borrowed buffer | mitigate | Copy float32 into owned `Vec<Vec<f64>>` BEFORE `py.detach()` (D-11); never pass a `PyReadonlyArray` borrow into the compute closure |
| T-08-02 | Tampering | non-contiguous/float64 input silently coerced | mitigate | Smoke slice rejects non-contiguous/non-f32 with a `PyValueError` (full typed `CatBoostValueError` in 08-02/08-03); no silent precision coercion (D-12) |
| T-08-03 | DoS | Rust panic crossing the FFI boundary aborts CPython | mitigate | `[lints] workspace=true` denies `unwrap`/`expect`/`panic`/`indexing_slicing` on PyO3 glue; every fallible path returns `PyResult` |
| T-08-04 | Tampering | feature-unification leaks cpu onto a rocm build | mitigate | `cpu`/`rocm` features forward to the facade passthrough; never an unconditional cpu dep — verified by the `cargo tree --features rocm` grep gate |
| T-08-SC | Tampering | cargo installs (pyo3/numpy/pyo3-arrow/maturin) | mitigate | RESEARCH Package Legitimacy Audit (lines 151-166): all five are PyO3-org/established Arrow authors, verdict OK; pyo3-arrow 0.19 confirmed via the Task-1 decision checkpoint |
