## Decisions Made

- **`.npy` per-stage freeze, not `model.json`:** text/embedding models cannot be exported to JSON (upstream `model_exporter.cpp:152` — "Can save model with text features only in cbm format"). The per-stage borders/leaf/staged/prediction arrays are frozen directly from the live single-thread trainer via the Python API (`get_leaf_values`, `_get_tree_splits` border parse, `staged_predict`, `predict`), plus the binary `model.cbm`. No fabricated values.
- **OLB pinned to 1:** the 16-row corpus is `< 1000`, so the data-dependent `OccurrenceLowerBound` resolves to 1 (A4). Recorded in `meta.json`/`summary.json` and asserted at generation time.
- **Two-fit corpus capture:** BoW is target-independent (no online estimation), so it does not exercise `online_order`/`calcer_encoding`; NaiveBayes (an online estimator) does. The capture runs both and merges, guaranteeing all 7 categories.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] token_stream hook inserted at file scope + mangled JSON quotes**
- **Found during:** Task 1 (first build attempt)
- **Issue:** The `perl -0777` insertion for the `token_stream` hook (i) placed the hook body AFTER the `Tokenize` function's closing brace (file scope → "expected unqualified-id") and (ii) mangled the `\"`-escaped JSON quote into a bare `"""`.
- **Fix:** Re-inserted the hook inside the function before the closing brace; replaced `\"` quote escapes with raw-string-literal quotes `R"J(")J"`. Corrected the driver's perl pattern to a robust single-line anchor.
- **Files modified:** `build_instrumented_trainer.sh`, vendored `tokenizer.cpp` (uncommitted)
- **Verification:** token_stream fires 96 events in the smoke dump; no triple-quote in any patched source.
- **Committed in:** `f411da4` (driver fix; vendored patch uncommitted)

**2. [Rule 1 - Bug] Header sink redefinition collision**
- **Found during:** Task 1 (second build attempt)
- **Issue:** `base_text_feature_estimator.h` transitively includes `text.h`, so both header sinks (`CbInstr065Log`/`CbInstr065Fmt17`) landed in the same TU → "redefinition of CbInstr065Log".
- **Fix:** Wrapped the header sink in `#ifndef CB_INSTR065_SINK_DEFINED` so exactly one definition per TU. Applied to both headers and to the driver's `ensure_header_sink`.
- **Files modified:** `build_instrumented_trainer.sh`, vendored `text.h` + `base_text_feature_estimator.h` (uncommitted)
- **Verification:** third build RC=0; all 7 hook strings present in the compiled `.so`.
- **Committed in:** `f411da4` (driver fix; vendored patches uncommitted)

**3. [Rule 3 - Blocking] online_order / calcer_encoding perl patterns did not match**
- **Found during:** Task 1 (first build attempt)
- **Issue:** The multi-line `perl -0777` slurp patterns for `online_order` and `calcer_encoding` were whitespace-fragile and silently produced no substitution (the `grep -q` guard masked the no-op).
- **Fix:** Applied the two header hooks directly via Edit, then rewrote the driver to use deterministic single-line-anchor inserts (with an `awk` fallback for `calcer_encoding`) plus a post-patch `grep` assertion.
- **Files modified:** `build_instrumented_trainer.sh`, vendored `base_text_feature_estimator.h` (uncommitted)
- **Verification:** online_order (2) + calcer_encoding (32) fire in the smoke dump.
- **Committed in:** `f411da4` (driver fix; vendored patch uncommitted)
