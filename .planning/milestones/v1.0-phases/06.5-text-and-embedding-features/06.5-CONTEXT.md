## Phase Boundary

Deliver the **text & embedding feature-encoding layer** so all six calcers produce upstream-matching numeric encodings ≤1e-5 vs **catboost 1.2.10** (`thread_count=1`), per-stage (borders / splits / leaf values / staged approx + final prediction), with **tokenizer parity nailed first** as the load-bearing risk (D-11). **In scope:** tokenizer + dictionary/digitizer, the 6 calcers (3 text, 2 embedding) with their online/ordered estimation, and the wiring that feeds calcer outputs into the existing quantization→tree path.

<decisions>
## Implementation Decisions

### Tokenizer & Dictionary (load-bearing — D-11 first risk)
- **D-01 (sequencing):** The bit-identical tokenizer + digitizer stream is the **first gate** — its own plan/wave. Every calcer plan is **blocked** on it (SC-1). No calcer oracle is attempted until the token stream matches. Transcribe verbatim from `tokenizer.cpp` and `text_digitizers.h`/`dictionary.h`.
- **D-02 (option coverage):** Implement the tokenizer + dictionary option surface needed to (a) reproduce upstream **defaults** (`TTokenizerOptions` / `TTextProcessingOptions`) and (b) cover the options the **frozen FEAT-01 fixtures** exercise (ByDelimiter delimiter split, lowercasing, frequency-based dictionary build with default `occurrence_lower_bound`/`max_dictionary_size`). Pin defaults to upstream. Exotic tokenizers (BPE/subword, custom regex separators) not reachable by the fixtures are **deferred**, not implemented speculatively.

### Calcer correctness & online/ordered estimation
- **D-03 (online estimation is mandatory):** NaiveBayes & BM25 (text) and LDA & KNN (embedding) are `IOnlineFeatureEstimator`s — **target-aware** (class priors / classification target) and computed over the **learn permutation** for leakage control, exactly like CTRs. They **MUST reuse the Phase-5 ordered/fold permutation infrastructure** (`cb-train::fold.rs`, ordered boosting). BoW is target-independent (simple estimation). Simplifying the online path breaks the per-stage oracle — no shortcut. (Verified: `base_text_feature_estimator.h:42 ComputeOnlineFeatures(learnPermutation,…)`; `embedding_feature_estimators.h` takes `target` + `classificationTarget`.)
- **D-04 (formula fidelity):** Calcer math is transcribed verbatim from `bow.cpp`, `naive_bayesian.cpp`, `bm25.cpp`, `embedding_calcers.cpp` (LDA/KNN). All summations/reductions route through `cb_core::sum_f64` (D-08 summation discipline) so ≤1e-5 parity holds; only documented upstream-order scatter cells use raw `+=`.

### Layering / placement
- **D-05 (no new crate):** Mirror the 06.3-15 layering discipline — **no new crate**. Tokenizer + dictionary + calcer primitive math land as new modules co-located with the feature-ingestion path (default: `cb-data`, which already owns Pool's raw columns + the `borders`/`quantize` path; purely-numeric primitives may live in `cb-compute`). The **per-fold online wiring** lands in `cb-train`. Calcer outputs are numeric features that flow into the **existing** quantization path (`cb-data::borders` / `quantize` / `quantized_pool`) → tree (SC-4). Pool already carries raw text (`Vec<Vec<String>>`) and embedding (`Vec<Vec<Vec<f32>>>`) columns from Phase 2 DATA-01 — **no Pool schema change** is required for ingestion. (Exact module split is researcher/planner discretion.)

### Oracle ground-truth sourcing (D-07)
- **D-06 (Python-reachable first):** Primary ground truth = catboost==1.2.10 (`.venv`) models trained on text/embedding pools + the vendored `pytest/canondata` text & embedding fixtures, asserted per-stage (borders/splits/leaf/staged/prediction). SC-4 end-to-end flow is validated this way.
- **D-07 (instrument the thin internals):** Tokenizer token stream, dictionary token→id assignment, per-calcer intermediate encodings, and online-estimation permutation order are **not cleanly Python-reachable**. Reuse the **persistent instrumented catboost 1.2.10 trainer** (`/tmp/cb_build313` + clang-18, built in 06.3-10/17) with env-gated `CB_INSTRUMENT_LOG` hooks for these. OFFLINE / RUN-ONCE; **no fabricated fixtures, no weakened tolerance, no `#[ignore]`** — escalate-don't-weaken (06.3 discipline). Vendored `catboost-master/` instrumentation patches stay uncommitted (D-09/D-12).

### Claude's Discretion
- Exact module boundaries between `cb-data` (ingestion/quantization-adjacent) and `cb-compute` (pure numeric primitives) for calcer math — planner/researcher decides.
- Whether the tokenizer-first gate is one plan or split (tokenizer plan + dictionary/digitizer plan) — sequencing principle (D-01) is fixed; granularity is flexible.
</decisions>
