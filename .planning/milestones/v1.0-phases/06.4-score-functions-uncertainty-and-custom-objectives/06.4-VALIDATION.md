# Phase 06.4 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Rust built-in `#[test]` (separate test files; no inline `#[cfg(test)]`) + `cb-oracle` `compare_stage` ≤1e-5 |
| **Config file** | per-crate `Cargo.toml`; oracle fixtures under `crates/cb-oracle/` |
| **Quick run command** | `cargo test -p cb-compute` (score/loss), `cargo test -p cb-model` (predict/apply), `cargo test -p cb-train` (metrics) |
| **Full suite command** | per-crate `cargo test -p cb-compute -p cb-model -p cb-train` (root disk ~full — avoid whole-workspace link per memory `disk-pressure-and-full-suite-verification`) |
| **Estimated runtime** | ~60–120 seconds per crate |

---

## Sampling Rate

- **After every task commit:** Run the touched crate's `cargo test -p <crate>` + the relevant existing split/loss oracle (no-regression on the shipped Cosine/L2 + scalar/N-dim locks).
- **After every plan wave:** Run `cargo test -p cb-compute -p cb-model -p cb-train` (per-crate, NOT whole-workspace — disk pressure).
- **Before `/gsd-verify-work`:** All per-crate suites green AND every existing Cosine/L2 split oracle green (the 05-19 Task A lock) AND the shipped scalar/N-dim loss oracles green.
- **Max feedback latency:** ~120 seconds.
