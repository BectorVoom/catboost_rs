## Decisions Made
- `Arc<dyn>` (not `Box`) so the objective instance is shared cheaply across the train loop; `Arc::ptr_eq` identity is the only sound `PartialEq` for an opaque trait object.
- `EvalMetric` dropped `Copy` (one-time mechanical refactor, mirroring the 6.2 `Loss` Copy-drop); `Clone` retained and cheap (an `Arc` bump for `Custom`, a bitwise copy otherwise).
- Custom objective dispatched in `compute_gradients` before the scalar loop; `approx_dimension == 1` only this phase (N-dim custom is out of scope).
- The self-oracle pins `leaf_method = Gradient` to MATCH the frozen `cross_entropy` fixture (`leaf_estimation_method=Gradient`/`iterations=1` per its `config.json`); the custom path's own Newton default is a separate, untested-here concern.
- PyO3 NOT implemented (D-09 — Phase 8); trait signatures mirror the Python `calc_ders_range`/`CustomMetric` callback so the future bridge is a thin adapter.
