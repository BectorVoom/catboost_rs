## Pattern Assignments

### `crates/cb-compute/src/runtime.rs` — the `Loss` enum (config, selector dispatch)

**Analog:** `Loss::Focal { alpha, gamma }` (`runtime.rs:63-68`) for parametric variants; `Loss::Mae` (`runtime.rs:69-74`) for non-parametric.

**Parametric-variant precedent** (`runtime.rs:63-68`):
```rust
Focal {
    /// Class-balance weight `alpha ∈ (0,1)` (`focal_alpha`).
    alpha: f64,
    /// Focusing exponent `gamma > 0` (`focal_gamma`).
    gamma: f64,
},
```

**What the new code changes:** Add variants following this exact shape, CatBoost-native field names, defaults from `error_functions.h` (D-6.1-03):
```rust
Quantile { alpha: f64, delta: f64 },   // defaults 0.5 / 1e-6 (error_functions.h:468-469)
Huber    { delta: f64 },               // MANDATORY upstream — no default
Tweedie  { variance_power: f64 },      // MANDATORY upstream — 1<p<2
Lq       { q: f64 },                   // MANDATORY upstream — q>=1
Expectile{ alpha: f64 },               // default 0.5
// non-parametric:
LogCosh,
Poisson,
Mape,
```
**Critical constraint (load-bearing):** `Loss` derives `PartialEq` only — **NOT `Eq`** (`runtime.rs:46`; `f64` is not `Eq`). The doc comment at `runtime.rs:43-45` explains why. New `f64`-carrying variants MUST NOT break this; do not add `#[derive(Eq)]`.

**MSLE note (D-6.1-06 / RESEARCH A1):** MSLE is **NOT** a `Loss` variant — it is metric-only upstream (`enum_helpers.cpp:200,533-549`). It is added to `EvalMetric` (metrics.rs) only, never here.

### `crates/cb-compute/src/loss.rs` — der1/der2 transcriptions (utility, elementwise transform)

**Analog:** `rmse_der1/der2` (the per-fn citation template), `mae_der1/der2` (der2=0 + deadzone), `sigmoid` + `logloss_der1` (inline-link precedent), `focal_der*` (parametric-args precedent).

**Per-fn citation template** (`loss.rs:35-45`):
```rust
/// RMSE first derivative for one object: `der1 = target - approx`.
/// `error_functions.h:391` — `TRMSEError::CalcDer(approx, target) = target - approx`.
#[must_use]
pub fn rmse_der1(approx: f64, target: f64) -> f64 { target - approx }
```

**Inline-link precedent** (`loss.rs:54-57`, `sigmoid`) — Poisson copies this `exp()`-inside-der pattern (cb-train stores RAW approx, never exp-approx storage; compute `exp(approx)` inline exactly as Logloss computes `sigmoid` inline):
```rust
pub fn sigmoid(approx: f64) -> f64 {
    let e = approx.exp();
    1.0 - 1.0 / (1.0 + e)
}
```

**der2=0 + deadzone precedent** (`loss.rs:161-179`, `mae_der1`/`mae_der2`) — Quantile generalizes this verbatim (MAE = Quantile α=0.5); MAPE copies the `der2 = 0` shape:
```rust
pub fn mae_der1(approx: f64, target: f64) -> f64 {
    let val = target - approx;
    if val.abs() < QUANTILE_DELTA { 0.0 }
    else if val > 0.0 { QUANTILE_ALPHA }
    else { -(1.0 - QUANTILE_ALPHA) }
}
pub fn mae_der2(_approx: f64, _target: f64) -> f64 { 0.0 }
```

**Parametric-fn signature precedent** (`loss.rs:117`, `focal_der1`) — Tweedie/Lq/Huber/Expectile/Quantile take their params as trailing `f64` args:
```rust
pub fn focal_der1(approx: f64, target: f64, alpha: f64, gamma: f64) -> f64 { … }
```

**What the new code adds (all transcribed with `error_functions.h:line` doc comments — see RESEARCH per-loss tables):**
| New fn | der1 | der2 | Cite |
|--------|------|------|------|
| `logcosh_der1/2` | `-(approx-target).tanh()` | `-1/cosh²(approx-target)` | h:405-425 |
| `lq_der1/2(…,q)` | `q·sign·\|·\|^(q-1)` | `-q(q-1)\|·\|^(q-2)` (q≥2 only) | h:539-568 |
| `huber_der1/2(…,delta)` | clamp diff to ±δ | `-1` in-band else `0` | h:1596-1632 |
| `expectile_der1/2(…,alpha)` | `e>0?2α·e:2(1-α)·e` | `e>0?-2α:-2(1-α)` | h:500-537 |
| `poisson_der1/2` | `target - approx.exp()` | `-approx.exp()` (inline exp — see sigmoid precedent) | h:657-676 |
| `tweedie_der1/2(…,p)` | `t·e^((1-p)a) - e^((2-p)a)` | `t(1-p)e^((1-p)a) - (2-p)e^((2-p)a)` | h:1634-1665 |
| `mape_der1/2` | `±1/max(1.0,\|target\|)` (f32-domain `1.f` — Pitfall 7) | `0` | h:607-630 |
| `quantile_der1/2(…,alpha,delta)` | generalize `mae_der1` to arbitrary α | `0` | h:457-498 |

**Domain-guard precedent:** `FOCAL_P_MIN` clamp (`loss.rs:103`, `sigmoid(approx).clamp(1e-13, 1-1e-13)`) — mirror for positive-domain losses (Poisson exp overflow, MAPE division, Lq pow-near-zero); return typed `CbError`, never panic (`unwrap()` banned).

**Source/test separation (mandatory):** new der unit tests go in `loss_test.rs` / per-loss `*_der_test.rs`, NEVER inline `#[cfg(test)]` in `loss.rs`. `loss.rs` has no inline test mod today — keep it that way.

### `crates/cb-backend/src/kernels.rs` — `#[cube]` generics-float kernels (kernel, elementwise transform)

**Analog:** `mae_gradient_kernel` (`kernels.rs:60-74`) for non-param der2=0; `logloss_gradient_kernel` (`kernels.rs:40-51`) for inline-`exp`/`sigmoid`; `focal_gradient_kernel` (`kernels.rs:104-`) for kernels that need loss params.

**Generics-float + inline-exp precedent** (`kernels.rs:40-51`) — Poisson's `exp()` kernel copies this `F::exp` form:
```rust
#[cube(launch)]
pub fn logloss_gradient_kernel<F: Float>(approx: &Array<F>, target: &Array<F>, der1: &mut Array<F>) {
    if ABSOLUTE_POS < approx.len() {
        let e = F::exp(approx[ABSOLUTE_POS]);
        let p = F::new(1.0) - F::new(1.0) / (F::new(1.0) + e);
        der1[ABSOLUTE_POS] = target[ABSOLUTE_POS] - p;
    }
}
```

**Deadzone/branch precedent** (`kernels.rs:60-74`, `mae_gradient_kernel`) — Quantile/MAPE copy the `mut g` + if-as-STATEMENT branch shape (NOT if-as-expression — CubeCL conditionals manual):
```rust
let mut g = F::new(0.0);
if val > delta { g = alpha; }
else if val < F::new(0.0) - delta { g = F::new(0.0) - (F::new(1.0) - alpha); }
der1[ABSOLUTE_POS] = g;
```

**Loss-param-as-Array precedent** (`kernels.rs:104-111`, `focal_gradient_kernel`) — Huber/Tweedie/Lq/Expectile/Quantile pass `alpha`/`delta`/`q`/`p` as **length-1 `Array<F>`** (read at index 0), NOT scalar kernel args, to keep the kernel fully generic over `F: Float` (AGENTS.md generics-float; a generic scalar arg needs the heavier `ScalarArgType + CubeElement` bound):
```rust
pub fn focal_gradient_kernel<F: Float>(
    approx: &Array<F>, target: &Array<F>, der1: &mut Array<F>,
    alpha: &Array<F>, gamma: &Array<F>,
) { … }
```

**CubeCL rules (load-bearing, AGENTS.md):** kernels MUST use generics-float (never hard-code `f32`/`f64`); read the CubeCL manual before writing kernel code; on ANY CubeCL build error load `cubecl_error_guideline.md` before fixing — blind fixes prohibited.

### `crates/cb-backend/src/cpu_runtime.rs` — `compute_gradients` dispatch (dispatch, loss→kernel)

**Analog:** the exhaustive `match loss { … }` in `compute_gradients` (`cpu_runtime.rs:212-237`). This is an **exhaustive match** — every new `Loss` variant forces a compile error here until handled (RESEARCH Runtime State Inventory: "a feature, not a hazard").

**Constant-der2 precedent** (`cpu_runtime.rs:231-236`, `Loss::Mae`) — MAPE/Quantile (der2=0) and any other constant-hessian loss copy the "no hessian kernel, fill the vec directly" shape:
```rust
Loss::Mae => {
    let der1 = launch_binary_f64(approx, target, BinaryKernel::MaeGradient);
    // MAE / Quantile hessian is the constant 0.0 (no kernel needed).
    let der2 = vec![0.0_f64; approx.len()];
    Ok(Derivatives { der1, der2 })
}
```

**Two-kernel precedent** (`cpu_runtime.rs:226-230`, `Loss::Focal`) — smooth losses with a real der2 (LogCosh/Huber/Lq/Expectile/Poisson/Tweedie) launch BOTH a gradient and a hessian kernel:
```rust
Loss::Focal { alpha, gamma } => {
    let der1 = launch_focal_f64(approx, target, alpha, gamma, false);
    let der2 = launch_focal_f64(approx, target, alpha, gamma, true);
    Ok(Derivatives { der1, der2 })
}
```

**What the new code changes:** add one `match` arm per new variant; destructure params (`Loss::Huber { delta } => …`) the way `Loss::Focal { alpha, gamma }` does; route param-carrying kernels through a `launch_*` helper that builds the length-1 param `Array<F>`.

### `crates/cb-train/src/boosting.rs` — leaf-delta + auto-LR wiring (service, per-iteration)

**Analog A — Exact leaf branch** (`boosting.rs:709-723`): the **hardcoded** `QUANTILE_ALPHA`/`QUANTILE_DELTA` the Quantile generalization must parameterize.
```rust
LeafMethod::Exact => {
    let residuals: Vec<f64> = approx.iter().zip(target.iter())
        .map(|(&a, &t)| t - a).collect();
    let members = collect_leaf_residuals(leaf_of, &residuals, weights, n_leaves);
    members.iter()
        .map(|(r, w)| exact_leaf_delta(r, w, QUANTILE_ALPHA, QUANTILE_DELTA))  // ← hardcoded 0.5 / 1e-6
        .collect()
}
```
**What the new code changes (the ONE real-code item in the quantile family, RESEARCH Pattern 3):** thread the active loss's `(alpha, delta)` from `params.loss` through `compute_leaf_deltas` so `Quantile{alpha:0.7}` calls `exact_leaf_delta(r, w, 0.7, δ)`. **`exact_leaf_delta` is ALREADY α-general** — no change to `leaf.rs` (`leaf.rs:165-233`; tested at α=0/0.5/weighted in `leaf_test.rs:106-119`). This is wiring, not new algorithm.

**Analog B — exhaustive loss match** (`boosting.rs:637-645`, `autolr_target_type`): another exhaustive `match loss` that new variants must extend (map the new regression losses to `TargetType::Unknown` unless upstream's auto-LR table lists them):
```rust
const fn autolr_target_type(loss: Loss) -> TargetType {
    match loss {
        Loss::Rmse => TargetType::Rmse,
        Loss::Logloss | Loss::CrossEntropy => TargetType::Logloss,
        Loss::Focal { .. } | Loss::Mae => TargetType::Unknown,
    }
}
```

**Poisson exp-link wiring (the second real-code item, RESEARCH §Exp-Link):** der computes `exp(rawApprox)` inline (already covered in loss.rs/kernels.rs); the boosting loop's additive raw-approx update stays UNCHANGED; the final transform is `PredictionType::Exponent` (`cb-model`, oracle-locked Plan 04-02 — do NOT hand-roll). StagedApprox oracle compares RAW approx; Predictions compares `exp(raw)` (Pitfall 4 / A2 — validate with first Poisson fixture, Open Question Q1).

**Leaf-method pinning (Pitfall 2/3):** do NOT reproduce upstream's per-loss auto-default leaf-method switch. `BoostParams.leaf_method` (`boosting.rs:117`) and `leaf_estimation_iterations` are explicit fields; PIN them per fixture (see fixture analog) so cb-train's single-step leaf path (1 iteration only, `boosting.rs`) matches.

### `crates/cb-train/src/metrics.rs` — `EvalMetric` + `for_loss` (service, per-iter request-response)

**Analog:** the `EvalMetric` enum (`metrics.rs:43-48`), `for_loss` mapping (`metrics.rs:59-66`, exhaustive `match loss`), and the `eval` Rmse arm (`metrics.rs:113-126`).

**`for_loss` exhaustive-match precedent** (`metrics.rs:59-66`) — new `Loss` variants force a compile error here; map each to its default eval metric (MSLE/MAPE/Quantile/etc.):
```rust
pub fn for_loss(loss: Loss) -> Self {
    match loss {
        Loss::Rmse | Loss::Mae => Self::Rmse,
        Loss::Logloss | Loss::CrossEntropy | Loss::Focal { .. } => Self::Logloss,
    }
}
```

**Weighted-metric `eval` precedent** (`metrics.rs:113-126`, Rmse arm) — new regression metrics (incl. **MSLE-as-metric**, D-6.1-06) copy this `sum_f64`-folded weighted form. ALL folds route through `cb_core::sum_f64` (D-08); div guarded by the `total_weight <= 0` check at `metrics.rs:107`:
```rust
Self::Rmse => {
    let weighted_sq: Vec<f64> = approx.iter().zip(target.iter()).enumerate()
        .map(|(i, (&a, &t))| { let d = a - t; weight_at(i) * d * d }).collect();
    Ok((sum_f64(&weighted_sq) / total_weight).sqrt())
}
```

**What the new code changes:** add `EvalMetric` variants (incl. `Msle`), extend `for_loss`, add `eval` arms transcribing the matching `catboost/libs/metrics/metric.cpp` formula. MSLE log-space domain guard mirrors the Focal clamp / Logloss `clamp(1e-15, 1-1e-15)` precedent (`metrics.rs:136`).

## Shared Patterns

### Per-fn upstream-citation transcription style
**Source:** `crates/cb-compute/src/loss.rs:30-45` (and the module header `loss.rs:10-22`).
**Apply to:** every new der1/der2 fn and every new kernel doc-comment.
Each fn carries a doc comment with the exact `error_functions.h:<line>` citation and the verbatim upstream formula. This is the project's mandated documentation discipline — the planner should require it in every `action`.

### `cb_core::sum_f64` summation discipline (D-08 CI-grep ban)
**Source:** `leaf.rs:194,223` / `metrics.rs:104,125,140` / `boosting.rs` reductions.
**Apply to:** any reduction. der1/der2 are pure elementwise scalars (no sum at stake), but leaf/metric reductions MUST route through `cb_core::sum_f64` — raw `iter().sum()`/`fold(0.0,…)` is CI-grep-banned. Grep before declaring done.

### Domain guards → typed `CbError`, never panic
**Source:** Focal clamp `loss.rs:103,118` (`clamp(1e-13, 1-1e-13)`); `newton_leaf_delta` denom>0 guard (`leaf.rs:130-137`); Logloss metric clamp (`metrics.rs:136`); degenerate-weight guard (`metrics.rs:107`).
**Apply to:** all positive-domain/link losses (Poisson exp overflow, MSLE log-of-negative, MAPE division, Lq pow-near-zero, Tweedie exp) and parameter validation (`q≥1`, `1<variance_power<2`, `0≤alpha≤1`). `unwrap()` strictly prohibited in production; `thiserror`/`CbError` in libs.

### Source/test separation (mandatory CLAUDE.md rule)
**Source:** `metrics.rs:196-198` uses `#[path = "metrics_test.rs"] mod tests;`; `loss.rs` has no inline test mod.
**Apply to:** all new tests — der unit tests in a separate `*_test.rs` / `*_der_test.rs`, oracle tests in `cb-train`'s `tests/<loss>_oracle_test.rs`. NEVER an inline `#[cfg(test)] mod tests` in a production source file.

### Exhaustive `Loss` match enumeration (compile-time completeness gate)
**Source:** every `match loss` site — `cpu_runtime.rs:212`, `boosting.rs:638`, `metrics.rs:60`.
**Apply to:** adding a `Loss` variant breaks all three until handled — this is the completeness mechanism (D-05). Before declaring done: `rg "match .*loss|Loss::" crates/*/src` (RESEARCH Verification) to confirm no site is missed; grep test files too (Memory: "grep all test files when changing shared enums").
