**Comptime SharedMemory sizing** (Pitfall 3 — size MUST be a comptime `usize` const). The `BLOCK_REDUCE_SHMEM` const + `SharedMemory::<F>::new(...)` precedent, lines 22 and 597:
```rust

// ...
let mut partials = SharedMemory::<F>::new(BLOCK_REDUCE_SHMEM);   // line 597
```
The histogram line size per bit-width is a comptime const derived from `bits` (or a fixed 8-bit worst-case `2 * (1<<8)` upper bound with a used prefix). Add a new `pub(crate) const` for the worst-case histogram shmem size next to `BLOCK_REDUCE_SHMEM` — and couple it to the launch geometry with the existing `const _: () = assert!(...)` guard pattern (`gpu_runtime.rs` lines 76-80).

**In-kernel atomic-merge pattern** (D-03 / D-7.3-03). The `Atomic<F>::fetch_add` finalize, `block_reduce_atomic_kernel` lines 670-729:
```rust
#[cube(launch)]
pub fn block_reduce_atomic_kernel<F: Float>(
    input: &Array<F>,
    acc: &Array<Atomic<F>>,         // global accumulator, atomic-merged
    #[comptime] use_plane: bool,
) {
    // ... intra-cube wave-agnostic fold into `cube_partial` (plane_sum OR
    //     CUBE_DIM_X-strided shared-mem tree-reduce; NEVER a literal 32/64) ...
    if tid == 0u32 {
        acc[0].fetch_add(cube_partial);   // in-kernel atomic merge (order non-deterministic, D-03)
    }
}
```
The histogram kernel binds `bin_sums: &Array<Atomic<F>>` and `fetch_add`s each (feature,bin,channel) block-local partial into the global cell — but ONLY when more than one block contributes to a feature (upstream's `BLOCKS_PER_FEATURE > 1` guard); single-block features do a plain `Array<F>` write (mirror upstream `WriteThrough`, RESEARCH Code Examples line 294-307).

**Wave-agnostic fold (D-09 — no warp literal)**. The plane / shared-mem tree-reduce with strides from `CUBE_DIM_X`/`PLANE_DIM`, `block_reduce_atomic_kernel` lines 688-721:
```rust
if use_plane {
    let plane_total = plane_sum(val);
    // ... per-plane partials keyed by PLANE_POS; fold count = ceil(CUBE_DIM_X / PLANE_DIM) ...
} else {
    let mut shared = SharedMemory::<F>::new(BLOCK_REDUCE_SHMEM);
    let mut s = CUBE_DIM_X / 2u32;        // stride derives from CUBE_DIM_X — NEVER 32/64
    while s > 0u32 { /* tree-reduce */ s /= 2u32; }
}
```
Upstream's `threadIdx.x & 31` / `tiled_partition<32>` MUST be re-expressed via `UNIT_POS`/`CUBE_DIM_X`/`PLANE_DIM`/`UNIT_POS_PLANE` (Pitfall 4 / D-09).

**Bin-write shape** (read `cindex[indices[i]]` → bin; add `der1[i]` to channel 0, `weight[i]` to channel 1). The per-object read+write idiom, `histogram_scatter_kernel` lines 536-545:
```rust
#[cube(launch)]
pub fn histogram_scatter_kernel<F: Float>(
    der1: &Array<F>, weight: &Array<F>, contrib: &mut Array<F>,
) {
    if ABSOLUTE_POS < der1.len() {
        contrib[ABSOLUTE_POS] = der1[ABSOLUTE_POS] * weight[ABSOLUTE_POS];
    }
}
```
The histogram kernel generalizes this: bounds-guarded per-object read of `der1`/`weight`, plus `cindex`/`indices` (`&Array<u32>`) to compute the bin → accumulate into the per-block shared histogram (NOT a per-object scatter — this is the genuine D-03 fill, the new surface). der1 arrives UNWEIGHTED; the weight is the SECOND channel folded HERE (D-7.3-05).

**Generics-float + bounds-guard discipline** (AGENTS.md): every kernel is `<F: Float>`; every device read is under `if ABSOLUTE_POS < len` (T-7.1-01 idle-guard; e.g. `gradient_kernel` line 37). if-as-STATEMENT only (never if-as-expression), per every kernel in this file.

fn max_divergence(device: &[f64], baseline: &[f64]) -> (f64, f64) {

    let mut max_rel = 0.0_f64;
    for i in 0..baseline.len() {

use cb_core::sum_f64;
// per leaf: gather member contributions in ascending object order, then fold via sum_f64
let mut delta_members: Vec<Vec<f64>> = vec![Vec::new(); n_leaves];
