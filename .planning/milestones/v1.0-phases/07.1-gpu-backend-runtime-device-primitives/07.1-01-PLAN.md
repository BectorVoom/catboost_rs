First MVP vertical slice of Phase 7.1: wire ONE GPU runtime end-to-end, run ONE
device primitive (block reduction / sum), and self-oracle it against the Rust CPU
path on the real in-env gfx1100 GPU — then prove the other three backend arms
build. This is the thinnest slice that touches every layer (Cargo feature →
`SelectedRuntime` alias → generic launch helper → `#[cube]` kernel → rocm oracle).

rocm, and run a wave-size-agnostic GPU sum on rocm that provably matches
`cb-core::sum_f64`.

(cuda compile-gated), and the reduce half of GPU-01 — the foundation every later
7.x GPU kernel builds on.

real `SelectedRuntime` arms; `gpu_runtime.rs` with `launch_block_reduce_f64`;

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → device | Host-validated f64/f32 buffers cross to the GPU via `bytemuck::Bytes`; kernel reads them by index. |
| device → host | Device output buffer read back via `client.read_one`; a failed read must not masquerade as data. |
| build script → toolchain | `cubecl-hip-sys` build.rs shells out to system `hipconfig` to detect the HIP patch. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7.1-01 | Tampering/DoS (V5 input validation) | `block_reduce_kernel` device read | mitigate | Bounds-guard `ABSOLUTE_POS < input.len()` on every device read; zero-pad out-of-range lanes (`F::new(0.0)`); never index past `len()`. Host validates buffer length before launch. |
| T-7.1-02 | Tampering | `gpu_runtime.rs` read-back | mitigate | Map `client.read_one` errors to typed `CbError::Degenerate` (WR-05 pattern, cpu_runtime.rs:129-137); never return a silent all-zero buffer that masquerades as a valid reduction. |
| T-7.1-03 | Tampering (supply chain) | `cubecl-hip-sys` build.rs → `hipconfig` | accept | `cubecl-hip-sys` is an OK-verified, widely-downloaded crate (Package Legitimacy Audit, all OK); `hipconfig` is the system ROCm tool; no untrusted input flows into the build command. Low severity, non-blocking. |
| T-7.1-SC | Tampering | crates installs | mitigate | RESEARCH § Package Legitimacy Audit verified all cubecl crates OK; no `[ASSUMED]`/`[SUS]`/`[SLOP]` packages; re-pin is a `--precise` of an already-locked crate (no new package). No blocking-human checkpoint required. |
| T-7.1-04 | Memory safety | `unsafe { ArrayArg::from_raw_parts }` in launch helper | mitigate | Confined to `cb-backend` launch helpers (the established unsafe boundary); lengths derived from validated host buffers. |
</threat_model>
