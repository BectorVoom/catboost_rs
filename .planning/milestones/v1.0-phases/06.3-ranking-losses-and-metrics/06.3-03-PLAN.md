<objective>
Land the pairwise deterministic ranking losses — **PairLogit**, **PairLogitPairwise**, and
**LambdaMart** — each as a complete vertical slice (der → leaf path → oracle ≤1e-5). PairLogit
rides explicit `Pool.pairs` through the grouped seam's `Competitors` adjacency on the pointwise
leaf path; PairLogitPairwise adds the dedicated **Cholesky pairwise-leaf solve** (the single
biggest structural decision of the phase) reusing the in-house `cholesky_solve` already vendored
in `cb-compute/src/leaf.rs`; LambdaMart computes the per-group lambda gradient toward a target
metric. After this plan a user can train a pairwise-ranking model from explicit pairs and a
listwise LambdaMart model with upstream-matching predictions.

Purpose: Close Wave B of LOSS-04 (D-6.3-01) — the deterministic pairwise/listwise losses and the
pairwise Cholesky leaf path the `*Pairwise` variants require.
Output: PairLogit/LambdaMart der arms + Loss variants + pairwise_leaves.rs Cholesky path +
is_pairwise_scoring routing + three per-stage oracles + three frozen fixtures.
</objective>
