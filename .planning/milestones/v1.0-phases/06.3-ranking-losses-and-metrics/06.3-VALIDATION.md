# Phase 06.3 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Source: 06.3-RESEARCH.md § Validation Architecture (transcribed). Ranking losses
> (LOSS-04) + ranking metrics (LOSS-05), each oracle-locked ≤1e-5 vs catboost 1.2.10.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Rust built-in `#[test]`, separate test files only (INFRA-06 / CLAUDE.md — no inline `#[cfg(test)]`) |
| **Config file** | none — Cargo workspace test discovery |
| **Quick run command** | `cargo test -p cb-compute --tests` (der unit tests) |
| **Full suite command** | `cargo test -p cb-train --tests` (oracle suite incl. ranking) + `cargo test -p cb-compute --tests` |
| **Estimated runtime** | ~per-crate; verify per-crate under disk pressure (see caveat) |

> **Disk caveat** (memory: `disk-pressure-and-full-suite-verification`): the `cb-compute`
> test profile historically could not link under disk pressure. Verify **per-crate** and free
> disk before the full suite. Disk is healthier now (~67G free) but per-crate verification
> stays the default.

---

## Sampling Rate

- **After every task commit:** `cargo test -p cb-compute --tests` (fast der units) for der tasks; the per-loss oracle test for fixture tasks.
- **After every plan wave:** `cargo test -p cb-train --tests` (full ranking oracle + D-04 scalar/N-dim no-regression).
- **Before `/gsd-verify-work`:** Full `cb-train` + `cb-compute` suites green; D-04 scalar/N-dim oracles must remain green (no-regression on shipped pointwise locks).
- **Max feedback latency:** per-crate test runtime (seconds–low minutes).
