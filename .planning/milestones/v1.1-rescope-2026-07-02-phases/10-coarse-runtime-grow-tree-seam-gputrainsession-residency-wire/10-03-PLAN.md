<objective>
Deliver GPUT-02: the `GpuTrainSession` (cb-backend-internal) that owns ONE `ComputeClient` plus
the persistent device handles for the whole fit, with the quantized feature matrix uploaded
**once** above the iteration loop (no per-tree re-upload). `GpuBackend` (today a zero-sized unit
struct) gains `RefCell<Option<GpuTrainSession>>` interior mutability (Pattern 2). It overrides
`begin_device_training` to run the D-10-02 coverage gate and construct the session (returning
`Ok(true)`), `end_device_training` to drop it. The actual on-device grow + read-back-elimination
is GPUT-03 in plan 10-04 — this plan stops at session lifecycle + upload-once + the gate.

This wires the seam override that 10-04 fills in (`grow_tree_on_device`) and that 10-02's
wiring drives. The coverage gate is the per-fit all-or-nothing decision point (D-10-01/02).

Purpose: Stand up the residency container + the per-fit gate so depth-1/RMSE-or-Logloss/Plain/
fold=1 configs allocate one client and upload the matrix once; everything else falls back to CPU.
Output: GpuTrainSession + GpuBackend session field + begin/end overrides + coverage gate + in-env smoke test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → device (session upload) | quantized cindex/indices/weight cross into device buffers; malformed lengths/shape are the untrusted vector (internal numeric, no network/auth) |
| ComputeClient ↔ Handle | a Handle is bound to its allocating client; cross-client read is UB-adjacent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | Tampering / DoS | cindex length / 2^depth overflow at session begin | mitigate | reuse the existing guards — `LengthMismatch` on `cindex.len()==n_features*n`, `checked_shl`/`checked_mul` before any upload |
| T-10-03-02 | Tampering (UB-adjacent) | reading a Handle through a client other than the session's | mitigate | one ComputeClient per session held for the whole fit; residency rule enforced |
| T-10-03-03 | Repudiation (wrong result looks valid) | a covered session silently degrading instead of erroring | mitigate | gate decides Some/None once; a covered session that cannot proceed surfaces CbError (D-10-02), never a silent fallback |
| T-10-03-SC | Tampering | npm/pip/cargo installs | accept | no new packages installed (RESEARCH Package Legitimacy Audit — zero new deps) |
</threat_model>
