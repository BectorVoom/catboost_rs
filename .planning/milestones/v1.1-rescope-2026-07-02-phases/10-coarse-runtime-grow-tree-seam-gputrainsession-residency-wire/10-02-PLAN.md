<objective>
Wire the GPUT-01 seam into cb-train's boosting loop. `train` is already generic over
`R: Runtime` (`boosting.rs:1870`) so no signature change is needed. Before the per-iteration
loop, call `begin_device_training(...)` once (per-fit all-or-nothing gate, D-10-01). Inside the
loop, when the session is active, replace the CPU grower dispatch with a call to
`grow_tree_on_device`, mapping the returned `DeviceGrownTree` `(feature, bin_id)` splits to a
cb-train `GrownTree` via `border = feature_borders[feature][bin_id]` (Pattern 4 â€” the one
non-obvious correctness join). After the loop, call `end_device_training`. The CPU arm stays
**byte-unchanged** (D-04). A covered session that returns `None` mid-run is a hard
`CbError::Degenerate`, NOT a silent CPU graft (D-10-02).

At this wave the only available `Runtime` impls return the default `Ok(false)` (the GpuBackend
override lands in 10-03/10-04), so this plan is validated via the CPU-fallback path: a normal
fit is unchanged. The device path is exercised end-to-end by the Kaggle harness in 10-05.

Purpose: Make the device grow loop *reachable* from training behind the per-fit gate, with the
binâ†’border join correct, while guaranteeing the CPU path is untouched.
Output: device-grow branch + mapping helper in train_inner + a D-04 fallback test.
</objective>
