# builder_oracle_test root cause: score-function mismatch

`cargo test -p catboost-rs --test builder_oracle_test` has two failing tests
(`builder_regression_full_cycle`, `builder_binclf_full_cycle`), both diverging at
the **Predictions stage** (regression ~0.131, binclf ~0.0033). The "pre-existing
facade default drift" label carried across Phases 6.1–6.6 is **misleading**.

## Concrete root cause (confirmed by code, not yet empirically isolated)

The public facade trains with a **different split-scoring function** than the
fixtures were generated with, and **the facade exposes no way to override it**:

| Path | score_function | Set where |
|------|----------------|-----------|
| Facade `fit` (FAILING) | **Cosine** | `crates/catboost-rs/src/builder.rs:286` → `score_function_default()` → `crates/cb-train/src/boosting.rs:545` = `EScoreFunction::Cosine`. No `.score_function()` setter exists on `CatBoostBuilder`. |
| Fixtures `model_serde/{binclf,regression}` | **L2** | `config.json:26` `"score_function": "L2"` (pinned at generation). |
| Passing cb-train oracle (e.g. `loss_oracle_test.rs`) | **L2** | sets `EScoreFunction::L2` explicitly **and** loads pinned borders from `model.json`. |

Score function governs **split selection during tree growing** → Cosine vs L2 pick
different splits → different tree structure → predictions diverge at exactly the
structural magnitude observed. The test's `configured_builder` pins loss, depth,
lr, l2, seed, leaf-method, boost-from-average — everything *except* the one knob it
physically cannot reach through the public API. The test was **never capable** of
matching these fixtures.

## The reframe

Cosine is catboost's **real CPU default** (builder comment cites
`oblivious_tree_options.cpp:22`; corroborated by memory note
`cb-train-uses-l2-but-catboost-defaults-cosine`). So the **facade default is
correct**; the **Phase-4 fixture is the outlier** — generated by explicitly passing
`score_function=L2` to upstream catboost, a non-default choice. This is a
fixture/facade **config disagreement**, not a math bug in the facade.

## VERIFIED 2026-06-19 (/gsd-debug borders-vs-score-fn-builder)

A 2×2 (score_function × borders-source) isolation experiment settled it
**conclusively** — score function is the **sole** cause; borders are exonerated:

| score_function | borders | regression error | binclf error |
|----------------|---------|-------------------|--------------|
| **L2** | pinned *or* computed | **2.404e-8** | **2.843e-9** |
| **Cosine** | pinned *or* computed | **5.555e-1** | **7.261e-2** |

- Flipping Cosine→L2 (borders held fixed) collapses the error from ~0.56 to ~2e-8 →
  converges ≤1e-5 against both fixtures. The Cosine row reproduces the exact failing
  test values (1.2025143 / 0.2018587).
- Swapping computed↔pinned borders (score held fixed) changes predictions by < f64
  print precision → **borders contribute nothing**.

**Bonus finding (latent doc bug):** the `builder_oracle_test.rs` docstring claim that
computed borders reproduce upstream exactly for `numeric_tiny` is **false** — computed
yields 49 borders vs the fixture's pinned 2/2/0/3. It just happens not to affect
predictions here. Worth correcting the docstring when the fix lands.

## Fix direction (decided — VERIFIED)

**PREFERRED: expose a `.score_function(EScoreFunction)` setter** on `CatBoostBuilder`
and have `builder_oracle_test` opt into `L2` to match its fixtures. Keep
`score_function_default() = Cosine` (catboost's correct CPU default). Rationale:
- The fixtures encode a deliberate, valid CatBoost config (L2) that every passing
  per-crate cb-train/cb-compute oracle already locks. Regenerating them to Cosine
  would **discard that L2 oracle coverage** and re-pin model.json + predictions.npy
  across binclf+regression — more churn, more fragile.
- The real defect is a **missing public-API capability** (no score-function setter),
  not a wrong fixture. This is the smallest change that closes the facade↔fixture gap
  *and* adds a first-class catboost param to the builder.

**ALTERNATIVE (not recommended):** regenerate `model_serde/{binclf,regression}` with
`score_function=Cosine`. Cheaper in code, but loses the L2 oracle lock and re-pins
multiple committed artifacts.
