<objective>
Wire the GPUT-01 seam into cb_train's boosting loop so the device grow path is reachable from fit(): begin the session once per fit, grow one tree per iteration on device (or fall back to the CPU grower on Ok(None)), resolve each DeviceGrownTree's (feature, bin_id) splits to borders, and fold the tree into the Model exactly as a CPU-grown tree. Per-fit all-or-nothing (D-10-01).

Purpose: The device grow loop existed but was UNWIRED — this is the reachability step that closes GPUT-01 end-to-end and makes depth-1 device training actually run from the public fit path.
Output: the device branch + bin->border join + Ok(None) fallback in cb-train/src/boosting.rs.

## Artifacts this phase produces (this plan)
- Device branch in `train_inner`'s per-iteration body calling `begin_device_training` / `grow_tree_on_device` / `end_device_training`
- The `bin_id -> border = feature_borders[feature][bin_id]` resolution folding a DeviceGrownTree into the Model
- Per-fit all-or-nothing decision + Ok(None) -> CPU-grower fallback
</objective>
