<objective>
Deliver the sort/reorder, partition-bookkeeping, and fill/transform GPUT-16 primitives. The stable single-bit reorder (`reorder_one_bit`) is built first (it exercises exclusive-scan + scatter, both partly existing) and LSD radix sort composes from it. `TDataPartition` offset/size update produces the {Offset,Size} layout the depth>1 histograms (Phase 11) will key on. fill/gather/vector-arithmetic consolidate the trivial transforms.

Purpose: These primitives + scan + reduce complete the from-scratch substrate every later histogram/partition kernel stands on.
Output: reorder_one_bit + radix sort + partitions update + fill_transform in kernels.rs with serial self-oracles.

## Artifacts this phase produces (this plan)
- `reorder_one_bit` `#[cube]` (exclusive-scan of `(key>>bit)&1`, scatter to `zeroesBefore` / `totalZeros+onesBefore`) in `kernels.rs`
- `radix_sort` LSD multi-pass (per-digit histogram -> exclusive scan -> scatter) in `kernels.rs`
- `update_partition_offsets` / `update_partition_sizes` (TDataPartition {Offset,Size}) in `kernels.rs`
- `fill_transform` kernels (fill / gather / vector add/sub/mul/div) in `kernels.rs`
- `kernels/sort.rs`, `kernels/partitions.rs`, `kernels/fill_transform.rs` self-oracles + `mod` decls
</objective>
