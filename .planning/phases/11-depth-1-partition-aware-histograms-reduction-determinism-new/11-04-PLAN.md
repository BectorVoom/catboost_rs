<objective>
Add Newton der2 leaf estimation on device (GPUT-07) — required for the Logloss classification default. Extend the per-partition stat reduce with a 3rd Σ(der2·weight) channel, reuse the Phase 7.2 `LoglossHessian` der2 handle (const −1 for RMSE), and swap the leaf value from the gradient average to `newton_leaf_delta` for the Logloss arm. leaf_estimation_iterations is pinned to 1 (single closed-form step — the CPU oracle does NOT run an iterative walker or backtracking; A1/Pitfall 1).

Purpose: RMSE's der2 = −1 collapses Newton to `calc_average`, so RMSE needs no new leaf math; Logloss is the genuinely-new path. Keeping the der1/der2 recompute + `apply_leaf_delta` refinement fully device-resident (D-01, no per-iteration readback) preserves the residency that is the entire point of the milestone.
Output: the +der2 partition channel + widened read-back in mod.rs/kernels.rs, the Newton leaf value + `apply_leaf_delta` refinement, and a Newton self-oracle in grow_loop.rs.
</objective>
