# Phase 06.4 Plan 03: Wave-B Uncertainty Prediction Types (LOSS-06) Summary

Implemented the three RMSEWithUncertainty uncertainty `PredictionType`s —
`RmseWithUncertainty` (single-model variance), `VirtEnsembles` (per-ensemble
matrix), `TotalUncertainty` (mean / knowledge / data decomposition) — plus the
`apply_virtual_ensembles` model-slicing apply, closing the Phase-4 D-10 deferral.
Oracle-locked <=1e-5 against an OFFLINE-generated catboost 1.2.10
`virtual_ensembles_predict` / `predict(prediction_type=...)` fixture (D-6.4-02 full
Python-reachable). End-to-end: a Rust caller loads the RMSEWithUncertainty
`model.json`, slices the trained tree sequence into 5 virtual ensembles, and the
single-model variance / per-ensemble matrix / regression-uncertainty decomposition
all match upstream.

## What was built

- **`PredictionType::{RmseWithUncertainty,VirtEnsembles,TotalUncertainty}`
  (predict.rs):** three unit variants extending the Phase-4 enum (Copy+Eq intact —
  no params).
- **`apply_prediction_type` RmseWithUncertainty arm (predict.rs):** the single-model
  2-col transform over the DIM-MAJOR raw approx `[mean(0..n), log-scale(n..2n)]` ->
  OBJECT-MAJOR `(n,2)` `[mean, variance = exp(2*log-scale)]`. The variance is
  `CalcSquaredExponent(approx[1]) = exp(2x)` (Pitfall 6, `eval_processing.h:47`) —
  NOT `exp(x)` and NOT `x²`; the log-scale dim is `0.5*log(variance)`.
- **`apply_ve_prediction_type` (predict.rs, NEW):** the VE-aware transform over the
  object-major `(n,V,dim)` matrix. `VirtEnsembles` = identity with every ODD
  (log-scale) dim -> `exp(2x)` in place. `TotalUncertainty` = per object over its V
  ensembles: `mean = (1/V)Σ mean_e`, `knowledgeUncertainty = (1/V)Σ (mean_e-mean)²`
  (epistemic), `dataUncertainty = (1/V)Σ exp(2*log-scale_e)` (aleatoric) ->
  `[mean, knowledge, data]` per `CalcRegressionUncertaitny` (dimShift=2). All Σ via
  `cb_core::sum_f64` (D-08).
- **`apply_virtual_ensembles` (apply.rs, NEW):** the model-slicing per-ensemble
  apply transcribing `apply.cpp:526-600` EXACTLY — `eval_period = end/(2V)` (INTEGER
  division), guard `eval_period>0 && eval_period*V < end` returning a `ModelError`
  ("Not enough trees …", NOT a panic) on <2V+1 trees / V==0; `begin =
  end-eval_period*V`; ensemble 0 seeds from trees `[0,begin)` PLUS the per-dim bias;
  each ensemble adds the apply of its `eval_period`-tree slice (bias-free, the
  `treeStart>0` path) and copies the running sum forward (the `copyerLambda`,
  copy-forward seed). Default `unshrinkCoef=1` (A2, `model_shrink_rate=0`).
- **`apply_tree_slice_one` (apply.rs, NEW helper):** per-object dim-major leaf sum
  over a tree RANGE without bias (Σ via `sum_f64`); backs both
  `apply_virtual_ensembles` and `predict_raw_multi_biased`.
- **`predict_raw_multi_biased` (apply.rs, NEW):** the per-dimension-bias N-dim apply
  (Rule 2 correctness fix — see Deviations): seeds output dim `d` with
  `per_dim_bias[d]` instead of the scalar `Model.bias`, so the dim-1 log-scale bias
  `0.5*log(var)` is not dropped.
- **Offline generator + frozen fixture:** `uncertainty_predict_fixture.py` (run ONCE
  vs `.venv` catboost==1.2.10, the SAME RMSEWithUncertainty model shape as Plan 02 —
  11 trees, V=5, `model_shrink_rate=0`) -> frozen `model.json` +
  `rmse_with_uncertainty.npy` (n,2) + `virt_ensembles.npy` (n,5,2) +
  `total_uncertainty.npy` (n,3) + `config.json`.
- **`uncertainty_predict_test.rs` (NEW):** the per-type <=1e-5 oracle (3 prediction
  types) + a too-few-trees error test (T-06.4C-01).

## Key finding (the load-bearing one)

`ApplyScaleAndBias` (`scale_and_bias.cpp:13`) adds the model bias **only when
`treeStart == 0`**. In the VE slicing the base ensemble (trees `[0, begin)`) is the
only `treeStart == 0` apply — it carries the per-dim bias `[mean, 0.5*log(var)]`;
every subsequent ensemble slice `[begin+..)` runs `treeStart > 0` (bias-free, scale=1
no-op). The bias therefore appears exactly once per ensemble, carried forward via the
copy-forward seed. Adding the bias per slice would over-count by `(V-1)*bias` and
blow the <=1e-5 gate. This is also why the single-model `predict_raw_multi` (scalar
`Model.bias` seeded into every dim) produced the WRONG dim-1 log-scale — the variance
came out short by exactly `(bias_dim1 - bias_dim0)`, fixed by `predict_raw_multi_biased`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing critical functionality] per-dim bias for multi-dim-bias models**
- **Found during:** Task 3 (the RmseWithUncertainty oracle failed at the variance
  column: expected 5.2786, got 0.6511 — index 1 only).
- **Issue:** `predict_raw_multi` seeds EVERY output dim with the scalar `Model.bias`
  (06.4-02 kept `Model.bias` a scalar dim-0 f64). For RMSEWithUncertainty the
  `scale_and_bias[1]` is a genuine per-dim vector `[0.315, 1.361]` = `[mean,
  0.5*log(var)]`; dropping the dim-1 bias made the raw log-scale short by exactly
  `(1.361-0.315)=1.046`, so `exp(2*log-scale)` was wrong by `exp(2*1.046)`.
- **Fix:** Added `predict_raw_multi_biased(model, feature_values, per_dim_bias)`
  which seeds output dim `d` with `per_dim_bias[d]`; the oracle reads the per-dim
  bias from `scale_and_bias[1]` of model.json. The scalar `predict_raw_multi`
  surface is left byte-identical (D-04 untouched).
- **Files modified:** `crates/cb-model/src/apply.rs`, `crates/cb-model/src/lib.rs`
- **Commit:** d8d469a

**2. [Rule 3 - Blocking issue] VE transforms need a dedicated VE-aware entry point**
- **Found during:** Task 1 / Task 3 (the flat VE matrix length `n*V*dim` is not
  self-describing — `n` vs `V` is ambiguous, and TotalUncertainty reduces per-object
  over V ensembles).
- **Issue:** The plan sketched VirtEnsembles/TotalUncertainty as `apply_prediction_type`
  arms, but a single `&[f64]` cannot recover the `(n,V,dim)` structure.
- **Fix:** Added `apply_ve_prediction_type(prediction_type, ve, V, dim)` for the two
  VE types; `apply_prediction_type` keeps the self-describing single-model
  RmseWithUncertainty arm. The Task-1 RED test (already committed) was updated in
  Task 3 to call the VE-aware entry point.
- **Files modified:** `crates/cb-model/src/predict.rs`, `crates/cb-model/src/lib.rs`,
  `crates/cb-model/tests/uncertainty_predict_test.rs`
- **Commit:** d8d469a
