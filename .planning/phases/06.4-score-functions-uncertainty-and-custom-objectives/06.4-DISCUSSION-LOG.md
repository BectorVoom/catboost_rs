| Option | Description | Selected |
|--------|-------------|----------|

**User's choice:** Trait-object seam (recommended).
**Notes:** Method signatures mirror upstream's Python callback (`calc_ders_range`, custom-metric `evaluate`) so the Phase-8 PyO3 bridge is a thin adapter (D-09). Attaches to the `Loss` enum (already `Copy`/`Eq`-free, `runtime.rs:43-55`); planner resolves the Clone/PartialEq-vs-`dyn` tension. Oracle = reimplement a built-in (RMSE/Logloss) as a custom objective → match its existing oracle ≤1e-5. A generic param is viral and perturbs shipped monomorphizations; a closure-only API doesn't wrap cleanly for PyO3. Python callback bridge DEFERRED to Phase 8. → **D-6.4-05.**
