<objective>
Wave A (LOSS-09): add the five remaining split-score functions — SolarL2, NewtonL2,
NewtonCosine, LOOL2, SatL2 — to the existing single `EScoreFunction`/`multi_dim_split_score`
scorer seam, with the Newton variants consuming the summed `der2` hessian via the histogram
fill. End-to-end slice: a Rust caller can select any of the five score functions and get the
candidate-split score the upstream CUDA arithmetic dictates.

Purpose: complete the `EScoreFunction` API to upstream feature parity (Cosine/L2 shipped 05-19
Task A) without a parallel scorer (D-6.4-03 single-code-path), preserving the shipped Cosine/L2
split locks.

**WEAKENED-ORACLE CAVEAT (D-6.4-06 — NON-NEGOTIABLE):** the five functions are GPU-ONLY in
catboost 1.2.10 (CPU rejects them at `oblivious_tree_options.cpp:146`, verified live). There is
NO upstream-CPU training ground truth for them. The oracle is TRANSCRIBE-THEN-SELF-ORACLE:
assert the Rust port reproduces the hand-computed CUDA `score_calcers.cuh` arithmetic on chosen
leaf-stat inputs. This is a documented, user-confirmed weakening; it is NOT a ≤1e-5-vs-upstream
lock and must never be reported as one.

Output: `EScoreFunction::{SolarL2,NewtonL2,NewtonCosine,LOOL2,SatL2}`, 5 new `multi_dim_split_score`
arms, the Newton histogram hessian-fill branch, and `crates/cb-compute/tests/score_functions_test.rs`.
</objective>
