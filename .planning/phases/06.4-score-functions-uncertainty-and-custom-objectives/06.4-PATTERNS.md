### `crates/cb-compute/src/score.rs` — `multi_dim_split_score` 5 new arms (Wave A, LOSS-09)

**Analog:** the existing `L2` / `Cosine` match arms, SAME function, `score.rs:127-144`.

**Pattern to copy** (`score.rs:120-144`) — the numerator accumulator + per-arm transform:
```rust
let mut num_terms: Vec<f64> = Vec::new();
for leaves in per_dim_leaves {
    for &stats in leaves {
        num_terms.push(add_leaf_plain(stats, scaled_l2));
    }
}
let numerator = sum_f64(&num_terms);
match score_function {
    EScoreFunction::L2 => numerator,
    EScoreFunction::Cosine => {
        let mut den_terms: Vec<f64> = Vec::with_capacity(num_terms.len() + 1);
        den_terms.push(1e-100);                          // seeded first summand
        for leaves in per_dim_leaves {
            for &stats in leaves {
                let avrg = calc_average(stats.sum_weighted_delta, stats.sum_weight, scaled_l2);
                den_terms.push(avrg * avrg * stats.sum_weight);
            }
        }
        let denominator = sum_f64(&den_terms);
        numerator / denominator.sqrt()
    }
}
```

**Per-leaf terms to transcribe** (RESEARCH Strand 1 table, from `score_calcers.cuh`; each is a per-leaf scalar folded through `sum_f64`):
- **SolarL2:** `weight > 1e-20 ? (-sum*sum) * (1 + 2*ln(weight + 1.0)) / weight : 0.0`
- **LOOL2:** `let adjust = if weight>1.0 { weight/(weight-1.0) } else { 0.0 }; let adjust = adjust*adjust; if weight>0.0 { adjust*(-sum*sum)/weight } else { 0.0 }`
- **SatL2:** `let adjust = if weight>2.0 { weight*(weight-2.0)/(weight*weight-3.0*weight+1.0) } else { 0.0 }; if weight>0.0 { adjust*(-sum*sum)/weight } else { 0.0 }`
- **NewtonL2:** reuses the `L2` arm verbatim (`numerator`); the difference is the FILL (see histogram/tree below), not this formula (RESEARCH `:85`).
- **NewtonCosine:** reuses the `Cosine` arm verbatim; same — the FILL differs, not the math (RESEARCH `:86`).

Here `sum = stats.sum_weighted_delta`, `weight = stats.sum_weight`. **CRITICAL (Pitfall 8):** every per-leaf scalar must enter a `Vec` folded via `cb_core::sum_f64` — no `iter().sum()`/`fold(0.0,…)`. The `ln`/`adjust` factors are computed per leaf, pushed, then `sum_f64`'d, mirroring the existing arms.

**No-regression gate (Pitfall 3):** `L2`/`Cosine` arms stay byte-identical — do NOT touch `num_terms`/`den_terms` fold order. The five new arms are additive.

### `crates/cb-compute/src/histogram.rs` + `crates/cb-train/src/tree.rs` — Newton hessian fill (Wave A, LOSS-09, D-6.4-03)

**Analog (both halves already exist):** `reduce_leaf_stats` (`histogram.rs:49`, fills `sum_weight` with `Σ weight`) and `reduce_leaf_der2` (`histogram.rs:100`, sums `Σ der2*weight`). The Newton fill is the COMPOSITION: fill the `LeafStats.sum_weight` slot with the **positive summed hessian** instead of the weight count.

**`reduce_leaf_stats` weight-fill site to branch** (`histogram.rs:77-86`):
```rust
(0..n_leaves)
    .map(|leaf| {
        let deltas = delta_members.get(leaf).map_or(&[][..], Vec::as_slice);
        let weights = weight_members.get(leaf).map_or(&[][..], Vec::as_slice);
        LeafStats {
            sum_weighted_delta: sum_f64(deltas),
            sum_weight: sum_f64(weights),   // <-- Newton variants feed Σ(-der2) here instead
        }
    })
    .collect()
```

**Tree-search dispatch site** (`tree.rs:43-47` `split_score`, and the candidate builder `multi_dim_candidate_score` `:398-429`):
```rust
fn split_score(score_function: EScoreFunction, leaves: &[LeafStats], scaled_l2: f64) -> f64 {
    match score_function {
        EScoreFunction::Cosine => cosine_split_score(leaves, scaled_l2),
        EScoreFunction::L2 => l2_split_score(leaves, scaled_l2),
    }
}
```

**Mechanism (RESEARCH `:88-94`, Pitfall 2):** `IsSecondOrderScoreFunction` ⇒ only `NewtonL2`/`NewtonCosine`. For those, the `LeafStats.sum_weight` slot is populated with the **summed `der2` negated to a positive hessian** (RMSE der2 = `-1`, Logloss der2 = `-p(1-p)`, both ≤0 → negate to `+`). `LeafStats` struct is UNCHANGED; only the fill branches. First-order functions (Cosine/L2/SolarL2/LOOL2/SatL2) keep feeding `Σ weight` → byte-identical to shipped locks. Route the hessian sum through `cb_core::sum_f64` (reuse the `reduce_leaf_der2` `:115-119` fold, then negate).

**No-regression:** the existing `reduce_leaf_stats` weight path stays the default; the Newton branch is gated on `score_function` so the 05-19 Task A L2-vs-Cosine split lock is preserved.

### `crates/cb-compute/src/loss.rs` — RMSEWithUncertainty 2-dim der (Wave B, LOSS-08)

**Analog:** `multiclass_onevsall_ders` (`loss.rs:517`) and `multi_crossentropy_ders` (`loss.rs:554`) — the diagonal-hessian per-dimension der helpers that return an unweighted `(der1, der2)` pair.

**Pattern to copy** (`loss.rs:554-559`):
```rust
#[must_use]
pub fn multi_crossentropy_ders(approx_d: f64, target_d: f64) -> (f64, f64) {
    let p = sigmoid(approx_d);
    let der1 = target_d - p;
    let der2 = -p * (1.0 - p);
    (der1, der2)
}
```

**Formula to transcribe** (RESEARCH Strand 2, `error_functions.h:280-313`) — RMSEWithUncertainty is 2-dim with a diagonal hessian, so it returns BOTH dims at once (unlike the per-dim helpers above):
```text
diff = target[0] - approx[0]
prec = (-2.0 * approx[1]).exp()           // FastExp(-2*log-scale); f64::exp within 1e-5 (A4)
der[0] = diff                              // weight folded in by caller
der[1] = diff*diff * prec - 1.0
der2[0] = -1.0
der2[1] = -2.0 * diff*diff * prec
```
Single scalar target (`Y_ASSERT(target.size()==1)`), two approx dims. **Diagonal** → each dim gets an INDEPENDENT scalar Newton step via the existing 6.2 per-dim leaf path (the `multiclass_onevsall_ders` precedent), NOT a dense solve (Pitfall 4). Leaf default Newton/1-iter (`catboost_options.cpp:77-82`) — pin in fixture. Any accumulation into leaf stats routes through `cb_core::sum_f64`.

### `crates/cb-compute/src/custom.rs` (NEW) + `Loss::Custom` / `CustomObjectiveHandle` (Wave C, LOSS-07)

**Analog (trait shape):** the `Runtime` trait (`runtime.rs:821`, `Send + Sync`-bounded, methods returning `CbResult`). **No in-repo `Arc<dyn>`/`Box<dyn>` trait-object exists in production code** — this is the workspace's first; the mechanism below is transcribed from RESEARCH Strand 3 (verified-idiomatic), not from an existing analog.

**Trait pair to create** (RESEARCH `:186-206`):
```rust
// crates/cb-compute/src/custom.rs
pub trait CustomObjective: Send + Sync {
    fn calc_ders_range(
        &self,
        approxes: &[f64],
        targets: &[f64],
        weights: &[f64],            // empty == all 1.0 (upstream convention)
        ders: &mut [(f64, f64)],
    ) -> CbResult<()>;
}
pub trait CustomMetric: Send + Sync {
    fn evaluate(&self, approxes: &[f64], target: &[f64], weight: &[f64]) -> CbResult<(f64, f64)>;
    fn get_final_error(&self, error: f64, weight: f64) -> f64;
    fn is_max_optimal(&self) -> bool;
}
```

**The `dyn`-in-derived-enum newtype** (RESEARCH `:212-226`; required because `Loss` derives `Clone + PartialEq` at `runtime.rs:55` and bare `dyn` is neither):
```rust
#[derive(Clone)]
pub struct CustomObjectiveHandle(pub Arc<dyn CustomObjective>);
impl PartialEq for CustomObjectiveHandle {
    fn eq(&self, other: &Self) -> bool { Arc::ptr_eq(&self.0, &other.0) }  // pointer identity
}
impl std::fmt::Debug for CustomObjectiveHandle {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        f.write_str("CustomObjectiveHandle(<dyn>)")
    }
}
```
Then `Loss::Custom(CustomObjectiveHandle)` — `Loss`'s `#[derive(Debug, Clone, PartialEq)]` (`runtime.rs:55`) now compiles because the field implements all three manually. **Use `Arc`, not `Box`** (cheap `Clone`; the 6.2 refactor already converted by-value `Loss` call sites to `&Loss`/`.clone()`, see `builder.rs:47-53` "`Copy` is NOT derived"). The error type for `CbResult` is `cb_core::CbError` (`thiserror`, never `anyhow`).

### `crates/cb-model/src/apply.rs` — `apply_virtual_ensembles` (Wave B, LOSS-08)

**Analog:** `predict_raw_one` (`apply.rs:224`) — the per-tree leaf-contribution accumulator that already iterates the tree sequence and folds via `sum_f64`:
```rust
fn predict_raw_one(model: &Model, features: &[f32], cat_values: &[String]) -> f64 {
    let leaf_contributions: Vec<f64> = model.oblivious_trees.iter()
        .map(|tree| {
            let leaf = leaf_index_for(model, tree, features, cat_values);
            tree.leaf_values.get(leaf).copied().unwrap_or(0.0)
        })
        .collect();
    model.bias + sum_f64(&leaf_contributions)
}
```

**New slicing to transcribe** (RESEARCH Strand 2, `apply.cpp:526-600`; Pitfall 5):
```text
evalPeriod = end / (2 * virtualEnsemblesCount)     // INTEGER division
assert evalPeriod > 0 && evalPeriod*V < end         // need >= 2V+1 trees
begin = end - evalPeriod * virtualEnsemblesCount
ensemble[0] seed = apply trees [0, begin)
for vIdx in 0..V:
    add apply of trees [begin+vIdx*evalPeriod, begin+(vIdx+1)*evalPeriod) to running approx
    (except last) copy running sum forward to seed next ensemble

```

**`CalcRegressionUncertaitny` for TotalUncertainty** (RESEARCH `:153-160`, `eval_helpers.cpp:209-269`, `dimShift=2`):
```text
mean = (1/V) Σ_e approx[e*2][obj]
knowledgeUncertainty = (1/V) Σ_e (approx[e*2][obj] - mean)^2     // var of ensemble means (epistemic)
dataUncertainty = (1/V) Σ_e exp(2 * approx[e*2+1][obj])          // mean predicted variance (aleatoric)
output = [mean, knowledgeUncertainty, dataUncertainty]
```

## Shared Patterns

### Summation discipline (`cb_core::sum_f64`)
**Source:** `crates/cb-compute/src/score.rs:25,54` (`use cb_core::sum_f64;` then `sum_f64(&terms)`).
**Apply to:** ALL new parity-critical accumulation — SolarL2 `ln` factor, Newton hessian fill, VE `mean`/`var`/`dataUncertainty` Σ, custom-metric `evaluate` reductions. NEVER `iter().sum()` / `fold(0.0, +)` (D-08 CI-grep ban, Pitfall 8).
```rust
let terms: Vec<f64> = leaves.iter().map(|&s| add_leaf_plain(s, scaled_l2)).collect();
sum_f64(&terms)
```

### Diagonal N-dim der helper shape
**Source:** `crates/cb-compute/src/loss.rs:517,554` — unweighted `(der1, der2)` per object/dim, caller folds weight.
**Apply to:** RMSEWithUncertainty der (2-dim diagonal) + the custom-objective `(der1, der2)` contract.

### Two-column / N-column prediction transform
**Source:** `crates/cb-model/src/predict.rs:74-93` (`Probability`/`LogProbability`).
**Apply to:** the 3 uncertainty `PredictionType` arms (RmseWithUncertainty = pure transform; VirtEnsembles/TotalUncertainty = VE-aware).

### `Arc<dyn>` newtype in a `Clone + PartialEq` enum
**Source:** NEW — no in-repo analog; transcribed from RESEARCH `:212-226`. Manual `Clone` (Arc bump) + `Arc::ptr_eq` `PartialEq` + custom `Debug`.
**Apply to:** `Loss::Custom(CustomObjectiveHandle)` and `EvalMetric::Custom(CustomMetricHandle)`. Forces an `EvalMetric` `Copy`-drop (mirrors the shipped 6.2 `Loss` Copy-drop).

### Error handling
**Source:** `cb_core::{CbError, CbResult}` (`metrics.rs:218` `Err(CbError::Degenerate(...))`); `thiserror` in libraries; `unwrap()`/`anyhow` banned (CLAUDE.md, Security V5).
**Apply to:** custom-trait methods return `CbResult`; NaN/Inf der guards (the `Focal` `[1e-13,1-1e-13]` clamp precedent, `runtime.rs:71`).

### Oracle test file convention (source/test separation)
**Source:** `crates/cb-train/tests/multiquantile_oracle_test.rs`, `loss_oracle_test.rs`, `crates/cb-model/tests/predict_oracle_test.rs`; `cb-oracle` `compare_stage` ≤1e-5 (`crates/cb-oracle/src/compare.rs`); fixtures under `crates/cb-oracle/fixtures/`.
