## THE Governing Pattern: discriminator-gated second branch (carry-forward from Phase 5)

The single most load-bearing pattern for this phase is already proven in the codebase: **`ModelSplit::{Float, Ctr}`**. Phase 5 added CTR splits as a *parallel enum variant* without touching the float path. Every consumer (`apply`, `fstr`, `shap`, `cbm`, `json`, `from_trained`) matches the enum exhaustively and the `Float` arm is byte-identical to pre-CTR. Phase 6.6's non-symmetric tree variant MUST follow this exact shape — gate on `TreeVariant`/`grow_policy`, keep the oblivious arm untouched.

**Reference excerpt — the enum + exhaustive-match discipline** (`model.rs:65-98`, `apply.rs:196-201`):
```rust
// model.rs — the parallel-variant precedent to mirror as TreeVariant
pub enum ModelSplit {
    Float(Split),
    Ctr(CtrSplit),
}

// apply.rs — every consumer matches exhaustively; the Float arm stays byte-identical
fn passes_split(model: &Model, split: &ModelSplit, features: &[f32], cat_values: &[String]) -> bool {
    match split {
        ModelSplit::Float(s) => passes_float_split(s.feature, s.border, features),
        ModelSplit::Ctr(c) => passes_ctr_split(model, c, cat_values),
    }
}
```

**Apply this to:** `model.rs` (TreeVariant), `apply.rs`, `cbm.rs`, `json.rs`, `fstr.rs`, `shap.rs`, `from_trained`. The oblivious arm of every match is the existing code, moved verbatim under `TreeVariant::Oblivious(_)`.

## Pattern Assignments

### `crates/cb-train/src/tree.rs` — `leaf_wise_grower(policy)` (service, batch/transform) — FEAT-06

**Analog:** `greedy_tensor_search_oblivious_perturbed` (`tree.rs:526`) + `select_level_plain` (`tree.rs:573`)

**Grower skeleton pattern** (`tree.rs:526-559`): the existing oblivious grower loops `for _level in 0..depth { best = select_level_*(...); chosen.push(best) }` then `assign_leaves`. The non-symmetric grower replaces the fixed-depth level loop with a policy-parameterized queue (Depthwise = level-order; Lossguide = best-gain priority queue), producing a node graph instead of a flat `Vec<Split>`. Reuse `score_candidate`, `select_best_candidate` (strict `>` first-wins), and `assign_leaves` verbatim.

**Candidate enumeration to reuse** (`tree.rs:582-595`): float features ascending × borders ascending, scored through `score_candidate`. This per-leaf candidate scoring is the same primitive; only the *queue/expansion* differs by policy.

**Strict first-wins tie-break — DO NOT change** (`tree.rs:274-285`):
```rust
pub fn select_best_candidate(candidates: &[Candidate]) -> Option<Candidate> {
    let mut best: Option<Candidate> = None;
    let mut best_score = MINIMAL_SCORE;
    for &candidate in candidates {
        if candidate.score > best_score { // STRICT `>`: first-wins on equal gain
            best_score = candidate.score;
            best = Some(candidate);
        }
    }
    best
}
```

**New GrownTree fields:** follow the `GrownTree` precedent (`tree.rs:196-216`) which ALREADY carries parallel optional vectors (`ctr_splits`, `level_kinds`) that are EMPTY for the searches that don't use them. Add `step_nodes`/`node_id_to_leaf_id` (or a `variant` enum) the same way — empty for the oblivious grower so existing callers keep compiling unchanged.

**RESEARCH-specific gotchas the planner must thread into the plan:**
- Depthwise (Pattern 2, `greedy_tensor_search.cpp:1509`): `for curDepth { for leaf in level { if gain < 1e-9 continue; AddSplit; push left/right } }`.
- Lossguide (Pattern 3, `greedy_tensor_search.cpp:1806`): max-priority-queue; stop on empty OR `leaf_count >= max_leaves`; **replicate the upstream queue-penalty NON-recompute (lines 1875-1882) exactly** — penalize at enqueue only (Pitfall 4).

### `crates/cb-train/src/tree.rs` — float / per-object penalty multipliers (service, transform) — FEAT-04

**Analog:** `cat_feature_weight()` (`tree.rs:1095`)

**The exact pattern to extend** (`tree.rs:1095-1101`):
```rust
fn cat_feature_weight(count: usize, max_count: usize, model_size_reg: f64) -> f64 {
    if model_size_reg == 0.0 || max_count == 0 {
        return 1.0;
    }
    let ratio = count as f64 / max_count as f64;
    (1.0 + ratio).powf(-model_size_reg)
}
```
This is the *multiplicative* penalty already wired into `select_level_ctr_aware` (`tree.rs:1104`). Per RESEARCH Pitfall 6, the three FEAT-04 penalties differ in sign/placement:
- `feature_weights` — **multiplicative** gain factor in `SelectBestCandidate` (`score *= GetSplitFeatureWeight`), same insertion site as `cat_feature_weight`.
- `first_feature_use_penalties` + `per_object_feature_penalties` — **subtractive** in a separate `PenalizeBestSplits` pass (`score -= penalty * penalties_coefficient`), only when the feature is unused.

Add a `feature_weight(feature, &feature_weights)` helper next to `cat_feature_weight`, and a subtractive penalty pass after candidate scoring. Both fold through `cb_core::sum_f64` (the per-object unused-count sum especially).

let mut index = tree_start_offsets[tree_id];

    let step = step_nodes[index];          // {left_subtree_diff, right_subtree_diff}
