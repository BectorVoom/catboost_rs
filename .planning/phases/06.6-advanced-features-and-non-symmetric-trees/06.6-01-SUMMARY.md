## Decisions Made
- **Insertion sites:** `feature_weights` multiply at the `select_level_plain` candidate-scoring loop (the float analogue of the existing `cat_feature_weight` CTR site); the subtractive first-use/per-object penalties on the same per-candidate score, gated on the feature being globally unused. This matches the upstream `SelectBestCandidate` (gain multiply) + `PenalizeBestSplits` (subtraction) split.
- **No-regression gating:** a `FeaturePenalties::is_noop()` short-circuit + a `penalties_active` flag mean the entire penalty path is skipped when all vectors are empty, keeping `candidate.score` numerically untouched (D-6.6-05).
- **Fixture target feature:** all three penalty vectors target float feature 1 (the unpenalized favourite at split positions 3 and 9), confirmed to change the chosen splits vs an unpenalized baseline, so the oracle genuinely exercises the penalty math.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Fixture directory path corrected to the harness convention**
- **Found during:** Task 2 (fixture generation + test authoring)
- **Issue:** The plan's frontmatter listed fixtures under `crates/cb-oracle/tests/fixtures/penalty/`, but the established cb-train oracle `fixture()` helper resolves `crates/cb-oracle/fixtures/` (no `tests/` segment exists in the repo). Using the plan path would make the test unable to find the fixtures.
- **Fix:** Generated and committed the fixtures under `crates/cb-oracle/fixtures/penalty/{feature_weights,first_use,per_object}/`, matching the harness convention used by every other cb-train oracle test (e.g. `slice_first_oracle_test.rs`). The `.gitkeep` artifact from the plan was placed under that same penalty dir.
- **Files modified:** `crates/cb-oracle/fixtures/penalty/...`
- **Verification:** `cargo test -p cb-train --test penalty_oracle_test` passes all 3 tests (the fixtures resolve).
- **Committed in:** `fc11e54` (Task 2 commit)

**2. [Rule 2 - Missing Critical] per_object penalty magnitude raised 0.05 -> 0.1 so the fixture is a genuine test**
- **Found during:** Task 2 (fixture validation against an unpenalized baseline)
- **Issue:** An initial `per_object_feature_penalties=[0,0.05,0,0]` left the chosen splits byte-identical to the unpenalized baseline (the penalty margin was smaller than the split-gain gap), so the fixture would have passed even if the per-object code path were a no-op — a false-confidence oracle.
- **Fix:** Empirically tuned the per-object penalty to `0.1`, the smallest value that changes the chosen splits (feature 1 -> feature 0 at split position 3), so the fixture exercises the per-object subtraction.
- **Files modified:** `crates/cb-oracle/generator/gen_penalty_fixtures.py`, regenerated `penalty/per_object/*`
- **Verification:** Baseline-vs-penalty split-feature diff confirms divergence; oracle passes <=1e-5.
- **Committed in:** `fc11e54` (Task 2 commit)
