<objective>
Close the single partial gap left by Phase 06.6 verification (SC-3 / FEAT-06): the
`.cbm` SAVE path for a model trained by the Rust non-symmetric leaf-wise grower is
broken by the CR-02 sentinel mismatch. `leaf_wise_grower` initializes
`node_id_to_leaf_id` to `vec![0; node_count]` (tree.rs:1137) and writes only the leaf
slots, so every interior node keeps `0`. The cbm serializer (cbm.rs:200) treats any
id `!= u32::MAX` as a real leaf, so it over-counts `distinct_leaves` by the number of
interior nodes and `save_cbm()` fails with
`ModelError::SchemaVersion("non-symmetric leaf_values length is not a multiple of
approx_dimension")`. The existing round-trip oracle tests pass only because they
round-trip an UPSTREAM-LOADED model (whose interior slots are correctly decoded as
`u32::MAX` by cbm.rs:813) — never a Rust-grower-trained one.

This plan makes the three changes the verifier's `missing:` block specifies, 1:1:
  1. tree.rs finalization: init `node_id_to_leaf_id` to `vec![u32::MAX; node_count]`
     and set the `BuiltNode::Interior` slot to `u32::MAX` explicitly.
  2. tree.rs (lines ~1148-1149): replace the bare `as u16` step-node diff truncation
     with a checked `u16::try_from(...).map_err(...)?` returning a typed `CbError`
     (CR-01, bundled — same finalization loop, latent correctness bug).
  3. A NEW oracle test that trains a non-symmetric model via the Rust grower
     (`grow_policy=Depthwise`), `save_cbm` → `load_cbm` → `predict_raw`, asserting the
     reloaded predictions match the in-memory model ≤1e-5.

Purpose: Fully achieve the stated SC-3 goal "full train + non-symmetric apply +
.cbm/json round-trip oracle-locked ≤1e-5" for the GROWER-produced path (D-6.6-02 —
the round-trip is part of the non-symmetric engine gate, not a follow-on).
Output: a fixed `leaf_wise_grower` finalization loop, a new oracle test, and the
`cb-model` dev-deps wiring it needs.

Scope fence (do NOT touch): CR-03 (eval-set non-sym no-op), CR-04 (non-sym CTR split),
WR-01, WR-02 (.cbm→json non-identity), WR-03, WR-04 (shap recursion) are verifier-
classified LATENT / outside-fixture-scope and are explicitly NOT part of this gap
closure. The oblivious train / apply / .cbm / json path stays BYTE-IDENTICAL (D-6.6-05).
</objective>
