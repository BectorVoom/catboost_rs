## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Non-symmetric .cbm decode under-counted Lossguide distinct leaves**
- **Found during:** Task 2 (LeafValues stage: lossguide expected 24, decode produced 16)
- **Issue:** The 06.6-03 decode counted `distinct_leaves` as `(0,0)` step nodes only. Upstream's Lossguide `.cbm` compresses one-sided halts into `(d,0)`/`(0,d)` nodes that still carry a leaf value (`evaluator_impl.cpp:738` halts whenever the CHOSEN diff is 0). It also kept `node_id_to_leaf_id` as GLOBAL flat-array indices while slicing `leaf_values` per-tree-local, so the ids didn't index the local slice.
- **Fix:** Count distinct leaves as VALID (non-`u32::MAX`) `NonSymmetricNodeIdToLeafId` slots; localize each node's GLOBAL value-unit id to a 0-based per-tree leaf index (`(global - value_base) / dim`); re-globalize symmetrically in the encode so the round-trip stays struct-identical.
- **Files modified:** `crates/cb-model/src/cbm.rs`
- **Commit:** `6fcb0e8`

**2. [Rule 1 - Bug] json unflatten could not export a compressed one-sided-halt tree**
- **Found during:** Task 2 (model.json round-trip stage)
- **Issue:** `unflatten_non_symmetric` treated any non-`(0,0)` node as a 2-child interior node; a `(1,0)` node produced `right_id == id` (self-reference) — wrong nested form.
- **Fix:** A zero-diff side now emits a synthetic leaf child reading the node's own leaf id (the json nested form expands one-sided halts); the non-zero side descends to `id + diff`.
- **Files modified:** `crates/cb-model/src/json.rs`
- **Commit:** `6fcb0e8`

### Not needed (already wired by 06.6-04)
**3. [Plan scope] boosting.rs non-symmetric leaf-VALUE estimation**

It was ALREADY wired in 06.6-04: the leaf-wise grower's `grown.leaf_of` feeds the existing shared `compute_leaf_deltas` Gradient/Newton/Exact machinery (`leaf_value_leaf_of = grown.leaf_of`, `boosting.rs:3520`) with NO forked formula, and the resulting `leaf_values`/`leaf_weights` are pushed into `non_symmetric_trees` (`boosting.rs:3957`).
