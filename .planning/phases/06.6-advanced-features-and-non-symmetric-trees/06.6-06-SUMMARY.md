# Phase 06.6 Plan 06: LossFunctionChange + Non-Symmetric fstr (Gate-C) Summary

**`loss_function_change()` (MODEL-03 / D-12, completing the deferred Phase-4 partial) — the metric-delta importance built on `predict_raw` + `shap_values` + an additive Logloss `finalError`, oracle-locked <=1e-5 vs catboost 1.2.10 `get_feature_importance(type='LossFunctionChange', data=pool)` — PLUS `prediction_values_change` / `interaction` generalized from the bit-indexed oblivious loop to the non-symmetric node graph (`step_nodes` children + `node_id_to_leaf_id`, D-6.6-10), with the oblivious arm kept BYTE-IDENTICAL (D-6.6-05) and a non-symmetric Depthwise PVC + Interaction oracle-locked <=1e-5.**

## Accomplishments
- **`loss_function_change(model, cols, labels, n_features)`** in `fstr.rs` transcribing `loss_change_fstr.cpp:154-356`: for each feature `f`, `score[f] = finalError(approx - shap[·][f]) - finalError(approx)`, where `approx = predict_raw(model, cols)`, `shap[obj][f]` is feature `f`'s per-document SHAP contribution (`crate::shap_values`), and `finalError` is the additive Logloss mean per-object BCE (best value = Min, difference used verbatim). Reproduced the upstream vector bit-for-bit in Python first, then matched <=1e-5.
- **`FeatureImportanceType::LossFunctionChange`** variant added (completes the Phase-4 D-12 gap).
- **`prediction_values_change` generalized** (D-6.6-10): branches on tree variant. Oblivious arm = `pvc_accumulate_oblivious` (the literal pre-6.6 bit-indexed `leaf ^ (1<<bit)` loop, byte-identical). Non-symmetric arm = `pvc_accumulate_non_symmetric`, the node-graph `CalcEffectForNonObliviousModel` (`feature_str.h:149-228`): pass 1 seeds terminal `NodeInfo{value,count}` from `node_id_to_leaf_id` and pushes interior triangles in node order; pass 2 pops LIFO (bottom-up) combining child `NodeInfo` and accumulating `dif = (val1-avrg)^2*c1 + (val2-avrg)^2*c2` into `res[featureIdx]`. One-sided `(d,0)`/`(0,d)` halt nodes are BOTH terminal and interior (06.6-05 semantics).
- **`interaction` generalized** (D-6.6-10): oblivious bit-indexed arm byte-identical; non-symmetric arm = `interaction_accumulate_non_symmetric`, the per-tree signed DFS (`feature_str.cpp:226-264`) over a `(feature,sign)` path (left child sign=-1, right=+1), accumulating `sign1*sign2*delta` at terminals, then `Σ|treeSignedDelta|` per pair across trees — fed into the SAME `CalcFeatureInteraction` scoring (`score = sum/totalEffect*100`, descending) the oblivious arm uses.
- **`feature_count`** widened to span both `oblivious_trees` splits and `non_symmetric_trees` tree_splits.
- **catboost-rs facade**: made the `FeatureImportanceType` match exhaustive (Rule 3) and added `feature_importance_with_data(type, pool)` for the data-bearing LossFunctionChange path.
- **Oracle fixtures** (offline, catboost 1.2.10 `.venv`, thread_count=1, pinned seeds): binclf LossFunctionChange vector + its X/y, a Depthwise non-symmetric `.cbm`/json + its PVC / Interaction / LossFunctionChange vectors, plus the `gen_fixtures.py` generator.
