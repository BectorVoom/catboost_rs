<objective>
Extend the histogram scorer to the online-CTR-feature scoring path: rewrite `score_candidate_ctr_aware` / `select_level_ctr_aware` (used by `greedy_tensor_search_oblivious_with_ctr`) to score candidates from a per-level `BucketHistogram` over BOTH the float feature columns AND the CTR bin columns (`CtrFeatureColumn.bins`), replacing the per-candidate `assign_leaves_ctr_aware` + `reduce_leaf_stats` rescan.

Purpose: PERF-02 explicitly names the online-CTR-feature scoring path alongside the grow policies. CTR candidates are `Ctr{col,border}` splits whose test is `bin > border` over an integer bin column — structurally the same 2-channel histogram as a float feature, so they fold into the same primitive (RESEARCH §Coverage). This is the last CPU scoring path in scope.

Output: Rewritten CTR-aware level search in `crates/cb-train/src/tree.rs` scoring float + CTR candidates from one per-level histogram; the CTR oracle suite bit-exact.
</objective>
