<objective>
Wire the DEFAULT and dominant CPU path — the oblivious `SymmetricTree` grower (`greedy_tensor_search_oblivious` / `_perturbed` via `select_level_plain` and `select_level_perturbed`) — to score every candidate from the Plan-01 per-level `BucketHistogram` + `O(n_bins)` prefix scan + subtraction trick, replacing the per-candidate `assign_leaves`+`reduce_leaf_stats` full-dataset rescan. Introduce a reusable `GrowScratch` (incremental `leaf_of` + fixed-size histogram arrays) so the O(n) cost is paid ONCE per level, not once per candidate.

Purpose: This is the change that closes the ~250–450× slowdown for the dominant path (Spike 002/003) and makes PERF-01 true (flat `n_bins` sweep). The parity crux is already proven at the unit level (21-01); here the gate is the full oblivious oracle suite staying bit-exact (PERF-02).

Output: Rewritten candidate scoring inside `select_level_plain` / `select_level_perturbed` in `crates/cb-train/src/tree.rs`, a `GrowScratch` struct reused across levels, and an explicit deferral note for the ordered-boosting path.
</objective>
