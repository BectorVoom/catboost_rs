<objective>
Extend the histogram scorer to the non-symmetric leaf-wise growers: rewrite `best_split_for_leaf` (the shared leaf-wise scoring core used by `leaf_wise_grower` for both `Depthwise` and `Lossguide`, and reused by `region_grower`) to score its leaf's document subset from a per-leaf `BucketHistogram` + prefix scan + two-children subtraction trick, replacing the per-candidate `assign_leaves`/`reduce_leaf_stats` rescan over the docs.

Purpose: PERF-02 requires ALL CPU grow policies (SymmetricTree done in 21-02; Depthwise, Lossguide here) to use the histogram scorer while every ≤1e-5 oracle stays bit-exact. Because `best_split_for_leaf` is the single shared core (RESEARCH §Architecture), converting it once covers Depthwise, Lossguide, AND Region.

Output: Rewritten `best_split_for_leaf` in `crates/cb-train/src/tree.rs`; leaf_wise_grower and region_grower unchanged in structure but now histogram-backed; the non-symmetric and region oracle suites bit-exact.
</objective>
