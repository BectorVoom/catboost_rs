<objective>
Land the parity-critical CPU histogram primitives in cb-compute — a per-(leaf,feature,bin) 2-channel `TBucketStats` histogram (Σder1, Σweight), an `O(n_bins)` prefix scan that turns a feature's buckets into the per-border `LeafStats` array fed to the UNCHANGED score math, and the subtraction trick (child = parent − sibling) — behind bit-exact equivalence unit tests that MUST pass BEFORE any grow-loop wiring.

Purpose: The entire phase is a data-production change behind an unchanged score interface (RESEARCH §Summary). This plan builds and PROVES that data-production layer in isolation so the summation-order / tie-flip / subtraction-rounding risk (Pitfalls 1, 2, 4) is caught by fast unit tests, not by a downstream oracle regression. This is the parity crux the phase depends on.

Output: New pure-host primitives in `crates/cb-compute/src/histogram.rs` (mirroring the READ-ONLY device template `host_reference_hist2` in `crates/cb-backend/src/kernels/pointwise_hist.rs:106-163`, transcribed — NOT imported) plus equivalence tests in `crates/cb-compute/src/histogram_test.rs`.
</objective>

<artifacts_this_phase_produces>
New symbols introduced by THIS plan (cb-compute, pure host Rust, cubecl-free + rayon-free):
- `BucketHistogram` — struct holding a flat `Vec<f64>` in the frozen `(leaf * n_features * n_bins + feature * n_bins + bin) * n_channels + channel` layout (mirrors `pointwise_hist.rs:44-49`), plus `n_leaves`/`n_features`/`n_bins`/`n_channels` shape fields. `n_channels` = `approx_dimension` delta channels + 1 weight channel.
- `build_bucket_histogram(bins, der1, weight, leaf_of, n_leaves, n_features, n_bins, approx_dimension) -> BucketHistogram` — ONE object-order pass; each cell gathered in ascending object order then folded via `cb_core::sum_f64` (transcribes `host_reference_hist2`).
- `BucketHistogram::remove(&self, sibling) -> BucketHistogram` (or in-place `remove_assign`) — the subtraction trick (`TBucketStats::Remove`, `calc_score_cache.h:88`), per-cell `-=`.
- `scan_borders_to_leaf_stats(hist, feature, n_borders, approx_dimension) -> Vec<Vec<LeafStats>>` — the `O(n_bins)` prefix scan: for each border index produces the canonical-leaf-order `LeafStats` (each existing leaf split into false=`bins<=b` / true=`bins>b`, forward-bit convention), per dimension, to feed `multi_dim_split_score` / `split_score` UNCHANGED.
- `bin_of(borders: &[f64], value: f32) -> usize` — count of borders strictly less than `value` (upper-bound), consistent with `FeatureMatrix::passes_float`'s strict `>` (`tree.rs:360-365`); guards Pitfall 4.
- New tests in `crates/cb-compute/src/histogram_test.rs` (already registered `#[cfg(test)] mod histogram_test;` in lib.rs).
