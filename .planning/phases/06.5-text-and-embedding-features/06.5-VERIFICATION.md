## Known Residuals — Per Verifier Scrutiny

### Residual 1: BM25 per-stage normalized-border gap (SC-2, BLOCKER for FEAT-01)

**What the executors claimed:** BM25 calcer math is bit-exact ≤1e-5 against an independent closed-form reference. The BM25 per-stage trained-model border normalization (splits.npy reaches ±1.24 while raw BM25 scores are O(1e-3)) is a TRAINER/serialization concern deferred to a follow-on trainer-estimated-feature-normalization slice.

**Verifier finding:** The claim is real and accurately represented — there is no fabrication or hidden failure. The BM25 calcer math IS bit-exact at the encoding level (bm25_oracle_test 3/3 confirmed running). The deferred item is genuinely a trainer normalization concern (not a calcer-math defect), correctly diagnosed.

**But SC-2 as stated in ROADMAP reads:** "Text calcers BoW, NaiveBayes, BM25 produce upstream-matching encodings ≤1e-5." This wording is a full per-stage gate on all three. No bm25_oracle_splits_match_upstream / bm25_oracle_predictions_match_upstream test exists. The per-stage trained-model oracle for BM25 is not closed.

**Classification:** BLOCKER for SC-2 / FEAT-01. The normalization gap is a legitimate, non-fabricated follow-on concern; it does NOT indicate a calcer-math defect. But the per-stage parity bar for BM25 is not met, so SC-2 is not fully passed and FEAT-01 is not complete.

**Future phase coverage check:** Phase 6.6 success criteria address FEAT-03/04/05/06 (monotone constraints, penalties, feature selection, grow policies). Phase 6.6 does NOT mention BM25 normalized borders or estimated-feature quantization-grid parity. No later phase in the roadmap currently owns this gap. It is a real, unassigned gap.

### Residual 2: LDA binarization-stability argument (SC-3)

**What the executors claimed:** The LDA raw projection diverges 3.9e-2 per document from upstream (due to upstream's vendored-CLAPACK ssyev_ non-reference iterate). The documented PROJECTION_TOL=6e-2 applies to the raw vector only. The model per-stage oracle stays byte-identical because 0/16 documents cross the upstream projection split border (binarization-stability HARD gate), min margin 1.34.

**Verifier finding:** The argument is sound and the evidence is rigorous:
- The lda_oracle_scatter_inner_matches_upstream and lda_oracle_scatter_total_matches_upstream tests confirm scatter is ≤4.66e-10 (effectively exact) vs the instrumented dump.
- The lda_oracle_projection_divergence_does_not_cross_border test confirmed the HARD 0-crossing constraint with min_margin assertion >1.0.
- The upstream CLAPACK inconsistency (eigenvalue 376.67 with Rayleigh quotient 38.28, residual 338) is documented in-file and is a genuine upstream artifact, not papering over a Rust bug.
- lda_oracle_test: 5/5 passed, 0 ignored — confirmed by running the test.

**Classification:** VERIFIED. The documented intermediate tolerance is scoped correctly (raw projection vector only, not model outputs). The model per-stage parity is byte-identical and gated by a necessary-and-sufficient binarization-stability condition. No #[ignore], no weakened model oracle.

### Residual 3: SC-4 estimated-feature quantization-grid parity gap (SC-4)

**What the executors claimed:** The SC-4 *join* (text+embedding+numeric into one model, correct StagedApprox/Predictions ≤1e-5) is closed. The *quantization grid* (the exact border VALUES upstream stores for estimated features — KNN integer-vote border at 0.5 vs the Rust greedy-logsum midpoint 1.5; BoW digitization grid) is deferred. An XOR-structured non-degenerate corpus was prototyped and rejected because it forced the grid question into scope and staged/predictions did NOT match ≤1e-5 there.

**Verifier finding:** The degenerate-separating corpus design is an honest scoping decision that isolates the join question (now closed) from the grid-normalization question (still open). The degenerate corpus is not a fabrication — several features genuinely perfectly separate 8/8 classes in the fixture. The structure-invariant gating (leaf-value MULTISET, not ordered comparison) is methodologically sound for an ambiguous-orientation tie. The StagedApprox/Predictions ≤1e-5 HARD assertion is the correct parity contract for SC-4.

The grid gap is real but is the SAME root concern as the BM25 normalized-border gap (both are trainer estimated-feature normalization). It is not an independent additional blocker beyond the BM25 gap already classified.

**Classification:** VERIFIED for the SC-4 join question (pool→calcer→quantize→tree flow). The grid-normalization question is the same root concern as the BM25 gap (BLOCKER 1).

## Gaps Summary

The phase delivers 3 of 4 success criteria cleanly:

- **SC-1 (CLOSED):** Tokenizer parity is bit-identical across all three assertions (token stream, dictionary ids, TText). The D-01 gate passed.
- **SC-3 (CLOSED):** LDA uses a documented and justified intermediate tolerance on the raw projection vector only; the model per-stage parity is byte-identical (binarization stability, 0/16 crossings, min margin 1.34 >> divergence 3.9e-2). KNN is bit-exact at the neighbor-id level, making the model oracle strictly bit-exact. Both FEAT-02 calcers are at or above the required parity bar.
- **SC-4 (CLOSED for join question):** The mixed text+embedding+numeric Pool→calcer→quantize→tree pipeline is wired and the StagedApprox + Predictions HARD gate passes ≤1e-5 bit-for-bit. The degenerate-corpus design is an honest scoping isolation, not evasion.

**SC-2 (PARTIALLY CLOSED — one open gap):** BoW and NaiveBayes are fully per-stage oracle-locked ≤1e-5. BM25 calcer math is exact at the encoding level. However, the BM25 per-stage trained-model oracle — comparing Splits/LeafValues/StagedApprox/Predictions against the frozen catboost 1.2.10 BM25 fixture — does NOT exist, and the stored normalized border scale (±1.24) is not reproduced. The ROADMAP SC-2 wording is "BoW, NaiveBayes, BM25 produce upstream-matching encodings ≤1e-5" — BM25 is not at that bar. This is a real, legitimately-scoped gap (not a calcer-math bug — the upstream trainer's estimated-feature normalization is the root cause), but it means SC-2 and FEAT-01 are not fully closed.

**No later roadmap phase currently owns the BM25 normalized-border / estimated-feature quantization-grid gap.** Phase 6.6's success criteria address FEAT-03..06 (monotone, penalties, feature selection, grow policies), not estimated-feature normalization. The gap needs an explicit ownership decision.
