## Fixed Issues

### CR-01: Unguarded divide-by-`size` in public `total_scatter` / `between_matrix` produces silent NaN/Inf

**Files modified:** `crates/cb-compute/src/embedding_calcers.rs`, `crates/cb-compute/src/lda_linalg.rs`
**Commit:** 865a1ba
**Applied fix:** Added an `if !(size > 0.0)` guard (catches 0.0, -0.0, NaN) at the
top of both `total_scatter` and `between_matrix`, returning
`CbError::Degenerate` before any `total_size() / size` weight is computed.
Additionally hardened `cholesky_lower` so the SPD check uses `!(sum > 0.0)`
instead of `sum <= 0.0`, which now also rejects non-finite (NaN/inf) pivots that
the old comparison silently let through.

### WR-01: `KnnCalcer::update` class cast diverges from upstream `(ui32)target` on negative / fractional labels

**Files modified:** `crates/cb-compute/src/embedding_calcers.rs`
**Commit:** 0a3635a
**Applied fix:** Replaced the silent `target >= 0.0 ? target as usize : 0`
bucketing with explicit validation
(`target.is_finite() && target >= 0.0 && target.fract() == 0.0`), returning
`CbError::OutOfRange` on any non-integer / negative / non-finite label instead
of folding it into class 0.
**Note:** This is a label-domain semantic change (logic). Frozen binary fixtures
(labels 0/1) still pass the KNN oracle, but a developer should confirm no
intended caller relies on the old silent-bucketing behavior — flagged for human
verification.

### WR-02: KNN class index clamped into range, masking parallel-array desync

**Files modified:** `crates/cb-compute/src/embedding_calcers.rs`
**Commit:** a9ee028
**Applied fix:** Replaced `result.get_mut(class.min(feature_count-1))` with a
checked `result.get_mut(class).ok_or_else(...)?` that errors with
`CbError::OutOfRange` when `class >= feature_count`, mirroring the `ok_or_else`
pattern already used for the missing-id case two lines above.

### WR-03: `offline_text_features` does not validate `num_classes` against the calcer width / empty input

**Files modified:** `crates/cb-train/src/estimated/online_text.rs`
**Commit:** fcd0121
**Applied fix:** Added an explicit `if num_classes == 0 { return Err(...) }`
guard at the top of `offline_text_features`, consistent with the
`bm25_compute` / `naive_bayes_compute` precondition, so an empty-`texts` +
zero-class request errors instead of silently returning a width-1 all-zero
shape.

### WR-04: `softmax_in_place` silently no-ops on all-`-inf` / NaN input

**Files modified:** `crates/cb-compute/src/text_calcers.rs`
**Commit:** 135b860
**Applied fix:** Changed `softmax_in_place` to return `CbResult<()>` and error
(`CbError::Degenerate`) when the computed max is non-finite (empty / all-`-inf` /
NaN), rather than silently returning the input un-normalized. Caller
`naive_bayes_compute` propagates via `?`. The happy path (finite max) is
numerically unchanged, so fixtures are unaffected (NaiveBayes oracle green).

### WR-07: `naive_bayes_log_prob` unseen-token accumulation in an order-dependent running scalar

**Files modified:** `crates/cb-compute/src/text_calcers.rs`
**Commit:** b7235cc
**Applied fix:** Counted unseen token-pairs into `unseen_count` and computed the
adjustment once as `token_prior * (unseen_count as f64)` (exact, order-free)
instead of a running `+= token_prior` in HashMap/TText iteration order. The
NaiveBayes oracle (5/5) confirms parity is preserved within ≤1e-5.

### WR-08: dictionary / bigram sort comparator compares `Option`, masking OOB as `Ordering::Equal`

**Files modified:** `crates/cb-data/src/text/dictionary.rs`, `crates/cb-data/src/text/bigram_dictionary.rs`
**Commit:** b1baeab
**Applied fix:** Added `debug_assert!` that all `counts.get`/`tokens.get`
(`pairs.get`) lookups are `Some` inside both sort comparators, so a future
out-of-bounds index panics in debug instead of silently collapsing to
`Ordering::Equal`/`Less` and corrupting the deterministic
(count DESC, token/ngram ASC) order. Checked `.get` access (CLAUDE.md no-raw-index
rule) is retained.

### WR-05: tokenizer-options / dictionary key-consistency coupling not enforced

**Files modified:** `crates/cb-data/src/text/digitizer.rs`
**Commit:** 37fdd8a
**Applied fix:** Documented the invariant prominently (`# Invariant (WR-05)`) on
both `digitize_document` and `digitize_document_bigram`: the `tokenizer_options`
must match those used to BUILD the dictionary, otherwise tokens silently miss and
the document/bigram block becomes all-zero with no error. Directed callers to
drive both passes from `build_bow_estimated_features`, the single owning source.

### WR-06: dead brittle two-pass perl `calcer_encoding` hook rewrites source in place

**Files modified:** `crates/cb-oracle/generator/build_instrumented_trainer.sh`
**Commit:** 1f39f24
**Applied fix:** Deleted the dead `perl -pi -e` block (whose `$cb_prev` regex
could never match) entirely, keeping only the deterministic awk insertion that
already does the real work. Verified the script still parses (`bash -n`).

### IN-01 / IN-04: unused `LDA_DEFAULT_REG` constant and magic flush thresholds

**Files modified:** `crates/cb-compute/src/embedding_calcers.rs`
**Commit:** 5d46aa1
**Applied fix:** (IN-01) Added a doc NOTE that `LDA_DEFAULT_REG` has no in-tree
caller and is retained for parity reference only. (IN-04) Promoted the inline
`128` / `32` flush thresholds to named constants
`INCREMENTAL_CLOUD_BASE_FLUSH_FLOOR` and `INCREMENTAL_CLOUD_ADDITIONAL_FLUSH_BATCH`
with `lda.cpp` references, and used them in `add_vector`.

### IN-05: empty-dictionary `unknown_token_id` collides with first valid id under Insert

**Files modified:** `crates/cb-data/src/text/dictionary.rs`, `crates/cb-data/src/text/bigram_dictionary.rs`
**Commit:** 7ab3729
**Applied fix:** Documented on both `unknown_token_id` methods that the empty
dictionary yields id `0` (colliding with a first token under `Insert`), that the
digitizer default `Skip` policy never emits it, and that `Insert` over an empty
dictionary is undefined / not used (matches upstream formula).

### IN-03: two narrow `Option` states for the online-text calcer is a state-validity smell

**Files modified:** `crates/cb-train/src/estimated/online_text.rs`
**Commit:** 52bb908
**Applied fix:** Replaced the `nb_state: Option<..>` / `bm_state: Option<..>`
"exactly one is Some" pair with a local `enum ActiveState { NaiveBayes(..),
Bm25(..) }`, making the invalid (both-None / both-Some) states unrepresentable
and removing the per-iteration `Degenerate("missing ... state")` guards. All 8
online_text unit tests pass.
