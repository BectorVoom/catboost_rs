<objective>
Move the O(n·nf) per-level bucket-histogram accumulation INTO the parallel region for the
UNPERTURBED oblivious split search: restructure `GrowScratch` + `select_level_plain` so each
feature's `into_par_iter` task BOTH builds/derives its feature's level histogram AND scores
it (upstream `CalcStatsAndScores` shape), reusing per-task scratch and preserving the
subtraction trick — deleting the separate serial whole-partition build from the per-level hot
path.

Purpose: Phase 21 left the expensive accumulation serial and parallelized only the cheap
scoring pass (Spike 005: serial_fraction≈0.41, Amdahl 16-thread ceiling ≈2.2×, measured
end-to-end only ~1.5–1.9×). Fusing accumulate+score per feature collapses the serial fraction
(Spike 006: 5.0× @16t on the isolated per-level work) and is byte-for-byte identical to the
current path because bins are feature-major and each cell belongs to exactly one feature
(D-03). This is a refactor, not a numerics change — the Plan 01 guard + the ≤10⁻⁵ oracle suite
are the safety net.

Output: fused `select_level_plain` + restructured `GrowScratch` in `crates/cb-train/src/tree.rs`.
</objective>
