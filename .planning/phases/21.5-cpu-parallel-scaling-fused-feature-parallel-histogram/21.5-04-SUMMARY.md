### Out-of-scope discovery (logged, not fixed)
**1. [SCOPE BOUNDARY] Pre-existing stale `monotone` Region-rejection test**
- **Found during:** Task 1 oracle sweep.
- **Issue:** `monotone_oracle_test::monotone_non_symmetric_and_region_are_typed_errors` asserts
  `grow_policy=Region` is *rejected* ("Region OUT", Phase 6.6). Phase 12 built the CPU Region
  grower, so Region is now accepted → the stale assertion fails.
- **Why not a fusion regression:** the fusion touches only `select_level_plain`/
  `select_level_perturbed` (oblivious symmetric). The Region grower is a separate
  non-symmetric dispatch that never routes through the fused pass; `region_e2e_test` and
  `device_region_fit_test` pass. It is a policy-rejection test with no numerical oracle
  fixture — not a parity break.
- **Disposition:** logged to `deferred-items.md`; sweep continued.

## Issues Encountered
- **Disk-pressure landmine (environment, not code):** linking all ~60 cb-train oracle test
  binaries at once (~1.24 GB each in debug, ~71 GB) overflowed the root disk AND the harness
  output-capture tmpfs (both on `/dev/nvme0n1p7`), producing ENOSPC on `cargo test -p cb-train`.
  Worked around by running each already-linked binary DIRECTLY, one at a time, deleting each
  after capturing its result (`target/cbtrain_logs/`), keeping peak disk bounded. Every binary
  that ran is reported faithfully. Documented in EVIDENCE §6. No code impact.
