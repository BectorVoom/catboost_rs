## Decisions Made
- **Single source of truth:** allocating variants (`build_bucket_histogram`, `scan_and_score_borders`) delegate to the `_into` variants; scoring math is `multi_dim_split_score_into` reused verbatim — no split-score arithmetic copied into `histogram.rs`.
- **Buffer reclaim in fused entry:** `fused_feature_scan_and_score` `mem::take`s its histogram Vec into a temporary `BucketHistogram`, scores it, then moves the Vec back into `scratch.data` so the allocation persists across `map_init` reuse (D-07). Disjoint `&mut` field borrows (`data` vs `scan`) via destructuring let the returned `&'s [f64]` (tied to `scan`) coexist with the `*data =` reclaim.
- **Subtraction-trick guard uses integer data:** integer der1 + unit weights make `parent − false == true` bit-exact, mirroring the existing `bucket_histogram_remove_equals_fresh_sibling` convention.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Test builds wrappable histograms via `build_bucket_histogram` instead of directly wrapping `build_bucket_histogram_into` output**
- **Found during:** Task 2 (guard test)
- **Issue:** The plan's Task 2 action says to build the smaller sibling "via `build_bucket_histogram_into`" and then compose with `relocate_sub`/`add_relocated`. Those methods operate on `BucketHistogram`, but its fields are private to the `histogram` module, so an integration test (separate crate) cannot wrap a raw `Vec<f64>` from `build_bucket_histogram_into` into a `BucketHistogram`.
- **Fix:** The test builds the `BucketHistogram` objects for the subtraction trick via `build_bucket_histogram` (which delegates to `build_bucket_histogram_into`, so the `_into` accumulation is still exercised), AND additionally calls `build_bucket_histogram_into` directly and asserts its cells match, via the public `channel()` accessor over the known frozen flat layout. This locks `build_bucket_histogram_into` explicitly while still driving the real `relocate_sub` + `add_relocated` composition. `build_bucket_histogram_into` is also fully exercised inside `fused_feature_scan_and_score` in the level-0 case.
- **Files modified:** crates/cb-compute/tests/fused_feature_parity_test.rs
- **Verification:** `cargo test -p cb-compute --test fused_feature_parity_test` — both cases pass byte-for-byte.
- **Committed in:** 06e4d0d (Task 2 commit)
