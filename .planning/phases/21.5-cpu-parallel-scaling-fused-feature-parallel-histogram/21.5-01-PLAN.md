<objective>
Land the reusable single-feature FUSED accumulate+score primitive in `cb-compute` that
Phase 21.5's `tree.rs` refactor (Plans 02/03) will call inside its per-feature
`into_par_iter().map_init(...)` region, and establish the RED-first byte-identity guard test
that locks the fused path to the current two-pass path BEFORE any production hot-path code
changes.

Purpose: Spike 005/006 proved that feature-outer / object-inner per-feature accumulation is
byte-for-byte identical to the serial whole-partition `build_bucket_histogram` (bins are
feature-major, each histogram cell belongs to exactly one feature), so the accumulation can
move into the parallel region with NO fixed-point and NO oracle re-baseline. This plan
delivers the buffer-reusing primitive (D-07) and the guard (D-05) so the tree.rs fusion is a
pure refactor guarded by both this microtest and the existing ≤10⁻⁵ oracle suite.

Output: `build_bucket_histogram_into`, `scan_and_score_borders_into`, `FusedFeatureScratch`,
and `fused_feature_scan_and_score` in `cb-compute` (exported), plus the guard test file.
</objective>
