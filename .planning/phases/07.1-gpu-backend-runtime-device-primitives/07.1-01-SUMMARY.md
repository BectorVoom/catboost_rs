- Replaced the three inert `()` `SelectedRuntime` placeholders with real CubeCL GPU runtime types, preserving the mutual-exclusion cfg chain verbatim — zero runtime `match` (GPU-02, D-7.1-01).

## Decisions Made
- **Atomic-free default finalize (Open Q1):** the kernel emits one partial per cube and the host folds them with `cb-core::sum_f64`; the in-kernel atomic-finalize variant is Plan 02. This sidesteps the unresolved f64-atomic-on-HIP question and keeps the slice green.
- **`BLOCK_REDUCE_SHMEM` usize const for the SharedMemory size:** the cubecl topology constant `CUBE_DIM` is `u32`, but `SharedMemory::new` wants a comptime `usize` — using `CUBE_DIM` produced `E0308`/`E0277`. A module-level `const BLOCK_REDUCE_SHMEM: usize = 32` is the permitted launch-geometry/shared-mem size; the reduction *stride* still derives from `CUBE_DIM_X`/`PLANE_DIM` so D-09 holds.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] SharedMemory size must be a comptime `usize`, not the `u32` topology `CUBE_DIM`**
- **Found during:** Task 2 (block_reduce_kernel)
- **Issue:** `SharedMemory::<F>::new(CUBE_DIM)` failed to compile — inside `#[cube]`, the `CUBE_DIM` symbol resolves to the cubecl topology constant (`u32`), but `new` takes a comptime `usize` (E0308), cascading into an `E0277 NativeExpand<u32>: Into<usize>` at the `#[cube(launch)]` macro. The plan's pseudocode named `CUBE_DIM` for the size.
- **Fix:** Loaded the CubeCL error-solution guide (`mismatched types.md`) per AGENTS.md, then introduced a module-level `const BLOCK_REDUCE_SHMEM: usize = 32` for the size; the stride remains `CUBE_DIM_X`/`PLANE_DIM`. This honors Pitfall 3 (comptime-const size) and D-09 (no warp-size literal in the stride).
- **Files modified:** crates/cb-backend/src/kernels.rs
- **Verification:** `cargo check --features wgpu` RC=0; rocm/wgpu/cpu oracles pass.
- **Committed in:** 4d51092 (Task 2 commit)

**2. [Rule 3 - Blocking] cpu-hardcoded spike-test mounts broke under non-cpu test builds**
- **Found during:** Task 2 (wgpu reduce test)
- **Issue:** Widening the `kernels` module mount to all backends (Task 1) exposed the existing `#[cfg(test)] mod gradient;` / `mod scatter;` mounts under `--features wgpu` test builds. Those test files hard-code `cubecl::cpu::CpuRuntime`, which does not exist without the `cpu` feature (`E0433: cannot find cpu in cubecl`).
- **Fix:** Gated the `gradient`/`scatter` test mounts to `#[cfg(all(test, feature = "cpu"))]`. The new `reduce` test runs over the generic `SelectedRuntime`, so it stays `#[cfg(test)]` and builds under every backend.
- **Files modified:** crates/cb-backend/src/kernels.rs
- **Verification:** `cargo test --features wgpu kernels::reduce` 3/3; `cargo test --features cpu` still runs gradient/scatter (27 filtered + 3 reduce pass).
- **Committed in:** 4d51092 (Task 2 commit)
