fn run_reduce<F>(input: &[F], use_plane: bool) -> Vec<F>

## Shared Patterns

### Generic device launch (hostŌåödevice transfer)
**Source:** `crates/cb-backend/src/cpu_runtime.rs:46-138` (`launch_binary_f64`).
**Apply to:** `gpu_runtime.rs` and both `_test.rs` harnesses.
**Canonical sequence:** `Device::default()` ŌåÆ `Runtime::client(&device)` ŌåÆ
`client.create(Bytes::from_elems(v.to_vec()))` ŌåÆ `client.empty(num * size_of::<f64>())`
ŌåÆ `kernel::launch::<F, R>(ŌĆ”, ArrayArg::from_raw_parts(handle, len), ŌĆ”)` ŌåÆ
`client.read_one(out)` ŌåÆ `bytemuck::cast_slice::<u8, f64>(&bytes)`. The single
generalization vs the analog: hard-coded `cubecl::cpu::CpuRuntime` becomes
`crate::SelectedRuntime` and its `::Device` association.

### Typed-error read-back (no zero-buffer masquerade, no unwrap)
**Source:** `cpu_runtime.rs:129-137` (WR-05 pattern, repeated at
`:181-189`, `:218-226`, `:284-292`, `:337-345`, `:439-447`).
**Apply to:** every production launch helper in `gpu_runtime.rs`.
```rust
let bytes = client.read_one(out_handle)
    .map_err(|e| CbError::Degenerate(format!("CubeCL device read-back failed: {e:?}")))?;
```
A read-back failure must surface as a typed `CbError`, never a silent all-zero
buffer (which would masquerade as a valid reduction). Production code: no
`unwrap`/`panic`/indexing (workspace lints, `Cargo.toml:10-14`).

### Generics-float `#[cube]` kernel authoring
**Source:** `kernels.rs:1-31` (header + `gradient_kernel`); comptime/length-1-arg
at `kernels.rs:124-153`; if-as-statement at `kernels.rs:85-91` / `:306-312`.
**Apply to:** `block_reduce_kernel`, `block_scan_kernel`.
**Rules:** `<F: Float>` (no hard-coded float), bounds-guard every device read,
if-as-STATEMENT only, no normal-Rust-fn calls inside `#[cube]`, no literal warp
size (use `PLANE_DIM` / `CUBE_DIM_X`; the `SharedMemory::new` SIZE may be the
comptime `CUBE_DIM` const).

### Source/test separation + module mount
**Source:** `kernels.rs:538-548` (mount `mod gradient;` / `mod scatter;` under
`#[cfg(test)]`); test bodies in `kernels/gradient.rs`, `kernels/scatter.rs`.
**Apply to:** mount `mod reduce;` and `mod scan;` in `kernels.rs`; bodies in the
new `_test`-style files. NO `#[cfg(test)] mod tests {}` in any production file
(CLAUDE.md mandatory rule).

### Test-only lint allowance
**Source:** `lib.rs:1` ŌĆö `#![cfg_attr(test, allow(clippy::unwrap_used, ŌĆ”, clippy::indexing_slicing))]`.
**Apply to:** the new test files may use `.unwrap()` / indexing (as
`gradient.rs:46` does); production `gpu_runtime.rs` may not.
