# Phase 7.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Derived from 07.1-RESEARCH.md § "Validation Architecture".

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Rust built-in `#[test]` (workspace standard; `approx 0.5` for float assertions) |
| **Config file** | none (cargo test); **per-crate, NEVER full-workspace** (disk pressure, D-7.1-10) |
| **Quick run command** | `cargo test -p cb-backend --no-default-features --features rocm kernels::reduce` (in-env, gfx1100) |
| **Full suite command** | `cargo test -p cb-backend --no-default-features --features rocm` + `--features wgpu` (host build+run) + `cargo build -p cb-backend --no-default-features --features cuda` (compile-only) |
| **Estimated runtime** | ~30–90 seconds per feature build; rocm oracle runs in seconds |

> **Disk-pressure rule (D-7.1-10 / MEMORY):** NEVER run a full-workspace `cargo test` — the `cb-compute` test profile cannot link under disk pressure. Always scope to `-p cb-backend`. **ROCm GPU tests run locally/manually, NEVER in GitHub Actions** (standing constraint, umbrella D-06).

---

## Sampling Rate

- **After every task commit:** Run `cargo test -p cb-backend --no-default-features --features rocm <touched module>` + `cargo build` for the other three features (`cpu`/`wgpu`/`cuda`).
- **After every plan wave:** All four feature builds + the rocm oracle modules landed so far (`kernels::reduce`; `kernels::scan` once Plan 02 lands).
- **Before `/gsd-verify-work`:** rocm reduce+scan oracles green on gfx1100; wgpu build+run green; cuda build green; divergence reported.
- **Max feedback latency:** ~90 seconds (single-feature build + rocm oracle).
