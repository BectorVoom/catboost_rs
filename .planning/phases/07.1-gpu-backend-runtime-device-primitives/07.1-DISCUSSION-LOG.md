## Runtime cfg-arm wiring (GPU-02/04/05)

| Option | Description | Selected |
|--------|-------------|----------|
| Per-feature cfg arm → cubecl runtime type + gated cubecl feature | Mirror the live `cpu` arm; one `#[cfg]` arm per backend; zero runtime match | ✓ |
| Single arm with runtime backend enum dispatch | A runtime `match` selects backend | (rejected — violates GPU-02 / umbrella D-02) |

**Choice:** Per-feature cfg arm, recommended default. → D-7.1-01/02/03.
**Notes:** Preserves the existing mutual-exclusion `not(...)` cfg chain in `lib.rs:36-46`; cubecl feature flags (`hip`/`wgpu`/`cuda`) gated on the matching cb-backend Cargo feature; cb-compute stays cubecl-free.

## Primitive slice & parity structure (D-03)

| Option | Description | Selected |
|--------|-------------|----------|
| Foundational block scan (incl/excl) + block/plane reduce with atomic finalize | Port the minimal slice downstream kernels need; defer segmented/sort | ✓ |
| Port full `cuda_util` primitive set up front | Includes segmented_scan/segmented_sort/partitions_reduce now | (rejected — speculative, oversized for narrowest-first slice) |

**Choice:** Foundational slice, recommended default. → D-7.1-06/07.
**Notes:** Mirrors `cuda_util/kernel/{scan,reduce}.cu(h)` structure; segmented/sort variants deferred to the histogram/score sub-phases that consume them.
