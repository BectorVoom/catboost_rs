## Phase Goal (MVP — user story)

**As a** developer training CatBoost-rs on a GPU, **I want to** have the model's per-split L2 score and the chosen best split computed device-resident on rocm, **so that** the tree-grow loop can select splits without round-tripping bulk histogram data to the host.

After this plan: a real device computation (L2 score over a device histogram + best-split argmin) runs on rocm AND passes its cross-oracle against the FROZEN CPU reference — a thin working vertical slice, not a horizontal layer.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host driver → device kernel launch | numeric training data (histogram handle, lengths, n_bins/n_features) crosses into device launch geometry; no external/untrusted/network input |
| device → host read-back | only the O(blocks) BestSplit[] descriptor crosses back |

This is a pure in-process GPU compute path in `cb-backend` — no network, no auth, no attacker-controlled serialization. The realistic surface is memory-safety / device-buffer index correctness.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07.5-01-01 | Tampering | device shared-mem / score buffer OOB write | mitigate | comptime size guards coupling launch geometry to `SharedMemory` allocation (gpu_runtime.rs `const _: () = assert!` precedent ~109-137); extend for the score/argmin buffers |
| T-07.5-01-02 | Tampering | integer overflow in (feature*n_bins+bin) candidate index math | mitigate | `checked_mul` on the candidate/leaf index products → `CbError::Degenerate` on overflow (clone launch_pointwise_hist2_into guard ~1023-1037) |
