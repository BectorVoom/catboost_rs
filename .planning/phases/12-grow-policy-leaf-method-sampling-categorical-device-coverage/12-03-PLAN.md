<threat_model>
## Trust Boundaries

Numerical GPU kernel work only — NO network, auth, session, upload, or untrusted-input surface. The one boundary is the host↔device seam (internally-generated quantized bins/partition offsets/node diffs cross into `#[cube]` kernels).

| Boundary | Description |
|----------|-------------|
| host→device (seam) | Quantized bins + partition state cross into the non-sym grow kernel; node-diff encoding uses checked `u16` conversion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05 | Tampering/DoS | node-diff / leaf-id integer sizing (`u16`/`u32::MAX` sentinel) | mitigate | checked `u16::try_from` child diffs; `u32::MAX` interior sentinel (CR-01/CR-02); `check_depth`. |
| T-12-02 | Tampering (integrity) | device score/histogram SUM | mitigate | fixed-point `Atomic<u64>` k=30 deterministic reduction (holds ε=1e-4). |
| T-12-06 | DoS | `-inf` JIT-reject on HIP → kernel fails to build | mitigate | finite `f32::MIN`/`f32::MAX` sentinel in `#[cube]`; ROCm smoke build gate. |
| T-12-SC | Tampering | npm/pip/cargo installs | accept | N-A — no external packages (RESEARCH Package Legitimacy Audit). |

Load-bearing gate is the ε=1e-4 device-vs-CPU oracle, NOT a security boundary.
