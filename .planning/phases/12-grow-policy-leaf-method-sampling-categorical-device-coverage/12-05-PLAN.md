<objective>
W3: Exact weighted-quantile leaf-value estimation on device (GPUT-19), for Quantile/MAE/MAPE-family objectives — distinct from the Newton der2 path (GPUT-07, Phase 11). Upstream `EstimateExact` is fully GPU: `SegmentedRadixSort` → `SegmentedScanVector` → `ComputeWeightedQuantileWithBinarySearch` (§6.3/§5.6). The device reproduces `cb-compute/src/leaf.rs::exact_leaf_delta`: `weightsWithTargets[i] = weights[i]/max(1,|target[i]|)`, `needWeights = totalWeight·α`, per-leaf-bin sort of targets by weight, weight prefix sum, binary search for the doc whose cumulative weight first reaches `needWeights`, then the α/δ adjustment.

Purpose: cover the quantile-loss leaf-estimation family on device, reusing Phase-10 segmented primitives; add the segmented-sort primitive both Exact and MVS need (Open Q1 / A1 — `sort.rs` currently exposes only whole-buffer `run_radix_sort`).
Output: segmented-sort primitive + `exact_quantile` kernel + self-oracle + exact-leaf gate arm.
</objective>
