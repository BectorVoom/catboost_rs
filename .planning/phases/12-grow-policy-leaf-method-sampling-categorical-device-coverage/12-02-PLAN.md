<objective>
Build the CPU Region grow-policy path FIRST (D-03a) — the single largest lift in Phase 12 and a v1.0-escalated gap ("Region OUT"). Region has NO existing CPU reference: `validate_grow_policy` (boosting.rs:1331) rejects `EGrowPolicy::Region` and `TreeVariant` has no Region variant. This plan lifts the rejection, adds a Region grower, a new `TreeVariant::Region(RegionTree)` **path** model (distinct from the non-symmetric node graph), the walk-until-diverge apply, and the json round-trip — establishing the ≤1e-5 CPU oracle that Plan 04's device Region kernel checks against.

Purpose: Pitfall 1 — the device ε=1e-4 gate is uncheckable without a CPU Region oracle; this plan creates it. Region is upstream's `TRegionModel` (`AddRegionImpl`, §6.6): an oblivious-like path walked while the computed split matches the stored direction; leaf = depth reached at divergence; `MaxLeaves = MaxDepth+1`.
Output: a trainable+serializable+applicable Region model with a frozen ≤1e-5 oracle fixture.
</objective>
