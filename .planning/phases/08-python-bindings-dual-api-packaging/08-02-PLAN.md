<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python kwargs ‚Üí param registry | Untrusted kwarg names/values cross into the validator at fit() |
| Facade error ‚Üí Python exception | Internal typed error crosses out to the catchable Python layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Tampering (correctness) | unsupported param silently ignored ‚Üí silently wrong model | mitigate | D-05/D-07: `validate_params` rejects KnownNotYet/Unknown params with a typed `CatBoostParameterError` at fit() ‚Äî no silent acceptance |
| T-08-06 | Information disclosure | error messages leaking internal facade detail | accept | Facade messages are already user-facing typed strings (no secrets in a library); actionable detail is the goal (PYAPI-05) |
| T-08-07 | DoS | panic in Levenshtein/registry indexing | mitigate | `[lints] workspace=true` deny gate; registry is a static slice iterated without indexing; closest-match uses checked iteration |
</threat_model>
