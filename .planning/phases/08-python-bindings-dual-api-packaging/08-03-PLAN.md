<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python framework object → ingest adapter | Untrusted Pandas/Arrow/Polars buffers (arbitrary dtype/nullability/strides) cross into Rust |
| Borrowed buffer → owned columns → compute | The GIL-release boundary; borrows must not survive it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-08 | Tampering / DoS | use-after-free of a borrowed buffer across `py.detach()` (free-threaded) | mitigate | D-11: every adapter COPIES into `OwnedColumns` before returning; the compute closure receives only owned data — asserted by `ingest_py_test.rs` |
| T-08-09 | Tampering (correctness) | silent float64→float32 coercion changing oracle results | mitigate | D-12: reject float64 with a `CatBoostValueError` ("pass float32"); never coerce silently |
| T-08-10 | Tampering | nullable Arrow / ambiguous object column misread as features | mitigate | Validate null_count==0 and require `cat_features` for non-numeric columns; else `CatBoostValueError` |
| T-08-11 | Tampering | out-of-bounds from a length-mismatched Pool column | mitigate | Reuse `OwnedColumns::into_pool()` length checks (`CbError::LengthMismatch`); never re-implement or index unchecked |
</threat_model>
