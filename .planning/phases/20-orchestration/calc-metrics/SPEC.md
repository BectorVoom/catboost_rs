# ORCH-04 — Standalone `eval_metric` / `calc_metric` Surface

## 1. Context

`catboost_rs` is a Rust rewrite of CatBoost, oracle-tested ≤10⁻⁵ against the
original C++ library. Its metric arithmetic already exists and is oracle-locked,
but it is reachable **only through the training loop** as
`cb_train::metrics::EvalMetric`, consumed per-iteration inside
`train_with_eval_sets` `[VERIFIED: CODEGRAPH crates/cb-train/src/metrics.rs:64;
crates/cb-train/tests/eval_metrics_oracle_test.rs]`. There is **no standalone
public surface** that computes a metric on caller-supplied predictions — the
upstream `catboost.utils.eval_metric(label, approx, metric, weight=, group_id=)`
free function `[VERIFIED: LOCAL crates/cb-oracle/generator/gen_ranking_fixtures.py:322
`from catboost.utils import eval_metric`]`.

This slice adds that standalone surface. It is the highest-value **genuinely
missing** feature that is simultaneously self-contained and backed by the
cleanest possible oracle: metrics on **fixed** predictions have no training or
quantization nondeterminism (the documented cross-cutting hazard —
`[VERIFIED: LOCAL MEMORY ctr-model-loading "quantization is run-to-run
nondeterministic so CTR fixtures are frozen"]`). The ranking half of the oracle
already exists: `ranking_corpus/ranking_metrics/*.npy` are generated by calling
`catboost.utils.eval_metric` on a FIXED `approx`
`[VERIFIED: LOCAL crates/cb-oracle/fixtures/ranking_corpus/ranking_metrics/;
gen_ranking_fixtures.py:320-351]`.

**Crate placement (user decision, locked):** a standalone `calc_metrics` module
in `cb-train`, re-exported and surfaced through the `catboost-rs` facade and the
Python bindings. No new `cb-orchestrate` crate for this slice
`[VERIFIED: LOCAL user AskUserQuestion answer 2026-07-18]`.

## 2. Scope and Non-Goals

### In scope

- A **metric-descriptor parser** `&str -> EvalMetric` covering the metric set
  already implemented in `EvalMetric` (with upstream param syntax).
- A **standalone final-value metric surface** in `cb-train`:
  compute a metric's `GetFinalError` value on caller-supplied
  `(label, approx, weight, group_id)`, for both flat and grouped (ranking)
  metrics, oracle-locked ≤10⁻⁵.
- A **multi-metric dispatch** that accepts a list of descriptor strings and
  returns one value per descriptor.
- **Rust facade** free functions `eval_metric` / `eval_metrics` on `catboost-rs`.
- **Python** `catboost_rs.utils.eval_metric` mirroring the upstream signature.
- **New oracle fixtures** for the flat metrics on FIXED predictions
  (`calc_metrics/` family); reuse of the existing ranking fixtures.

### Non-goals (explicit — documented, not silently dropped)

- **Staged `CatBoost.eval_metrics(data, metrics, ntree_start/end, eval_period)`**
  over tree prefixes. It couples to a model + `Pool` + staged prediction (a
  different oracle: `model.eval_metrics(...)`), and would break the failure
  isolation of this slice. Tree-prefix machinery partially exists
  (`apply.rs` `tree_start`/`tree_end` `[VERIFIED: CODEGRAPH crates/cb-model/src/apply.rs:594-613,712]`);
  staged eval is a **follow-on slice** (ORCH-04b), not this one.
- **Multi-dimensional / multiclass approx** metrics (label+approx are 1-D here).
  The in-scope metric set is single-dimensional `[INFERRED: EvalMetric flat
  variants are 1-D over `&[f64]` approx — metrics.rs:244]`.
- **Metrics not yet present in `EvalMetric`** (e.g. AUC binary non-grouped,
  Accuracy, F1, R2, MAE-as-metric, Precision/Recall non-grouped). Selecting one
  is a typed error; the spec documents the deferred set. Extending `EvalMetric`
  itself is a separate slice.
- **CTR / categorical / text models** — the surface consumes predictions, not a
  model; it is model-kind-agnostic and never applies a model.
- **GPU** — pure CPU arithmetic.

## 4. Typed Contracts

All new Rust code lives in a new module `crates/cb-train/src/calc_metrics.rs`
(prod) with tests in `crates/cb-train/src/calc_metrics_test.rs`, mounted via the
project pattern `#[cfg(test)] #[path = "calc_metrics_test.rs"] mod tests;`
`[VERIFIED: LOCAL MEMORY fstr03-plan "test mount pattern"]`.

```rust
// crates/cb-train/src/calc_metrics.rs

/// Parse a CatBoost metric-descriptor string into an `EvalMetric`.
///
/// Grammar (case-insensitive metric name; `:key=value` params in any order):
///   "RMSE" | "Logloss" | "MSLE"
///   "NDCG[:top=<i64>][:type=Base|Exp][:denominator=LogPosition|Position]"
///   "DCG[:top=<i64>][:type=Base|Exp][:denominator=LogPosition|Position]"
///   "MAP[:top=<i64>][:border=<f64>]"
///   "MRR[:top=<i64>][:border=<f64>]"
///   "ERR[:top=<i64>]"
///   "PFound[:top=<i64>][:decay=<f64>]"
///   "PrecisionAt[:top=<i64>][:border=<f64>]"
///   "RecallAt[:top=<i64>][:border=<f64>]"
///   "QueryAUC[:type=Classic|Ranking]"
/// Omitted params take the upstream defaults already encoded in `EvalMetric`
/// (top=-1, border=0.5, decay=0.85, dcg_type=Base, denominator=LogPosition,
/// auc_type=Classic).
///
/// # Errors
/// `CbError::Degenerate` on an unknown metric name, an unknown/duplicate param
/// key, or an unparseable param value. Never panics (deny-lints).
pub fn parse_metric(descr: &str) -> CbResult<EvalMetric>;

/// Compute one metric's FINAL value on fixed predictions (the standalone
/// `catboost.utils.eval_metric` surface).
///
/// - Flat metrics (RMSE/Logloss/MSLE/Custom) route through `EvalMetric::eval`;
///   `group_id` is ignored.
/// - Ranking metrics route through `EvalMetric::eval_grouped`; an empty
///   `group_id` is treated as one group.
/// `weight` empty => uniform 1.0. `approx[i]` is the RAW model output
/// (RawFormulaVal; Logloss applies sigmoid internally).
///
/// # Errors
/// `CbError::Degenerate` on length mismatch, empty eval set, non-positive total
/// weight, or non-contiguous `group_id` (delegated to the underlying seam).
pub fn calc_metric(
    metric: &EvalMetric,
    label: &[f64],
    approx: &[f64],
    weight: &[f64],
    group_id: &[u64],
) -> CbResult<f64>;

/// Multi-metric convenience: parse each descriptor and evaluate it. Returns one
/// `f64` per descriptor in input order.
///
/// # Errors
/// The first parse or evaluation error (typed `CbError`), never a panic.
pub fn eval_metric(
    label: &[f64],
    approx: &[f64],
    metrics: &[&str],
    weight: &[f64],
    group_id: &[u64],
) -> CbResult<Vec<f64>>;
```

```rust
// crates/catboost-rs/src/lib.rs (or a new metrics.rs module, re-exported)
```

## 5. Failure-Isolated Behavioral Specifications

Each spec has ONE primary reason a failing acceptance test would fail.

### ORCH-04-S1 — Metric-descriptor parser
- **Status:** implemented
- **Responsibility:** `&str -> EvalMetric` (name + params), nothing else.
- **Input:** `descr: &str` (arbitrary). **Output:** `CbResult<EvalMetric>`.
- **Dependencies:** `EvalMetric` variants + param enums (`DcgMetricType`,
  `DcgDenominator`, `AucType`).
- **Behavior:**
  - Given `"RMSE"` / `"logloss"` / `"MSLE"` (any case) → the matching flat
    variant.
  - Given `"NDCG:top=2:type=Exp:denominator=Position"` → `Ndcg { top:2,
    dcg_type:Exp, denominator:Position }`.
  - Given `"NDCG"` → `Ndcg { top:-1, dcg_type:Base, denominator:LogPosition }`
    (upstream defaults) `[VERIFIED: CODEGRAPH metrics.rs:76-95]`.
  - Given `"QueryAUC:type=Ranking"` → `QueryAuc { auc_type: Ranking }`.
  - Given `"NoSuchMetric"` or `"NDCG:bogus=1"` → `Err(CbError::Degenerate)`.
- **Invariants:** total function; no panic; unknown → typed error.
- **Out of scope:** metrics absent from `EvalMetric` (documented deferred set).
- **Traceability:** `metrics.rs:64-151`; `ranking_metrics.rs` param enums.

### ORCH-04-S2 — Flat final-value evaluation (RMSE / Logloss / MSLE)
- **Status:** implemented
- **Responsibility:** final metric value on fixed `(label, approx, weight)` for a
  flat metric; delegates the arithmetic to `EvalMetric::eval`.
- **Input:** `metric: &EvalMetric` (flat), `label/approx/weight: &[f64]`.
  **Output:** `CbResult<f64>`.
- **Behavior (Given/When/Then):**
  - Given fixed `label`, `approx`, and metric `RMSE`, When `calc_metric` is
    called with empty `weight`/`group_id`, Then the result equals
    `catboost.utils.eval_metric(label, approx, "RMSE")` within ≤1e-5.
  - Same for `Logloss` (approx = raw logit; sigmoid applied once internally) and
    `MSLE`.
  - Given a non-empty `weight`, Then the weighted value matches
    `eval_metric(..., weight=w)` ≤1e-5.
- **Invariants:** routes sums through `cb_core::sum_f64`; degenerate eval set →
  typed error, never NaN to the caller.
- **Acceptance test:** `calc_metrics_flat_oracle_test.rs` over new `calc_metrics/`
  fixtures.
- **Traceability:** `EvalMetric::eval` `[VERIFIED: CODEGRAPH metrics.rs:244-371]`.

### ORCH-04-S3 — Grouped ranking final-value evaluation
- **Status:** implemented
- **Responsibility:** final metric value on fixed `(label, approx, group_id,
  weight)` for a ranking metric; delegates to `EvalMetric::eval_grouped`.
- **Input:** `metric: &EvalMetric` (ranking), `label/approx: &[f64]`,
  `weight: &[f64]`, `group_id: &[u64]`. **Output:** `CbResult<f64>`.
- **Behavior:**
  - For each of NDCG, DCG, MAP, MRR, ERR, PFound, PrecisionAt, RecallAt
    (defaults + `top=2`) and QueryAUC (Ranking + Classic), Given the committed
    `ranking_corpus/ranking_metrics` `approx/target/group_id`, When `calc_metric`
    is called, Then the result matches the committed
    `<scenario>.npy` (itself `catboost.utils.eval_metric(...)`) within ≤1e-5.
  - Given an empty `group_id`, Then the whole set is one group (upstream
    single-group behavior).
  - Given a non-contiguous `group_id`, Then `Err(CbError::Degenerate)`.
- **Acceptance test:** `calc_metrics_ranking_oracle_test.rs` reusing the existing
  ranking fixtures — NO new fixtures needed.
- **Traceability:** `EvalMetric::eval_grouped` `[VERIFIED: CODEGRAPH
  metrics.rs:397-484]`; fixtures `[VERIFIED: LOCAL
  fixtures/ranking_corpus/ranking_metrics/]`.

### ORCH-04-S4 — Dispatch + validation (`eval_metric` list form)
- **Status:** implemented
- **Responsibility:** parse-then-evaluate a list of descriptors; route flat vs
  grouped correctly; surface all misuse as typed errors.
- **Input:** `label/approx: &[f64]`, `metrics: &[&str]`, `weight: &[f64]`,
  `group_id: &[u64]`. **Output:** `CbResult<Vec<f64>>` (one per metric, in order).
- **Behavior:**
  - Given `["RMSE","Logloss"]`, Then a 2-element vector matching each metric.
  - Given a ranking metric with empty `group_id`, Then single-group evaluation
    (no error).
  - Given `label.len() != approx.len()`, Then `Err(CbError::Degenerate)` (no
    panic) `[VERIFIED: CODEGRAPH metrics.rs:245-249]`.
  - Given an unknown metric string, Then `Err(CbError::Degenerate)` from S1.
- **Acceptance test:** unit tests in `calc_metrics_test.rs` (dispatch + each
  error path) + hand-computed small-vector value checks.
- **Traceability:** S1 + S2 + S3.

### ORCH-04-S5 — Rust facade `eval_metric` / `eval_metrics`
- **Status:** implemented
- **Responsibility:** ergonomic facade over S4 with `Option` weight/group and
  `CatBoostError` mapping; single-metric returns a scalar.
- **Input:** `label/approx: &[f64]`, `metric: &str` (or `&[&str]`),
  `weight: Option<&[f64]>`, `group_id: Option<&[u64]>`.
  **Output:** `Result<f64, CatBoostError>` (or `Result<Vec<f64>, CatBoostError>`).
- **Behavior:**
  - `eval_metric(label, approx, "RMSE", None, None)` == the S2 RMSE value.
  - `None` weight/group map to empty slices.
  - Underlying `CbError` → `CatBoostError` (no `unwrap`/`panic`).
- **Acceptance test:** facade test in `catboost-rs` (`tests/` or mounted
  `*_test.rs`) asserting parity with a known value and error propagation.
- **Traceability:** `feature_importance_with_data` mapping precedent
  `[VERIFIED: CODEGRAPH crates/catboost-rs/src/model.rs:169-193]`.

### ORCH-04-S6 — Python `catboost_rs.utils.eval_metric`
- **Status:** implemented
- **Responsibility:** PyO3 surface mirroring `catboost.utils.eval_metric`.
- **Input:** `label`, `approx` (sequences/np arrays → `Vec<f64>`), `metric`
  (`str` or `list[str]`), `weight=None`, `group_id=None`.
  **Output:** `float` for one metric, `list[float]` for a list.
- **Behavior:**
  - `utils.eval_metric(y, a, "RMSE")` → a float within ≤1e-5 of
    `catboost.utils.eval_metric(y, a, "RMSE")`.
  - `utils.eval_metric(y, a, ["RMSE","MSLE"])` → a 2-list.
  - A bad metric string raises a Python exception (mapped from `CatBoostError`),
    not a panic/abort.
- **Acceptance test:** Python parity test under the uv 3.12 venv (mirrors the
  FSTR-03 Python precedent) `[VERIFIED: LOCAL MEMORY fstr03-partial-dependence-plan]`.
- **Traceability:** `catboost-rs-py` binding pattern `[VERIFIED: LOCAL
  crates/catboost-rs-py/src/estimator.rs]`.
