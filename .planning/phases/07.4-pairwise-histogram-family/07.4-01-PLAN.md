**As a** Rust developer embedding the GPU pairwise/ranking training path (and the downstream 7.5 pairwise score/split consumer), **I want to** fill a device-resident 4-channel pairwise histogram for 5/6/7-bit non-binary features on rocm directly from the on-device pair/weight/bin handles, **so that** the pairwise split scorer can pick splits with no host round-trip and verifiable parity against the CPU pairwise path.
