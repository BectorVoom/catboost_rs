## Trust Boundaries

This is a local measurement/documentation phase: NO network surface, NO untrusted
input, NO new production code path (test harness only, run manually in-env). The
realistic boundary is **measurement integrity** — the risk that a number reported
as "GPU validation" was not actually produced on the GPU.

| Boundary | Description |
|----------|-------------|
| rocm device ↔ host (read-back) | Device-resident results read back to host for comparison against the Rust CPU baseline; a wrong feature-gate could make this CPU-vs-CPU |
| Cargo feature resolution ↔ SelectedRuntime | `cb-backend` resolves `SelectedRuntime` from features; an accidental `cpu` activation (e.g. via a `cb-train` dep) silently swaps the device |
