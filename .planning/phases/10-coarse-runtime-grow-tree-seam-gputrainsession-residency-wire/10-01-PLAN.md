<objective>
Add the GPUT-01 grow-tree trait seam to the `Runtime` trait in `cb-compute`: three new
methods (`begin_device_training`, `grow_tree_on_device`, `end_device_training`) plus the
host-typed `DeviceGrownTree` return struct. All three carry **default implementations**
returning `Ok(false)` / `Ok(None)` / `Ok(())` so every existing impl (`CpuBackend`, test
runtimes) compiles unchanged and transparently falls back to the CPU grower — preserving
D-04 (CPU byte-unchanged) for free. The trait stays CubeCL-free; only host types
(`Vec`/slices/`usize`/`f64`/`Loss`/`EScoreFunction`) cross the boundary.

Purpose: Establish the seam contract that 10-02 (cb-train wiring) and 10-03 (GpuBackend
session override) both depend on, without touching any backend or kernel code yet.
Output: Extended `Runtime` trait + `DeviceGrownTree` + seam unit test.
</objective>

- New struct `DeviceGrownTree` (in `cb-compute::runtime`, re-exported from `cb-compute`):
  fields `splits: Vec<(u32, u32)>` (per-level `(feature_index, bin_id)`),
  `leaf_values: Vec<f64>` (length `2^depth`, learning-rate-UNscaled per the CPU `GrownTree`

  - `grow_tree_on_device(&self, approx: &[f64], target: &[f64]) -> CbResult<Option<DeviceGrownTree>>` (default `Ok(None)`)
  - `end_device_training(&self) -> CbResult<()>` (default `Ok(())`)
- New test file `crates/cb-compute/src/runtime_test.rs` + its `#[cfg(test)] mod runtime_test;` include.
