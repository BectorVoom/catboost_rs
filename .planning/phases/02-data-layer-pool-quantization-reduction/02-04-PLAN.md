|----------|-------------|
| category string bytes → ui32 hash | a wrong CityHash variant silently produces different bins → categorical parity fails |
| attacker-controlled cardinality → perfect-hash map size | unbounded uniq count could overflow the bin index / exhaust memory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Tampering | CityHash64 variant mismatch (generic crate vs vendored) | mitigate | Port vendored `util/digest/city.cpp` exactly; bit-exact `assert_eq!` vectors including a >16-byte tail case |
| T-02-11 | DoS / Tampering | integer overflow on uniq cat count / bin index | mitigate | Bound uniq count to `u32::MAX` (MAX_UNIQ_CAT_VALUES); return CbResult error on overflow; no panic |
| T-02-12 | Tampering | NaN/null smuggled into a categorical column | mitigate | Categorical ingestion rejects NaN/null (enforced at the ingestion boundary, Plan 05); cat_hash operates on validated bytes only |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new package installs — CityHash64 is ported, not crate-sourced (RESEARCH Pitfall 4) |
