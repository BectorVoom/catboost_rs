<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed fixture → Rust reader | `.npy`/`config.json` parsed by `cb-oracle`; repo-controlled, but the read path must fail gracefully (no panic) on malformed data |
| CI runner → repo | CI executes only committed code + committed fixtures; no network at test time, no generator run |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | malformed/wrong-dtype `.npy` fixture | mitigate | `ndarray-npy::read_npy` returns `Err` (not panic) on bad header/dtype; `load_f64_vec` propagates via `OracleError`; `indexing_slicing`/`panic` lints deny accidental panics in the read/compare path |
| T-01-02 | Denial of Service | panic-on-overflow / index-panic in comparator | mitigate | comparator uses `.iter().zip().enumerate()` (no indexing) and returns `Result`; `clippy::panic` + `indexing_slicing` denied workspace-wide |
| T-01-SC | Tampering | crate/package supply chain (`thiserror`/`ndarray`/`ndarray-npy`/`serde`) | mitigate | all deps are canonical official-org crates (RESEARCH Package Legitimacy Audit, all OK); pin exact latest versions in `[workspace.dependencies]`; commit `Cargo.lock`. No `[ASSUMED]`/`[SUS]` packages — no blocking human checkpoint required this plan |
| T-01-03 | Tampering | `anyhow` leaking into core-lib non-test code | mitigate | structural ban (not in `[dependencies]`) + `scripts/check-no-anyhow.sh` grep backstop in CI (D-14) |
</threat_model>
