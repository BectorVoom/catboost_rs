## Gaps Summary

This phase achieved 4 of 9 observable truths. Two correctness bugs (CR-01, CR-02) found in code review remain unresolved in the codebase; they are masked by the uniform-weight oracle fixtures but will produce incorrect results for any non-uniform-weight training scenario. Four end-to-end per-stage oracle gates are missing or incomplete:

1. **PairLogit/PairLogitPairwise oracle tests never authored** (test files missing). The machinery and frozen fixtures exist; only the pair-weight normalization fix + the two oracle test files are missing.

2. **YetiRank/YetiRankPairwise/StochasticRank per-stage gates deferred**: the oracle tests currently pass on an absent-fixture invariant, not on actual ≤1e-5 parity. The end-to-end closure requires an instrumented catboost 1.2.10 trainer build that was infeasible at author time (toolchain absent, disk 95-97%). This is documented and the closure recipe exists; no code is wrong here, but the goal is not met.

3. **CR-02 double-weighting** in boosting.rs affects QueryRMSE and QuerySoftMax correctness for weighted training; the fix is a one-line branch on group_spans.

4. **CR-01 scale mismatch** in ranking_der.rs affects StochasticRank NDCG gradient for groups where ideal_dcg != 1.0; the fix is to pass pos_weights into calc_dcg_metric_diff.

5. **D-08 violations** (WR-01/WR-02) in make_zero_average and lambdamart_ideal_ndcg use raw float accumulation where cb_core::sum_f64 is required.

**LOSS-04 remains incomplete.** Three of six ranking losses (QueryRMSE, QuerySoftMax, LambdaMart) are oracle-locked ≤1e-5. PairLogit, PairLogitPairwise, YetiRank, YetiRankPairwise, and StochasticRank do not yet have passing end-to-end per-stage ≤1e-5 gates.

**LOSS-05 is complete.** All nine ranking metrics are oracle-locked ≤1e-5.
