<objective>
Close the two ranking_der.rs gaps from 06.3-VERIFICATION.md / 06.3-REVIEW.md that both live in
`crates/cb-compute/src/ranking_der.rs`: the **CR-01 BLOCKER** (StochasticRank NDCG gradient scale
mismatch in `calc_dcg_metric_diff`) and the **WR-02 WARNING** (D-08 raw-accumulation violation in
`lambdamart_ideal_ndcg`). Both are bundled here because they share a file (exclusive file ownership
for parallel execution) and are the only edits this plan makes.

CR-01: `calc_dcg_metric_diff` recomputes the raw unnormalized `1.0/ndcg_denominator(pos)` for
`old_weight`/`new_weight`, while the `cum_sum`/`cum_sum_up`/`cum_sum_low` prefix arrays were built
from `pos_weights` already normalized by `1/ideal_dcg` (compute_dcg_pos_weights). `doc_diff` and
`mid_diff` are therefore on mismatched scales for any group where `ideal_dcg != 1.0`. Fix per
06.3-REVIEW.md CR-01: thread the normalized `pos_weights` slice into `calc_dcg_metric_diff` and read
`pos_weights[old_pos]`/`pos_weights[new_pos]` directly (mirroring upstream `CalcDCGMetricDiff`
`posWeights[oldPos]`).

WR-02: replace the raw `score += ...` loop in `lambdamart_ideal_ndcg` with a collected-terms
`cb_core::sum_f64(&terms)` per the D-08 summation discipline.

Purpose: Restore correctness of the StochasticRank NDCG gradient for non-trivial graded-relevance
groups and bring the LambdaMart ideal-DCG accumulation into D-08 compliance.
Output: A `pos_weights`-parameterized `calc_dcg_metric_diff`, a `sum_f64`-based
`lambdamart_ideal_ndcg`, and a graded-relevance regression test that fails on the old code.

Deferral note (for verify-phase): this plan does NOT close the deferred YetiRank/YetiRankPairwise/
StochasticRank end-to-end TRAINER fixtures (truth #5) nor SC-3/D-07 instrumented-harness validation
(truth #7) — those remain documented gaps requiring the deferred instrumented catboost build.
LOSS-04 stays partially open after this gap-closure round.
