# Phase 6.3 Plan 18: StochasticRank End-to-End Closure (gap #3 / D-07) Summary

StochasticRank end-to-end per-stage oracle closes at <=1e-5 (Splits|LeafValues|StagedApprox|Predictions) by fixing two trainer-isolated bugs — per-tree noise seeding and per-query approx centering — making LOSS-04 fully satisfied (gaps #1/#2/#3 all closed).

## What shipped

- **Per-tree StochasticRank noise seeder** (`boosting.rs`): drives the grouped der off the same per-tree context-RNG advance as YetiRank (`YetiRankTreeSeeder`). The gradient/split der uses the per-tree DERIVATIVE recalc base (`recalc_seeds[0]`); a NEW leaf-value der re-compute uses the distinct LEAF-VALUE recalc base (`recalc_seeds[2]`). The per-group noise seed is `base + group_index` (`error_functions.h:1257`). Replaces the prior fixed-`params.random_seed`-every-tree path.
- **Per-query approx centering** (`ranking_der.rs` `stochastic_rank_group_der`): group-mean-centers the per-query approx at entry, matching catboost's group-mean-centered `bt.Approx` input (the AveragingFold approx is maintained zero-mean per query). This was the end-to-end residual root cause.
- **`Log2` parity** (`ranking_der.rs` `ndcg_denominator`): `log(x) * M_LN2_INV` (catboost `util/generic/ymath.h`), not the `std::log2` intrinsic.
- **D-07 per-tree noise oracle** (`stochasticrank_pertree_noise_oracle`): the Rust per-tree per-group `std_normal` stream reproduces `stochasticrank_pertree_noise_groundtruth.jsonl` (110 events / 40 streams / 10 base recalc seeds across 5 trees) bit-exact.
- **End-to-end gate activated** (`stochasticrank_end_to_end_per_stage`): the deferred present-fixture panic guard replaced with the full 4-stage `compare_stage` <=1e-5; all three StochasticRank gates (standalone draw-log, D-07 per-tree noise, end-to-end) pass.

## Root cause analysis (how the residual was found)

Tree 0 leaf values matched to 1e-9, splits matched fully, the D-07 noise oracle passed bit-exact — yet end-to-end LeafValues diverged 5e-5 from tree 1 on (count>2) groups. Static analysis confirmed the der formula line-by-line faithful (CalcDCGMetricDiff, CalcDCGCumulativeStatistics, density branches, SFA projection, nu/lambda/sigma/mu defaults, numerator/denominator types). The divergence was isolated by an **incremental rebuild** of the warm instrumented catboost 1.2.10 `_catboost.so` (one TU, `/tmp/cb_build313` + clang-18, loaded under the project `.venv` python3.13) with a new `srank_rawder` per-doc hook (approx/score/mean/derSum at %.17g). The `srank_rawder` fences proved catboost's `approxes[docId]` is GROUP-MEAN-CENTERED; feeding the centered approx to the Rust der reproduced catboost's raw der bit-for-bit, and the centered der through the projection gave the exact leaf value. The SFA projection's ~1/0.0036 amplification turned the per-query-mean offset into the observed 5e-5 leaf divergence.
