# Phase 06.3 Plan 05: Ranking Metrics (LOSS-05) Summary

**The nine ranking metrics — NDCG, DCG, MAP, MRR, ERR, PFound, PrecisionAt, RecallAt, QueryAUC — land as eval-only quantities on a widened `EvalMetric::eval_grouped` group seam, each transcribed per-group from catboost 1.2.10 `libs/metrics/` with a shared `compare_docs` tie-break, and oracle-locked ≤1e-5 over a FIXED approx (default + explicit top=2).**

## Accomplishments

- **Widened group seam (D-6.3-05):** `EvalMetric::eval_grouped(approx, target, weights, group_id, subgroup_id)` builds per-group `[begin,end)` spans from `group_id` (contiguous-unique run detection mirroring `build_query_info` / upstream `GroupSamples`), computes the per-group metric, and averages over groups. The flat `EvalMetric::eval` path (RMSE/Logloss/MSLE) is **byte-identical** (D-04) — ranking variants are routed away with a typed error.
- **9 ranking `EvalMetric` variants** with upstream defaults pinned from source: `top=-1` (full group), `decay=0.85` (PFound), `border=0.5` (`GetDefaultTargetBorder`, `data_processing_options.h:58`), `denominator=LogPosition` + `dcg_type=Base` (NDCG/DCG). `Eq` was dropped from `EvalMetric` (the ranking variants carry `f64` params).
- **`ranking_metrics.rs`** transcribes every per-group formula verbatim with `file:line`: DCG/NDCG (`dcg.cpp`), PFound (`pfound.h`), ERR/MRR (`metric.cpp:6035-6164`), Precision/Recall/MAP (`precision_recall_at_k.cpp`), QueryAUC Classic/Ranking (`auc.cpp`). The shared `compare_docs` tie-break (predicted desc → target asc → stable index, `doc_comparator.h:4-6`) is transcribed **once** and every approx-sorted metric routes through it.
- **Correct upstream edge semantics:** NDCG `IDCG<=0 → 1` (not 0, `dcg.cpp:147`); RecallAt `total_relevant==0 → 1`; MAP/DCG/NDCG/PFound/ERR/MRR `empty → 0`; the group-weighted vs uniform averaging split matches upstream's `Stats[1] += queryWeight` vs `Stats[1]++`. All reductions via `cb_core::sum_f64`; empty group / zero-relevant / IDCG==0 never divide (Security V5).
- **Per-metric ≤1e-5 oracle:** `gen_ranking_fixtures.py --metrics-eval` freezes upstream `catboost.utils.eval_metric` values over a FIXED known approx; `ranking_metrics_oracle_test.rs` gates 18 cases (every metric at default params + an explicit `top=2` for each @k metric exercising the tie/nth_element path + QueryAUC in both singleclass modes) via `compare_stage(Stage::Predictions, …)`.

## Decisions Made

- **Sibling `eval_grouped` (D-04):** the ranking metrics get a dedicated grouped entry point; the flat `eval` signature is untouched and the RMSE/Logloss/MSLE arithmetic is unchanged, so the shipped eval-metric + MSLE oracles stay green. Misuse in either direction (ranking metric through `eval`, or non-ranking through `eval_grouped`) is a typed `CbError::Degenerate`, never a panic.
- **FIXED-approx eval_metric ground truth:** because ranking metrics are eval-only, the Python-reachable truth is `eval_metric(label, approx, metric, group_id=…)` over a frozen approx vector — independent of any trained model (the Rust trainer need not reproduce a PairLogit model to gate a metric formula). Inputs stored as float64 so `cb_oracle::load_f64_vec` (the only npy loader) reads them all.
- **QueryAUC concordance transcription (Rule 1 fix):** the first transcription of `CalcAUC`'s weighted merge-sort inversion path diverged (0.5 vs upstream 0.48 on a tied group). `CalcAUC` provably reduces to the standard ranking-AUC concordance probability; replacing the merge-sort with the direct `O(size²)` concordance count is bit-identical for single-thread unit-weight small groups and far clearer.
- **QueryAUC dual-mode coverage:** the singleclass default is `Classic` (binary-class AUC, requires `target ∈ [0,1]`), so the graded-relevance corpus is tested with `type=Ranking` plus a `Classic` case over a binarized target.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] QueryAUC:Ranking concordance transcription**
- **Found during:** Task 2 (oracle gate `queryauc_ranking` failed 0.5 vs 0.48).
- **Issue:** the initial `ranking_auc_group` transcribed `CalcAUC`'s `optimisticAUC` inversion count + `deltaSum`/`pairWeightSum` weight corrections literally, but the naive accumulators did not reproduce the merge-sort's weighted result on a group with a non-adjacent equal-target pair.
- **Fix:** replaced with the direct concordance count `CalcAUC` reduces to (over ordered pairs with `target[i] > target[j]`, fraction with `approx[i] > approx[j]`, ties 0.5). Verified per-group against upstream eval_metric on each of the 5 corpus groups.
- **Files modified:** `crates/cb-train/src/ranking_metrics.rs`.
- **Commit:** `274fbb9`.

The plan's MAP formula note ("`(Σ_k [rel@k]·precision@k)/#relevant`") was refined to the exact upstream `CalcAveragePrecisionK` (`precision_recall_at_k.cpp:62-83`): iterate ALL docs in sorted order, increment `hits` on each relevant doc, add `hits/(pos+1)` only when `pos < k`, and normalize by `min(hits, k)`. Documented in the function citation.
