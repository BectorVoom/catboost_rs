## Accomplishments
- `build_query_info` builds the grouped (ranking) view from a Pool's group_id/subgroup_id/pairs with contiguous-unique run assertion (typed `CbError::Degenerate`, no panic) and group-local pair→competitor remapping (typed `CbError::OutOfRange` on cross-group/out-of-range pairs).
- `calc_ders_for_queries` grouped der seam slices approx/target/weights per group, validates every span before indexing, routes the per-group normalizer through `cb_core::sum_f64`, guards empty groups (returns 0, no divide), and returns a typed "ranking loss not yet wired" error for every loss arm (Plans 02–05 replace them).
- `Runtime::compute_gradients_grouped` sibling trait method with a host-side default impl delegating to the seam; the existing pointwise `compute_gradients` signature is byte-identical (D-04 no-regression — 154 cb-train lib tests + shipped scalar/N-dim oracles still green).
- OFFLINE `gen_ranking_fixtures.py` produces the frozen shared ranking corpus (5 varied-size groups over 12 objects + subgroup_id + explicit pairs), accepts `--inputs`/`--loss`/`--metric`, emits per-stage `.npy`, and was validated end-to-end against `.venv` catboost 1.2.10.

## Decisions Made
- **Compute-tier GroupSpan over cross-crate import:** cb-compute depends on cb-core/cb-data, never cb-train. The seam re-declares a plain-data `GroupSpan`/`Competitor` carrying the same shape as `cb-train::QueryInfo`; the trainer lowers its `QueryInfo` into `Vec<GroupSpan>` at the call site. This keeps the layering intact (RESEARCH Architectural Responsibility Map) and the grouped der host-side (no CubeCL kernel).
- **Sibling grouped trait method (default impl):** `compute_gradients_grouped` is added to the `Runtime` trait with a default body delegating to `calc_ders_for_queries`, so backends need not override it and the pointwise `compute_gradients` signature stays byte-identical (D-04).
- **Group weight = mean of member weights** via `sum_f64` (documented against upstream `groupWeights[group.Begin]`, which equals the mean for the uniform-within-group case); `1.0` when unweighted.
