## Decisions Made
- **WR-03 keys on the leaf method, not grouped-ness.** Gradient leaf needs the pairwise `sumWeight`; Newton leaf must NOT re-weight (der2 already folds the pair weight). YetiRank's catboost default leaf method is **Newton** (verified in the generated `model.json`), so the production YetiRank leaf rides the unit arm; the new Gradient `eff_weights` branch is the correct code for a Gradient-leaf YetiRank config and becomes oracle-exercised once the seed-plumbing gap is closed and a Gradient-leaf fixture is frozen.
- **End-to-end YetiRank/StochasticRank fixtures deferred un-weakened.** The 06.3-10 GO unblocked the build, but the end-to-end gate cannot pass at ≤1e-5 until the Rust sampler models the trainer's per-tree multi-fold seed derivation (a new seeding subsystem, Rule 4 — the same class as the 06.3-13 PairLogitPairwise split-scorer). Freezing now would commit a fixture the trainer fails against; the honest outcome is the recorded deferral.
