# Phase 06.3 Plan 17: YetiRankPairwise End-to-End Closure Summary

**One-liner:** Closed LOSS-04 gap #2 — the YetiRankPairwise end-to-end per-stage oracle now passes all four stages ≤1e-5 — by building the instrumented multi-tree pairwise trainer, calibrating the per-tree `YetiRankTreeSeeder` draw stream draw-for-draw, and fixing the real root cause: the WR-02 candidate-feature undercount (`feature_borders.len()`, counting every quantized float feature, not just selected-border ones).

## What shipped

1. **Instrumented pairwise multi-tree RNG path** (env-gated `CB_INSTRUMENT_LOG`, RUN-ONCE/COMMIT):
   - `train.cpp::TrainOneIteration`: per-tree `tree_rng_start/pre_gts/post_gts/pre_leaf/end` call-count fences (`LearnProgress->Rand.GetCallCount()`).
   - `greedy_tensor_search.cpp`: per-level `gts_level_rng` (CalcScores / SelectBestCandidate draw counts) + per-candidate `cand_score_rng` (distribution / StDev / draws).
   - `yetirank_helpers.cpp`: `update_pairs` (per-phase recalc seed + block_count), `update_pairs_query` (per-query seed), and `competitor_weight` (final sampled per-(winner,loser) weight) fences.
2. **Incremental rebuild** of the instrumented `_catboost.so` (4 incremental rebuilds via `build_instrumented_trainer.sh` over the persistent `/tmp/cb_build313` + `/tmp/clang18_prefix`; each minutes, all exit 0; disk stayed 52–60 GB free, well above the floor).
3. **Captured + froze** the per-tree draw-log ground truth for the exact fixture config at `ranking_corpus/yetirank_pairwise/yetirank_pairwise_tree_rng_groundtruth.jsonl`.
4. **Calibrated the seeder** (`yetirank.rs`): both losses draw the per-candidate normals; added `YetiRankTreeSeeder::call_count()` + `YetiRankTreeSeeds::recalc_seeds`; `cb-core` `TFastRng64::call_count()` mirrors upstream `GetCallCount()`.
5. **WR-02 fix** (`boosting.rs`): `yetirank_n_candidate_features = feature_borders.len()`.
6. **New draw oracle** (`yetirank_pairwise_tree_rng_oracle_test.rs`): asserts the seeder lands the context RNG on the trainer's per-tree fences (0/34/76/108/146/186) AND reproduces the deriv/learnfold/leafval recalc seeds bit-exact for all 5 trees.
7. **Activated the gate**: `yetirank_pairwise_end_to_end_per_stage` passes the full 4-stage ≤1e-5 compare; the dead absent-fixture branch is now a loud panic.

## The diagnosis path (why this was hard)

The end-to-end gate initially diverged at the **Splits** stage. The prior plan's note (ff10d51) claimed trees 0+1 matched and tree 2+ needed a non-derivable draw count. The instrumentation disproved both:

- The per-query deriv/leafval seeds the seeder derived were **already bit-exact** to the trainer (verified group-by-group), and the sampled competitor weights matched bit-exact for every group.
- Yet the **leaf-value der_sums diverged** even for tree 0. Tracing the seeder's persistent-RNG call count against the trainer's `tree_rng_start` fences exposed the seeder consuming **28/60/84/112** draws where the trainer consumed **34/76/108/146** — a fixed −6-per-tree shortfall.
- The shortfall localized to the per-level GTS draws: the seeder used `n_features = 3` while the trainer drew for `4` candidates. The model.json lists 4 float features but feature 2 has **0 selected borders** (it ends unused). The `!b.is_empty()` filter dropped it; the trainer still drew an Rsm + a normal for it every level.

Fixing the count to `feature_borders.len()` realigned every tree's draw stream and the full 4-stage gate went green.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] WR-02 candidate-feature undercount (the real root cause)**
- **Found during:** Task 4/5 (seeder calibration vs the instrumented per-tree fences).
- **Issue:** `boosting.rs::yetirank_n_candidate_features` filtered `!b.is_empty()`, counting only float features with SELECTED borders (3). The trainer draws an Rsm `GenRandReal1` + a `SelectBestCandidate` Box-Muller normal per ALL quantized float features (4). Corpus feature 2 ends unused but was a training candidate, so the count was short by 1, under-drawing each tree and desyncing the learnfold/leafval recalc seeds from tree 1 onward.
- **Fix:** `yetirank_n_candidate_features = feature_borders.len()`.
- **Files modified:** `crates/cb-train/src/boosting.rs`.
- **Commit:** `bc7b661`.

**2. [Rule 1 - Bug] Seeder skipped the per-candidate normals on the pairwise path**
- **Found during:** Task 4.
- **Issue:** `YetiRankTreeSeeder::next_tree` guarded the per-candidate `std_normal` draws behind `if !self.pairwise`, on the (now-refuted) hypothesis that the `*Pairwise` split path drew its `SetBestScore` normals from a child RNG. The `cand_score_rng` fence proved the normals advance the persistent RNG identically to the pointwise path.
- **Fix:** Always draw the normals; the `pairwise` flag is now a documented no-op.
- **Files modified:** `crates/cb-train/src/yetirank.rs`.
- **Commit:** `bc7b661`.
