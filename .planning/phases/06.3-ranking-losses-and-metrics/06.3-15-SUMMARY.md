- Ported `ComputeDerSums` (`pairwise_scoring.h:52-68`) and `ComputePairWeightStatistics` (`pairwise_scoring.h:72-103`, OneFeature branch) to `compute_der_sums` / `compute_pair_weight_statistics`, with the `BucketPairWeightStatistics` struct mirroring `TBucketPairWeightStatistics`.

## Decisions Made
- **cb-compute-local leaf-solve twin (not a cb-train dependency):** crate layering forbids cb-compute depending on cb-train. The twin transcribes the same matrix assembly + 2×2 closed form + `MakeZeroAverage` and calls the identical `crate::pairwise_cholesky_solve`, so the leaf solve is bit-faithful to `cb_train::calculate_pairwise_leaf_values`.
- **CbResult return types:** the artifact table showed bare return types, but the threat model's T-06.3-15-01 mitigation requires bounds-guarding to a typed error. The three public fns return `CbResult` so a malformed index from the trainer trust boundary surfaces `CbError::OutOfRange` rather than panicking. Downstream plans 06.3-16/17 will `?`-propagate.
- **Independent-reference self-oracle:** the first-candidate score test validates against a separate Gaussian-elimination solver (different algorithm) rather than a single hand-typed scalar, catching transcription bugs on either side at ≤1e-9.
