## Accomplishments

- **QueryRMSE der** transcribed verbatim (`error_functions.h:879-933`): per group `queryAvrg = Σ(target-approx)·w / Σw` (queryCount>0 guard, both numerator and denominator through `cb_core::sum_f64`), then per object `der1 = (target-approx-queryAvrg)·w`, `der2 = -1·w`.
- **QuerySoftMax der** transcribed verbatim (`error_functions.cpp:531-577`): per-group max-shifted softmax (`exp(Beta·(approx-maxApprox))·w` — the `calc_softmax` NaN-guard discipline, no overflow), share `p = expApprox·w / Σ`, `der1 = Beta·(-sumWTargets·p + w·target)`, `der2 = Beta·sumWTargets·(Beta·p·(p-1)-Lambda)`; `sumWTargets<=0` or `weight<=0` → ders 0 (no divide).
- **Loss::QueryRmse + Loss::QuerySoftMax{lambda,beta}** added to the enum with full doc-cited derivations; `Loss::validate` rejects `lambda<0` / `beta<=0`; `QUERYSOFTMAX_LAMBDA_DEFAULT=0.01` / `QUERYSOFTMAX_BETA_DEFAULT=1.0` (`loss_description.cpp:209-216`).
- **Grouped der dispatch** in the boosting loop: `is_grouped_loss(&Loss)` gates the der site — ranking losses build the `QueryInfo` view ONCE (`build_query_info`), lower it to `Vec<GroupSpan>`, call `Runtime::compute_gradients_grouped`, and flatten the per-group ders (contiguous `[begin,end)` spans) back into the object-order buffer the pointwise path emits. The non-ranking der site is **byte-identical** (D-04). A new `train_ranking` entry + `RankingData{group_id,subgroup_id,pairs}` thread the group structure through a new `train_inner` parameter; the existing `train`/`train_cat`/`train_with_eval_sets` pass `RankingData::default()`, so the 24 `BoostParams` construction sites are untouched.
- **Per-stage oracles** (`queryrmse_oracle_test`, `querysoftmax_oracle_test`) train over the shared frozen ranking corpus and gate `Splits` / `LeafValues` / `StagedApprox` / `Predictions` ≤1e-5 vs catboost 1.2.10 — no `#[ignore]`, no weakened tolerance. QueryRMSE rides the Newton leaf, QuerySoftMax the Gradient leaf (read from each fixture's `model.json`), both at Cosine split score; predictions are RAW (identity) via the production `cb_model::predict_raw`.
- The **QuerySoftMax fixture** was frozen OFFLINE (`gen_ranking_fixtures.py --loss QuerySoftMax`) against `.venv` catboost 1.2.10; QueryRMSE reuses the 06.3-01 smoke fixture.

## Decisions Made

- **Leaf-estimation method per loss (RESEARCH A4 resolved empirically):** QueryRMSE's upstream default `leaf_estimation_method` is **Newton** (its querywise der carries a real `der2 = -w`); QuerySoftMax's is **Gradient** (NOT Newton). Both were read directly from the fixture `model.json` `tree_learner_options` and pinned per oracle — an initial Newton guess for QuerySoftMax diverged at LeafValues, the fixture revealed Gradient.
- **Cosine split score:** the ranking corpus pins no `score_function`, so the catboost CPU default (Cosine) applies. L2 (the regression-skeleton default used in the prior loss waves) diverged at tree-0 split 1 — both ranking oracles pin Cosine.
- **Querywise der is already weighted.** Upstream folds the per-object weight INTO `der1`/`der2` for these losses (the `weight·target` term and the `p` numerator). The shared boosting loop then re-weights by `weight`; at the unweighted corpus (`weight==1`) this is a no-op, so parity holds. The general weighted-ranking case (`weight != 1`) would double-weight in the shared leaf loop — explicitly deferred (the Wave-A corpus is unweighted; a fold-into-leaf-only routing is a hardening item).
- **Group structure threaded via `RankingData` + `train_ranking`** rather than widening `BoostParams` — keeps the 24 existing `BoostParams` construction sites byte-identical and the new field optional.
