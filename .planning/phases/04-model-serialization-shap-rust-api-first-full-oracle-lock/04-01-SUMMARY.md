## Accomplishments

- cb-train now accumulates per-leaf summed training-document weights (one `Vec<f64>` per tree, length 2^depth) in the boosting loop — closes RESEARCH Pitfall 1 (SHAP / PredictionValuesChange / Interaction all weight on leaf weights; their absence silently zeroed every downstream fstr/SHAP path).
- Canonical serializable `cb-model::Model { oblivious_trees, bias, float_feature_borders }` with per-tree `leaf_weights`, reusing `cb_train::Split` (re-export, not redefined) — the substrate every later serialize/apply/explain plan operates on, needing no training pool.
- Committed genuine `flatc --rust` FlatBuffers bindings for the vendored model/features/ctr_data schema, compiling against `flatbuffers = "25.12.19"` (D-01: no hand-rolled bindings).
- `cb-oracle::model_json` parses upstream per-tree nested `leaf_weights` with `#[serde(default)]` (backward-compatible — Phase-3 fixtures without the field still parse) plus a `leaf_weights()` accessor.
- Offline fixture-generation manifest staged in `gen_fixtures.py` for the downstream oracle locks (D-13).
- Leaf-weights oracle lock test passes 2/2 (per-tree sum == doc count for the unweighted fixture, and Rust-captured == upstream model.json leaf_weights ≤1e-5).

## Deviations from Plan

### Accepted Deviation (user-approved)

**1. [Rule 3 — Blocking, resolved at checkpoint] flatc generation command: per-file → `--gen-all`**
- **Found during:** Task 2 / Task 3 (checkpoint)
- **Issue:** The plan's per-file command `flatc --rust -o ... model.fbs features.fbs ctr_data.fbs` produces files that reference each other's include paths. The schemas use cross-file includes; generating self-contained bindings required `flatc --rust --gen-all -I catboost-master`.
- **Fix:** Used `flatc --rust --gen-all -I catboost-master ...` so each `*_generated.rs` inlines its schema includes and compiles standalone. Output is still pure flatc output (D-01 intact); no hand editing, no planus fallback.
- **Files modified:** `crates/cb-model/src/generated/{model,features,ctr_data}_generated.rs`
- **Verification:** `cargo build -p cb-model` exits 0 against flatbuffers 25.12.19; generated headers confirm machine generation; user typed "approved" at the blocking-human checkpoint explicitly accepting this deviation.
- **Committed in:** `a6801b9` (Task 2 commit)

---
