## Wave grouping & oracle batching

| Option | Description | Selected |
|--------|-------------|----------|
| Grouped family waves | Additive narrowest-first by family, each wave its own oracle gate: smooth (LogCosh/Lq/Huber/Expectile) → positive-domain/link (Poisson/Tweedie/MAPE/MSLE) → quantile family (Quantile). | ✓ |
| Single all-in-one plan | One plan + one combined oracle pass — coarse gate, blurs regressions. | |
| Per-loss plan | One plan + gate per loss — maximal isolation, ~10 plans of ceremony. | |

**User's choice:** Grouped family waves.
**Notes:** Smaller blast radius and unambiguous regression attribution while keeping plan count reasonable. Exact wave-to-plan mapping left to the planner.

## Loss-parameter API surface

| Option | Description | Selected |
|--------|-------------|----------|

**User's choice:** Enum variants + upstream defaults.
**Notes:** `Quantile{alpha,delta}`, `Expectile{alpha}`, `Huber{delta}`, `Tweedie{variance_power}`, `Lq{q}`; non-parametric LogCosh/Poisson/MAPE/MSLE. CatBoost-native string parsing (`"Quantile:alpha=0.7"`) deferred to Phase 8.
