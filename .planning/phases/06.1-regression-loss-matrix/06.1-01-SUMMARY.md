## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] LogCosh Exact leaf method was using the MAE weighted-median optimizer**
- **Found during:** Task 3 (LogCosh oracle FAILED at LeafValues: expected -0.2838, actual -0.3136, diff 0.0298 — splits matched, leaves diverged).
- **Issue:** The plan's `<action>` asserted "the leaf path needs NO new code" because `compute_leaf_deltas` already dispatches Newton/Exact. But the existing `Exact` arm hardcoded `QUANTILE_ALPHA`/`QUANTILE_DELTA` — it is a MAE-only weighted-median. Upstream's `CalcOneDimensionalOptimumConstApprox` is a per-loss switch: for LogCosh it runs `CalculateOptimalConstApproxForLogCosh` (a 100-iteration bisection on the residual bracket solving `Sum tanh(d - r) = 0`), which is a different optimum entirely.
- **Fix:** Added `logcosh_exact_leaf_delta` (cb-compute::leaf) transcribing the upstream bisection, threaded `loss` into `compute_leaf_deltas`, and dispatched the Exact arm by loss (`Loss::LogCosh => logcosh_exact_leaf_delta`, `_ => exact_leaf_delta` quantile). Lq/Huber/Expectile use Newton (unaffected) and were already GREEN.
- **Files modified:** `crates/cb-compute/src/leaf.rs`, `crates/cb-compute/src/leaf_test.rs`, `crates/cb-compute/src/lib.rs`, `crates/cb-train/src/boosting.rs`.
- **Commit:** `1d43436`

**2. [Rule 2 - Missing critical functionality] Loss::validate range-guard wired into the training entry**
- **Found during:** Task 2/3 (threat register T-06.1.01-01/02 assigns `mitigate` to out-of-domain q/delta/alpha).
- **Issue:** The plan specified validating in "the Loss constructor path", but `Loss` is a bare enum with no constructor — there was no chokepoint enforcing `q>=1`/`delta>0`/`alpha in [0,1]`. Without it an out-of-domain param produces NaN/Inf derivatives that poison the histogram and leaf reductions (T-06.1.01-01/02).
- **Fix:** Added `Loss::validate() -> CbResult<()>` (typed `CbError::OutOfRange`, no `unwrap`/`panic`) and called `params.loss.validate()?` at the top of `train_inner` so every train path (numeric/eval/cat) is guarded.
- **Files modified:** `crates/cb-compute/src/runtime.rs`, `crates/cb-train/src/boosting.rs`.
- **Commit:** `6ddb49b` (validate fn) + `1d43436` (train_inner gate)

**3. [Rule 3 - Blocking issue] Three existing oracle-test `match loss` helpers broke at compile time**
- **Found during:** Task 3 (full `cargo test -p cb-train` failed to compile: non-exhaustive `match loss` in `slice_first_oracle_test.rs`, `leaf_methods_oracle_test.rs`, `autolr_e2e_test.rs`).
- **Issue:** Adding the four `Loss` variants made test-side exhaustive matches (loss -> target column, loss -> TargetType) non-exhaustive.
- **Fix:** Extended each match — smooth losses map to `load_regression_target()` (regression family) and `TargetType::Unknown` (not in the auto-LR table). No oracle assertion weakened.
- **Files modified:** the three test files above.
- **Commit:** `1d43436`
