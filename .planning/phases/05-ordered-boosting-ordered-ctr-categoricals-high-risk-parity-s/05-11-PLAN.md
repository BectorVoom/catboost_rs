<objective>
Build the categorical-CTR INGESTION + per-fold online-CTR MATERIALIZATION half of
the ORD-05 training pipeline (the upstream `greedy_tensor_search.cpp` AddTreeCtrs
+ per-fold online-CTR-during-growth path). Add a NEW cat-aware `train_cat` entry
point that threads categorical columns into training — computing per-feature
learn-set cardinalities and, for each tensor-CTR candidate projection, materializing
a per-document combined-projection ONLINE CTR feature column (read-before-increment,
quantized to CTR bins) the tree search can split on. The numeric `train()` path
stays byte-identical.

Purpose: Close the structural root cause of the live ORD-05 gap — `cb_train::train()`
has NO categorical-column input (cat_cardinalities hardcoded `&[]` at boosting.rs),
so `tensor_ctr_candidates` is always fed an empty cat set and emits nothing during
real training. This plan supplies the cat columns and the materialized CTR-feature
columns; Plan 05-12 scores them into the tree, bakes ctr_data, and closes the e2e
oracle.

Output: `crates/cb-train/src/ctr/ctr_feature.rs` (materialize_ctr_feature +
CtrFeatureColumn), a `train_cat` entry point in boosting.rs that computes cat
cardinalities and materializes a CTR-feature column per candidate, and a RED→GREEN
materialization test. Implements ORD-05 per D-05.
</objective>
