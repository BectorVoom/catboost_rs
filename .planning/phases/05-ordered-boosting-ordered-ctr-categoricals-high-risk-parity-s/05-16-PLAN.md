<approach_decision>
PREFERRED: option (b) — gate/parametrize the structural-divergence sub-test on
`permutation_count >= 2`. Rationale (justified per the verifier's instruction to
pick the most upstream-faithful option):

- At `permutation_count=1`, upstream itself builds the lone learning `Folds[0]` as
  the IDENTITY (`shuffle = foldIdx != 0`), so Ordered structure search runs on
  object order. On a synthetic dataset with no randomness, Ordered per-segment
  scoring on the identity order CAN coincide with Plain — this is faithful
  behavior, not a bug. Asserting divergence at pc=1 is therefore asserting a
  false premise.
- At `permutation_count>=2`, upstream builds genuine non-identity learning folds
  (`fold.cpp:54` `shuffle = foldIdx != 0` → folds 1..pc-1 are shuffled), and the
  structure search picks a learning fold whose per-segment body/tail order differs
  from object order — so Ordered legitimately diverges from Plain. This is the
  faithful regime in which the divergence assertion is meaningful.
- Option (b) becomes sound ONLY after 05-15 corrects the pc>=2 averaging-fold
  draw position; hence the `depends_on: ["05-15"]` and `wave: 2`.

FALLBACK: option (c) — if, after correctly constructing a pc>=2 config, the small
synthetic dataset still does not diverge (the L2 scores remain tied even under a
non-identity learning fold), retire the divergence sub-test with an in-file
rationale comment referencing `ordered_boost_e2e_oracle_test` (the authoritative
≤1e-5 ORD-02 oracle) and drop a tracked note. The aliveness gates already prove
the branch is not dead code; the e2e oracle proves structural correctness against
upstream. Do NOT silently delete — document the rationale in the file.

Option (a) (synthesize a new diverging dataset under the identity fold) is
rejected as the least upstream-faithful: it engineers a dataset to force
divergence under a regime where upstream itself does not diverge, masking rather
than reflecting the true behavior.
</approach_decision>
