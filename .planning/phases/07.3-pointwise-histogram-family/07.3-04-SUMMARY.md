## FROZEN binSums seam (reused UNCHANGED across the THIRD kernel family)

```text
index(feature, bin, channel) = (feature * n_bins + bin) * 2 + channel   channel 0 = Σ der1, channel 1 = Σ weight
buffer length = n_features * n_bins * 2 floats   (n_bins = BINARY_BINS = 2 for the binary family)
```

The binary kernel is a different `#[cube]` fn from the non-binary and half-byte kernels, but it writes the SAME interleaved index with `n_bins=2`, so the 7.5 score/split consumer reads one layout regardless of feature bit-width. The seam, buffer-length formula, and der-handle-in -> binSums-handle-out hand-off are byte-identical to Plans A/B/C (D-7.3-01). The `fullPass=false` multi-part offset remains a 7.5 forward dependency (documented in Plan A, not filled here).

## Decisions Made
See `key-decisions` frontmatter.
