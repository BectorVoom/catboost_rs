**As a** Rust developer embedding the GPU training path (and the downstream 7.5 score/split consumer), **I want to** fill a device-resident 2-channel pointwise histogram for 8-bit non-binary features on rocm directly from the on-device der/weight/bin handles, **so that** the split scorer can pick splits with no host round-trip and verifiable parity against the CPU histogram.

|----------|-------------|
| host launch helper → device kernel | In-process buffer lengths/shapes cross here; the only untrusted-shape surface (no network/auth/file/user input) |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
