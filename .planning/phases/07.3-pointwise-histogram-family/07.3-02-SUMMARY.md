## FROZEN binSums seam (reused UNCHANGED — Plan A contract)

```text
index(feature, bin, channel) = (feature * n_bins + bin) * 2 + channel   channel 0 = Σ der1, channel 1 = Σ weight
buffer length = n_features * n_bins * 2 floats   (n_bins = 1 << bits)
```

Plan B changes ONLY which `bits` (hence `n_bins`) is selected host-side; the kernel write index, the buffer length formula, and the der-handle-in -> binSums-handle-out seam are byte-identical to Plan A. This layout stays FROZEN for Plans C/D and the 7.5 score/split consumer.

## Decisions Made
See `key-decisions` frontmatter. Headline: the Plan A kernel was already comptime-bits-general, so Plan B is a host-dispatch + oracle slice with the seam reused unchanged; the direct global atomic merge needs no per-bit shared-mem prefix change (the inner/outer bit split is an additive perf follow-up, not a correctness requirement); non-one-byte border counts are typed-error-rejected, reserving half-byte/binary for Plans C/D.
