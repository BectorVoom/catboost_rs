# Phase 7.2 Plan 02: On-device Logloss/CrossEntropy + Quantile/MAE der families on the der seam Summary

**Logloss/CrossEntropy der1 (sigmoid-gradient, binary launch) + der2 (single-input hessian launch) and Quantile/MAE der1 (parametric alpha/delta launch) + constant-0 der2 handle now run device-resident over SelectedRuntime, self-oracled vs the cb-compute::loss baselines on rocm gfx1100 with reported divergence <=1e-14 (most fixtures bit-exact 0.0).**

## Accomplishments
- Extended the Plan-01 der seam additively with the Logloss/CrossEntropy family: `DerBinaryKernel::LoglossGradient` (der1, reusing `logloss_gradient_kernel`) and the new `DerUnaryKernel`/`launch_der_unary_handle`/`launch_der_unary` single-input hessian seam (der2, reusing `logloss_hessian_kernel`). Logloss AND CrossEntropy route to the SAME kernels (Pitfall 6 / D-09) — confirmed by an explicit reuse oracle (0.0 divergence vs `cross_entropy_der1`).
- Added the parametric launch family: `DerParamKernel`/`launch_der_param_handle`/`launch_der_param`, passing `alpha`/`delta` as length-1 `Array<F>` device buffers (the `launch_quantile_f64` precedent, `SelectedRuntime` swapped in) to keep the kernel generics-float. MAE routes through `QuantileGradient` at `(QUANTILE_ALPHA, QUANTILE_DELTA)` (WR-04 — no duplicate MAE kernel), verified bit-identical to both `mae_der1` and `quantile_der1{0.5}`. Params are read via a non-panicking `param_pair()` accessor surfacing `CbError::Degenerate` (D-13).
- Wired Quantile/MAE der2 = 0.0 through the Plan-01 `const_der_handle(0.0, n)` (Pitfall 5 — no quantile hessian kernel) and oracle-checked the handle reads all-0.0 == `cb_compute::quantile_der2`/`mae_der2`.
- 10 new self-oracle tests (gradient_gpu suite now 15 total): rocm gfx1100 15/15 GREEN, REPORTED divergence 0.0 for every quantile/mae/crossentropy fixture and <=1e-14 for logloss (the sigmoid exp). wgpu build RC=0; cuda build RC=0 (compile-only). SC-4 holds: cb-compute cubecl-free, cb-core/cb-model byte-unchanged.

## Decisions Made
- **Logloss == CrossEntropy (Pitfall 6 / D-09):** one kernel, two enum arms — the Rust seam collapses both to `logloss_gradient_kernel`/`logloss_hessian_kernel`; CrossEntropy gets no kernel of its own.
- **MAE == Quantile{0.5, 1e-6} (WR-04):** MAE routes through `QuantileGradient` at the default params; no MAE kernel exists.
- **Quantile/MAE der2 = 0.0 (Pitfall 5):** the constant-0 device handle from `const_der_handle`, not a kernel.
- **der1/der2 UNWEIGHTED (A1):** matches the `cb-compute::loss` baseline; weight folded downstream by 7.3 — no `weights` arg added (honoring the locked Plan-01 contract).
- **Single launch geometry per family (IN-02):** each der family routes both its public entry points through one private `launch_der_*_into(client, ...)` helper, preserving the Plan-01 same-client launch+read-back discipline (a CubeCL Handle is bound to its originating client).
