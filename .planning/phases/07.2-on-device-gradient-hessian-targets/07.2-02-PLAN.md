<objective>
Second MVP vertical slice: additively extend the Plan-01 der seam to the
**Logloss/CrossEntropy** and **Quantile/MAE** derivative families, reusing the
`launch_der_binary` / handle / `const_der_handle` machinery already landed. Each family
runs device-resident over `SelectedRuntime`, returns der1/der2 as device HANDLES (SC-3),
and is self-oracled against the `cb-compute::loss` CPU baseline on `rocm` (gfx1100).

Logloss/CrossEntropy add a kernel-backed der2 (`logloss_hessian_kernel`) — proving the
`launch_der_unary` (single-input hessian) seam shape. Quantile/MAE add the parametric
launch (`alpha`/`delta` as length-1 `Array<F>`) — proving the `launch_der_param` seam shape
and the constant-der2 handle (Pitfall 5). After this plan a user can compute Logloss,
CrossEntropy, Quantile, and MAE der1/der2 device-resident, each oracle-checked.

Purpose: Cover the binary-classification + robust-regression der families on the seam.
Output: `DerBinaryKernel::LoglossGradient`, `launch_der_unary` (logloss hessian),
`launch_der_param` + `DerParamKernel` (quantile), and their `gradient_gpu.rs` oracle tests.

Honors: D-7.2-01 (Logloss/CrossEntropy + Quantile/MAE in the narrow start), D-7.2-03 (reuse
the authored kernels), D-7.2-04 (device handles), D-7.2-05 (cb-backend-only), D-7.2-06
(report-not-sign-off oracle), D-7.2-07 (per-crate). der stays UNWEIGHTED (A1). Logloss==
CrossEntropy (D-09 / Pitfall 6); MAE==Quantile{0.5, 1e-6} (WR-04 / Pitfall 5).
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
