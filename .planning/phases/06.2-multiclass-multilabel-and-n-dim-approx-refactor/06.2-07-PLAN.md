<objective>
Close GAP-2 / CR-02 (Critical, 06.2-REVIEW.md): the dim-major derivative buffer
corrupts the sampling path on a multi-output model. `weighted_der1` (length
`approx_dimension * n`, dim-major) is passed UNMODIFIED to
`bootstrap(params.bootstrap_type, &weighted_der1, …)` / `mvs_lambda(&weighted_der1, …)`
/ `score_st_dev(params.random_strength, &weighted_der1, …)` (boosting.rs:1981-2034),
which all assume a per-object length `n`. For Bernoulli/MVS this draws `dim*n` uniforms
and masks per-(dim,object) instead of per-object (breaking the RNG-phase parity and the
documented per-object sampling); `derivatives_std_dev_from_zero` (score.rs:166-172)
divides by `weighted_der1.len() == dim*n` instead of `n`, scaling `scoreStDev` by
`1/sqrt(dim*n)`. Latent under the in-scope fixtures (`bootstrap_type=No`,
`random_strength=0`) but corrupts the moment a user enables either knob on a
multi-output loss.

The upstream parity model is now known (greedy_tensor_search.cpp:92-107
`CalcDerivativesStDevFromZeroPlainBoosting` and mvs.cpp:37-65 `CalcMeanGradValue`):
sampling and std-dev operate PER OBJECT on the L2 norm of the multi-dim gradient
(`sqrt(Σ_d der[d][i]²)`), and the std-dev divisor is `n` (the per-dimension length),
NOT `dim*n` — `sum2 = Σ_d L2NormSquared(der[d])` over all `dim*n` elements but
`sqrt(sum2 / front().size())` where `front().size() == n`. This is the correct,
tractable per-object aggregation, so this plan implements full parity (preferred over
gating per the gap brief).

Folds in two cheap, clearly-correct WARNINGs in the same gradient/score subsystem:
WR-05 (softmax/OneVsAll `target_class < k` bound at the backend call site) and WR-01
(the `multi_dim_candidate_score` whole-buffer fallback on a bad stride).

Purpose: make the multi-output sampling / random_strength path correct (per-object,
RNG-phase-faithful) so it is no longer a latent corruption in the public API.

Output: per-object derivative aggregation feeding `bootstrap`/`mvs_lambda`, the
corrected `derivatives_std_dev_from_zero` divisor, the WR-05 typed bound, the WR-01
typed-error/empty-slice stride guard, and a regression test.
</objective>
