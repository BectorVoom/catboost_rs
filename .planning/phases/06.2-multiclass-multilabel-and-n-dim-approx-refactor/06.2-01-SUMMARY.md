# Phase 6.2 Plan 01: Wave-0 Compute-Tier N-Dim Approx Refactor Summary

**Threaded a dimension-major `approx_dimension` through `Runtime::compute_gradients` and the `CpuBackend` per-dimension kernel-launch loop, and dropped `Copy` on `Loss` — all byte-identical to the pre-6.2 scalar path at dim=1, with no new loss math.**

## Performance

- **Duration:** ~20 min
- **Completed:** 2026-06-16
- **Tasks:** 2 (both `tdd="true"`)
- **Files modified:** 9 (3 in the plan's `files_modified` + 6 cross-crate non-Copy ripple)

## Accomplishments

- `Runtime::compute_gradients` now carries `approx_dimension: usize` (after `target`) and takes `&Loss`; `approx` is the dimension-major flat buffer `approx[d*n_objects+i]` (D-6.2-01), with `Derivatives` documented as the matching dim-major layout.
- `Loss` no longer derives `Copy` (the Wave-3 `MultiQuantile{alpha:Vec<f64>}` ripple, surfaced now as a one-time refactor); `Clone` retained.
- `CpuBackend::compute_gradients` validates shape (`approx.len() % dim == 0`, `target.len()==n`, `dim != 0`) with typed `CbError::LengthMismatch` (T-6.2-01a, no panic), then runs an OUTER `for d in 0..approx_dimension` loop over `approx[d*n..d*n+n]` reusing the existing per-loss kernel launchers (extracted to `compute_gradients_one_dim`), concatenating into dim-major `der1`/`der2`.
- At `approx_dimension==1` the loop runs exactly once over `approx[0..n]` — proven byte-identical (exact `==`) by a new dim=1 byte-identity test, plus a multi-dim per-dimension concatenation test and an n-dim shape-error test.

## Task Commits

1. **Task 1: Drop Copy on Loss; thread approx_dimension through the Runtime trait seam** — `6eaf0da` (refactor)
2. **Task 2: Widen CpuBackend::compute_gradients to a per-dimension kernel-launch loop (dim=1 unchanged)** — `f209bd5` (refactor)

_Both tasks are `tdd="true"`; this is a pure mechanical refactor whose RED guard is the existing scalar der1/der2 suite staying green, augmented by the new dim=1 byte-identity / multi-dim / shape-error tests (the Wave-0 D-04 anchor at the unit level)._

## Files Created/Modified

- `crates/cb-compute/src/runtime.rs` — `Loss` `Copy` derive dropped; `compute_gradients` trait method gains `approx_dimension` and `&Loss`; `Derivatives` doc'd dimension-major (`buf[d*n_objects+i]`).
- `crates/cb-backend/src/cpu_runtime.rs` — `compute_gradients_one_dim` extracted (single-dimension body, math unchanged); `CpuBackend::compute_gradients` widened with `approx_dimension`, shape guards, and the per-dimension outer loop.
- `crates/cb-backend/src/cpu_runtime_test.rs` — all call sites updated to `&Loss` + `approx_dimension`; added `dim1_is_byte_identical_to_scalar_path`, `multi_dim_separable_concatenates_per_dimension`, `ndim_shape_mismatch_is_error_not_panic`.
- `crates/cb-train/src/boosting.rs` — `autolr_target_type`/`validate_leaf_method`/`compute_leaf_deltas` take `&Loss` (`match *loss` for Copy-field bindings); `compute_gradients` call passes `&params.loss` + `approx_dimension=1`; `compute_leaf_deltas` call passes `&params.loss`.
- `crates/cb-train/src/metrics.rs` — `EvalMetric::for_loss` takes `&Loss`.
- `crates/cb-train/src/boosting_test.rs`, `crates/cb-train/src/metrics_test.rs` — `&Loss` call sites.
- `crates/cb-train/tests/one_hot_oracle_test.rs` — `&Loss` + `approx_dimension=1` call site.
- `crates/catboost-rs/src/builder.rs` — `CatBoostBuilder` drops `Copy` (non-Copy `loss` field); `loss` cloned into `BoostParams` in `boost_params(&self)`.

## Decisions Made

- **Drop `Copy` on `Loss` now**, before any new variant, so the Wave-3 `MultiQuantile{alpha:Vec<f64>}` non-Copy requirement is a one-time cross-crate refactor (per D-6.2-05 / D-6.1-03 and the plan's explicit Task 1 instruction).
- **Trait takes `&Loss`** rather than `Loss` by value, so the boosting loop can pass `&params.loss` without moving out of borrowed `params`.
- **Per-dimension reduction stays an OUTER loop** reusing existing kernel launchers (extracted into `compute_gradients_one_dim`), never fused into `0..dim*n` — the byte-identity invariant at dim=1 (RESEARCH Pitfall 1).
