## N-Dim Approx Memory Layout

| Option | Description | Selected |
|--------|-------------|----------|
| Dimension-major flat `Vec<f64>` (`approx[d*n+i]`) | Single contiguous allocation, outer=dimension matching upstream `TVector<TVector<double>>`; dim=1 == today's scalar buffer | ✓ |
| `Vec<Vec<f64>>` | Per-dimension allocations; literal mirror of the C++ type but fragmented | |
| Object-major flat (`approx[i*dim+d]`) | Contiguous but groups by object, diverging from upstream reduction order | |

**Auto-selected:** Dimension-major flat buffer (recommended).
**Notes:** Memory-efficiency-first + the need for a bit-identical dim=1 reduction order (so the D-04 checkpoint is a true no-behavior-change proof) dictate this. → D-6.2-01.
