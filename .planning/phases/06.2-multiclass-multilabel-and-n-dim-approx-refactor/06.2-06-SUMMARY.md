# Phase 06.2 Plan 06: Public N-dim Apply + Class-Label Round-Trip (CR-01) Summary

Closed CR-01 (Critical, 06.2-REVIEW.md): the public multi-dimensional apply path. A `.cbm`/json-loaded MultiClass / MultiClassOneVsAll / MultiLogloss / MultiCrossEntropy / MultiQuantile model now predicts ALL `approx_dimension` outputs through the public `predict_raw_multi` -> `apply_multiclass_prediction` path (was: silently dimension-0 only), and a loaded multiclass model recovers its original class labels via `class_to_label` round-tripped from `model_info.class_params` / the `.cbm` InfoMap (was: an empty stub from 06.2-03).

## What Shipped

### Task 1 — `predict_raw_multi` (commit b7b3f26)
- New public `predict_raw_multi(model, feature_values) -> Vec<f64>` returning the DIMENSION-MAJOR raw approx of length `approx_dimension * n` (`out[d*n + i]`).
- Per object: compute the forward-bit-order `leaf` ONCE (extracted `leaf_index_for`, dimension-agnostic, shared with the scalar `predict_raw_one`). Per tree, `n_leaves = leaf_values.len() / dim` (checked), read `leaf_values.get(d*n_leaves + leaf)` for each `d`; sum per-(dim,object) tree contributions through `cb_core::sum_f64` (D-08); `bias` seeded once per output slot (RESEARCH Pitfall 6).
- `predict_raw_cat` routes `approx_dimension > 1` to the dim-aware accumulator and keeps the BYTE-IDENTICAL scalar path at `<= 1` (D-04 on the public surface, proven by a `to_bits()` unit test).
- All access is checked `.get` (`indexing_slicing` deny, T-6.2-06-01): an out-of-range leaf contributes 0.0, never a panic.

### Task 2 — class label deserialize round-trip (commit a7795bf)
- **json**: added a `model_info` field to `ModelJsonDoc` (`#[serde(default)]` + `skip_serializing_if = Option::is_none`), parse `class_params.class_to_label` (fallback `multiclass_params`) in `from_doc`, emit `class_params` in `to_doc` only when `class_to_label` is non-empty.
- **.cbm**: write a `class_params` JSON-STRING value into the `TModelCore` InfoMap on save (only when non-empty); `read_class_to_label` parses it back in `decode_cbm`.
- Both paths replace the 06.2-03 `class_to_label: Vec::new()` stub. Scalar models emit no `model_info`/InfoMap, so their bytes stay byte-identical (D-04). Parsing is checked / non-panicking (`ModelError::Deserialize` on malformed JSON, T-6.2-06-02).

### Task 3 — public load->predict oracle (checkpoint, auto-approved)
- `tests/multiclass_apply_oracle_test.rs`: for each of the 5 frozen catboost 1.2.10 fixtures, `cb_model::load_json(model.json)` -> `predict_raw_multi` -> `apply_multiclass_prediction` compared to the object-major `predictions.npy` at <= 1e-5. NO `from_trained` in the predict path; NO `#[ignore]`, NO weakened tolerance.
  - softmax -> Probability/Softmax; onevsall -> Probability/OneVsAll; multilogloss + multicrossentropy -> Probability/MultiLabel; multiquantile -> RawFormulaVal.
  - Plus a `Class`-transform test proving the public load recovers `class_to_label [0,1,2]` from `model_info.class_params`.
