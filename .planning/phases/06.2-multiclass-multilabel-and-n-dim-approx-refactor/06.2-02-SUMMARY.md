# Phase 6.2 Plan 02: Wave-0 Train/Model N-Dim Approx Refactor + D-04 Hard Checkpoint Summary

**Widened the `cb-train` boosting loop and `cb-model` serialization from a scalar per-object approx to the DIMENSION-MAJOR flat buffer `approx[d*n+i]` (leaf-major transpose on the wire), then re-ran ALL existing scalar oracle fixtures green at `approx_dimension=1` — proven byte-identical (`== 0.0` diff) by a test-first gate, with no new loss math.**

## Performance

- **Duration:** ~12 min
- **Completed:** 2026-06-16
- **Tasks:** 3 (all `tdd="true"`)
- **Files modified:** 10 (4 production + 1 test created + 6 test-literal ripple + 1 validation)

## Accomplishments

- **Task 1 (boosting loop, dim-major):** the `cb-train` boosting loop now carries the DIMENSION-MAJOR approx buffer `approx[d*n+i]` of length `approx_dimension*n` at every seam — init (`vec![bias; approx_dimension*n]`), the gradient call (`compute_gradients(..., approx_dimension)`), the per-object weighting (`idx % n`), the leaf-delta solve, the approx update, and the staged record. `compute_leaf_deltas` is dispatched per-dimension over `approx[d*n..d*n+n]` in an **OUTER `for d` loop** (never fused into `0..dim*n`, the Pitfall-1 invariant), reusing the existing scalar solver verbatim. `loss_approx_dimension(&Loss)` returns `1` for every scalar loss; `cb_train::Model` gained an `approx_dimension` field.
- **Task 2a (test-first D-04 gate):** `ndim_dim1_identity_test.rs` trains `regression_skeleton` (RMSE) through the now-N-dim `train` path at `approx_dimension=1` and asserts the staged approx AND leaf values diff vs an independent scalar reference is **EXACTLY `== 0.0`** (not a `<=1e-5` tolerance). No `#[ignore]`, no weakened tolerance.
- **Task 2b (cb-model N-dim serialization, D-04 GREEN):** `cbm.rs` writes the real `ApproxDimension` (no longer hardcoded `1`) and transposes the dim-major training buffer to **LEAF-MAJOR** `leaf_values[l*dim+d]` on save (un-transposing on load via separate value/weight cursors); `json.rs` does the same leaf-major transpose plus a per-dim bias vector. At `dim=1` leaf-major == dim-major so the `.cbm`/json bytes are byte-identical. `cb_model::Model` gained `approx_dimension` (default 1).

## Decisions Made

See the `key-decisions` frontmatter above. Headline: the boosting loop and serialization are fully dimension-major NOW (sourced from a `loss_approx_dimension` helper that returns 1 for scalar losses), with the per-dimension reduction kept as an OUTER loop and the wire transposed leaf-major — so at `dim=1` the entire scalar pipeline is byte-identical (`== 0.0`), and the multi-output losses in Plans 03..05 only override the dimension count, not the plumbing.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] New `approx_dimension` field ripple into Model struct literals**
- **Found during:** Task 2b (adding `approx_dimension` to `cb_train::Model` and `cb_model::Model`)
- **Issue:** Six `cb-model` integration-test files and one `cb-train` test construct `Model { ... }` literals; the new required field broke their compilation (`E0063 missing field approx_dimension`).
- **Fix:** Added `approx_dimension: 1` to each `Model` literal (test-only; the production construction paths in `from_trained`/`reconstruct_model`/`from_doc` set the real dim).
- **Files modified:** `crates/cb-model/tests/{json,fstr,shap,apply,cbm,predict}_oracle_test.rs`, `crates/cb-train/tests/ctr_split_scoring_test.rs`
- **Verification:** `cargo test -p cb-model --tests` + `cargo test -p cb-train --tests` green.
- **Committed in:** `ea282e9` (Task 2b commit).

**2. [Rule 3 - Scoping] MVS grad-norm / tree.rs cross-dim NOT widened (kept dim=1-neutral)**
- **Found during:** Task 1
- **Issue:** The plan's `bootstrap.rs::derivative_l2_norms` (the MVS grad norm `sqrt(Σ_d der²)`) does not exist verbatim — the MVS norm is `mean_grad_value` (per-object `|der|`). Widening it to a cross-dim `sqrt(Σ_d)` requires knowing `(n, dim)` inside `bootstrap()`, which is out of the dim=1 gate's scope and risks perturbing the byte-identity.
- **Fix:** Left the MVS norm element-wise (neutral at `bootstrap_type:No`, which every in-scope fixture pins — RESEARCH A3). `tree.rs` cross-dim split-score column likewise left at dim=1 (explicitly deferred to Plan 06.2-03 by the plan, Pitfall 7 / Open Q2). `predict.rs` apply path is dim-agnostic at dim=1 (no softmax/per-dim-sigmoid added here, per the plan).
- **Files modified:** none (deliberate no-op — these are Plan 06.2-03 tasks).
- **Verification:** the full scalar oracle suite (incl. MVS `bootstrap_oracle_test`) is green at dim=1.

**Total deviations:** 2 (1 auto-fixed Rule-3 blocking field ripple; 1 deliberate Rule-3 scoping no-op matching the plan's explicit deferrals).
