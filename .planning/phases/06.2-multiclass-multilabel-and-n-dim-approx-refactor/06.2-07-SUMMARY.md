# Phase 06.2 Plan 07: Multi-output sampling CR-02 fix + WR-05/WR-01 guards Summary

**Per-object L2-norm derivative aggregation makes bootstrap/MVS/random_strength on multi-output models RNG-phase-faithful, the random_strength std-dev now divides by n (not dim*n), and the softmax/OneVsAll target_class bound + multi_dim_candidate_score stride guard are typed-error-hardened.**

## Accomplishments
- **CR-02 (Critical):** the dim-major `weighted_der1` (length `approx_dimension*n`) is now reduced to a per-OBJECT derivative `der_obj[i] = sqrt(Σ_d weighted_der1[d*n+i]²)` (length `n`, summed through `cb_core::sum_f64`) before being fed to `bootstrap`/`mvs_lambda`, so `set_sampled_control`/`mvs_sample_weights` draw and mask PER OBJECT and the persistent RNG advances by `n`, not `dim*n` (mvs.cpp:50-55, greedy_tensor_search.cpp:92-107).
- `derivatives_std_dev_from_zero`/`score_st_dev`/`model_size_multiplier` now take an explicit per-object count `n`: `Σ(wd²)` is summed over the FULL dim-major buffer but divided by `n` (and the `ln(n)` model-size multiplier uses `n`), matching `CalcDerivativesStDevFromZeroPlainBoosting` / `CalcScoreStDev`.
- **WR-05:** `compute_softmax_gradients`/`compute_onevsall_gradients` now return `CbResult` and reject `target_class >= k` with `CbError::OutOfRange` instead of silently emitting the degenerate no-`+1` (`-p`) gradient.
- **WR-01:** `multi_dim_candidate_score` replaces `der1.get(..).unwrap_or(der1)` with `unwrap_or(&[])`, so a stride mismatch scores 0 for that dimension instead of substituting the whole wrong-dimension buffer.
- **WR-04:** added per-object `debug_assert_eq!(sample_weights.len(), n)` / `control.len() == n` guards after `bootstrap`.
- D-04 byte-identity holds at `approx_dimension == 1` (`sqrt(wd²)==|wd|`, buffer length `== n`, `idx % n == idx`), confirmed by the full re-lock.

## Decisions Made
- Implemented full upstream parity (per-object L2-norm aggregation) rather than gating multi-output sampling, per the gap brief.

- WR-01 kept the `-> f64` signature of `multi_dim_candidate_score` (it sits inside the candidate-scoring hot loop) and used the empty-slice zero-score path rather than threading `CbResult` through the entire greedy search.
